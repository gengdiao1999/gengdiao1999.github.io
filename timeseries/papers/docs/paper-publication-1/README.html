二、背景（Background）

Wi-Fi 是移动设备最常用的网络接入技术之一。为了实现"快速连接"，手机、平板等终端会维护一份首选网络列表（Preferred Network List, PNL），里面记录了用户过去连接过的 AP 的 SSID。在 802.11 主动扫描（active scan）阶段，设备会周期性向外广播 probe request 帧，把 PNL 里的 SSID 全部"喊"出来。文献 [1] 指出，一台终端在某些场景下每秒可发送多达 50 个 probe request，其中 98% 的包都包含 SSID。

攻击者只要在目标区域部署无线 sniffer，就能在几秒到几分钟内捕获这些 SSID，进而造成多层次的隐私泄露：

  • 位置泄露：SSID 经常自带语义，例如 "Corp.XXX net"、"HK Airport wifi"、"Tsinghua University"。约 49% 的 SSID 是可识别的，往往直接对应工作单位、机场、酒店等地点 [2]。
  • 身份识别：将多个 SSID 组合起来，可以像指纹一样把用户从人群中"挑出来"。单日抓取的 SSID 集合能唯一识别 54.6% 的用户；一个月上升到 90.58%。
  • 偏好画像与社交关系：SSID 集合还能推断用户兴趣、身份、移动轨迹、朋友网络 [3,4,5]。

现有两类补救方案：

  1. 减少 probe request 中发送的 SSID 数量（如 Bonné 等 [6]）——通常需修改 802.11 协议栈 [7]，部署代价高；
  2. MAC 地址随机化（iOS 8+、Android 6.0+、Windows、Linux）——只解决"哪一台手机"的问题，未解决"哪些 SSID"的问题 [8]。

所以业界缺一种既不依赖协议改造、也不依赖操作系统升级的客户端/服务端协同隐私保护策略。本文正是要补这个缺口。

三、目的（Purpose / Problems Solved）

  • 痛点 1：现有保护方案对 SSID 内容泄露无效。解决方案：保留真实 SSID、额外向 PNL 注入"假 SSID"，让攻击者无法区分真假，从而打破唯一识别。
  • 痛点 2：完全消除 PNL 差异不可行（PNL 容量有限、动态更新 [15]）。解决方案：只减少差异而非消除差异，在容量上限 k 内最大化 PNL 间相似度。
  • 痛点 3：用户会移动，简单"全局共享 SSID 池"不现实。解决方案：利用用户行为相似性——只参考与目标用户 PNL 相似、且物理位置接近的"邻居/sibling"，按需"借用"SSID。
  • 痛点 4：注入假 SSID 不能影响上网体验。解决方案：把假 SSID 数量限制为 k（实验中取 5~7），并通过周期更新避免污染真实关联。
  • 痛点 5：缺乏可量化的"隐私泄露"度量。解决方案：把"被识别概率"近似为"是否拥有唯一 SSID 子集"，并用 Cosine-idf 相似度作为可计算代理。

四、核心原理（Principles）

4.1 系统总览

整篇文章的核心思想可以浓缩成一句话：把每个用户 PNL 模糊（blur）成"周围人的样子"，让攻击者无法用 SSID 集合把你区分出来。具体做法分三步：

  1. 离线度量：基于腾讯 Wi-Fi 万能钥匙 4 座城市、一个月、27M 用户的真实连接日志，统计每个用户每天的 SSID 集合、AP 经纬度、POI 类型。
  2. 相似性建模：把用户间的 SSID 集合相似度转成社交网络（共享 SSID 即连边），定义 Cosine-idf 相似度衡量两个 PNL 接近程度。
  3. PNL Refinement：每个用户取相似 + 物理邻近的 sibling，用基于协同过滤（CF）的算法从 sibling 的 PNL 中"推荐"少量未连接过的 SSID，写入设备 PNL；后续周期性轮换。

4.2 关键概念

  • PNL（Preferred Network List）：用户过去连接过的 AP 的 SSID 列表。本文用"一个月内连接过的所有 SSID"近似。
  • SSID 集合（daily SSID set）：模拟攻击者一次抓包能看到的 SSID 子集，本文用"一天内连接过的 SSID"近似。
  • Sibling：与目标用户共享 SSID 最多、Cosine-idf 相似度最高的邻居。
  • h-hop 邻居：社交网络中与目标用户距离不超过 h 跳的用户集合。1-hop 平均规模约 24 人。
  • faked SSID：被额外写入 PNL、用于混淆的 SSID；设备在 probe 时会真实广播它们。
  • Cosine-idf 相似度：在 Cosine 相似度基础上乘以 SSID 的 idf 权重。

4.3 数学原理

Cosine-idf 相似度（论文公式 1-2）：

$$ C(S_u, S_v) = \frac{\sum_{s \in S_u \cap S_v} (\log \frac{|U|}{|U_s|})^2}{\sqrt{\sum_{s \in S_u} (\log \frac{|U|}{|U_s|})^2} \cdot \sqrt{\sum_{s \in S_v} (\log \frac{|U|}{|U_s|})^2}} $$

其中 $f_s = |U_s|/|U|$ 是 SSID s 的流行度。

SSID-用户权重（tf-idf 风格，论文公式 7）：

$$ w_{u,s} = f_{u,s} \times \log\frac{|U|}{|U_s|} $$

用户对候选 SSID 的打分（论文公式 8）：

$$ R_{u,s} = w_u + \frac{\sum_{v \in U} (w_{v,s} - \bar{w}_v) \cdot \text{Sim}(S_u, S_v)}{\sum_{v \in U} \text{Sim}(S_u, S_v)} $$

优化目标（论文公式 3-5）：

$$ \max_{A \subset S} \sum_{u \in U} \sum_{v \in U \setminus u} \text{Sim}(S_u + A_u,\ S_v + A_v) \quad \text{s.t.}\quad |A_u| \le k $$

4.4 与现有技术的差异

现有方案不足本文差异
Bonné 等 [6] 减少 SSID 发送量需改协议、收益有限不改协议，仅在 PNL 端注入假 SSID
Lindqvist 等 [7] 加密 challenge-response通信开销大、AP 端改造纯客户端/App 端方案
MAC 地址随机化不防 SSID 泄露攻击者即便拿到 SSID 也无法反推真实身份
k-anonymity 等通用匿名模型SSID 是强标识符利用行为相似性，模糊化而非删除

五、算法详解（Algorithm）

5.1 输入 / 输出

  • 输入：所有用户 $U$ 在某区域某时刻的 PNL $S_u$；当前 AP 集合与位置；用户所在经纬度。
  • 输出：每个用户的新增 faked SSID 集合 $A_u$（$|A_u| \le k$），与精炼后的 PNL $P_u^T = S_u^{T-1} + A_u$。
  • 运行位置：App 服务端集中计算再下发；论文给出基于集中式管理服务器的实时流式推荐架构 [17]。

5.2 核心模块

  1. Cosine-idf 相似度计算模块。
  2. tf-idf 权重模块。
  3. CF 评分模块：用公式 8 对未连接过的 SSID 打分。
  4. BFS 候选扩展模块：1-hop 不够 k 个时拓展到 h-hop。
  5. fallback 模块：找不到候选时回退到"区域最热门 k SSID"。
  6. PNL 更新模块：Android 端通过改 /data/misc/wifi/wpa_supplicant.conf 落盘。

5.3 伪代码

def refine_pnl(U, S, k=5, max_hop=3):
    A = {}
    w = compute_tfidf_weights(U, S)
    sim = compute_cosine_idf(U, S)
    for u in U:
        Lu, visited, queue = [], set(), deque([u])
        visited.add(u)
        while queue and len(Lu) < k * 5:
            cur = queue.popleft()
            for v in neighbor_1hop(cur, sim):
                if v in visited: continue
                visited.add(v); queue.append(v)
                for s in S[v]:
                    if s in S[u]: continue
                    score = rating(w, sim, u, v, s)
                    Lu.append((s, score))
                if len(Lu) >= k * 5 and len(visited) > 20: break
        if len(Lu) == 0:
            A[u] = topk_popular_ssids(S, k)
        else:
            Lu.sort(key=lambda x: -x[1])
            A[u] = [s for s, _ in Lu[:k]]
    return A

5.4 关键数学

  • 候选分数本质是 user-based CF 预测的"伪评分" $R_{u,s}$；
  • 邻居扩展采用 BFS 限制 1-hop 优先；
  • 相似度更新周期建议为天级。

5.5 复杂度分析

单用户最坏 $O(|N_u^h| \cdot |S_v|)$；系统级可分布式化（每区域一台服务器），结合 LSH 可降到亚线性。

5.6 训练与推理

论文没有显式训练阶段：模型基于统计量"实时计算 + 周期推送"；推理：服务端算 1-hop 邻居权重 → 排序 → 推送；客户端写 PNL。

5.7 示例

火车站 (76 用户)、大学 (116)、商场 (119) 三场景。k=5 时火车站相似度 0.11→0.23，随机法只到 0.16。k=7 时三个场景分别提升 119%/140%/147%。

八、关键创新点（Key Innovations）

  • + 大规模真实测量驱动的隐私建模：基于 27M 用户 / 4M AP 的真实连接日志，定量给出"90% 用户可被唯一识别"这一惊人结论。
  • + Cosine-idf 作为隐私泄露的代理指标：把"识别风险"转化为可计算、可优化的相似度函数。
  • + 行为相似性 + 物理邻近的混合邻居构造：1-hop 优先、BFS 扩展到 h-hop，避免跨城胡乱推荐。
  • + 协同过滤改造为 PNL 模糊化器：评分同时考虑用户相似度与 SSID 重要度（tf-idf）。
  • + 工程闭环：服务端集中式流式推荐 + Android 端 PNL 改写：兼顾隐私与连接速度。

九、实验与结果（Experiments）

  • 数据集：腾讯 Wi-Fi 万能钥匙一个月连接日志（27M 用户、250M 会话、4M AP）+ POI 数据（16 类地点）。三场景：火车站 (76)、大学 (116)、商场 (119)。
  • Baseline：随机选取 k 个 SSID；k=0 不加假 SSID。
  • 主要指标：Cosine-idf 相似度、可识别用户比例、PNL 长度分布。
  • 关键结果数字：
    • 单日 SSID 集合可唯一识别 54.6% 用户；一个月 90.58%。
    • k=5 时火车站/大学/商场相似度从 0.11/0.07/0.16 提升到 0.23/0.15/0.38。
    • k=7 时 PNL 相似度提升 119.21% / 140.52% / 147.35%。
    • 原 PNL 长度 0~4 用户提升 95.57% / 139.17% / 103.41%。
    • 相似度 > 0.6 的兄弟节点 75% 距离在 2km 以内。
  • 消融实验：去掉 tf-idf → 退化为"哪个 SSID 都行"；去掉 1-hop 限制全图 CF → 噪声大；k 从 0 增到 1 提升最显著。
  • 效率分析：客户端只改一个配置文件，无额外网络流量；服务端每天周期跑一次。k=5~7 不会影响 Wi-Fi 连接速度。
