二、研究背景

2.1 KPI 异常检测的重要性

在现代互联网服务（如搜索引擎、在线购物、社交网络）中，密切监控服务性能并检测异常至关重要。典型的 KPI（关键性能指标）包括：

  • 搜索页面浏览量（PV）：成功服务的查询数量，直接影响搜索引擎收入
  • 慢响应数量（#SR）：数据中心的重要性能指标
  • 搜索响应时间 80 分位数（SRT）：衡量用户搜索体验的关键指标

这些 KPI 数据以时间序列形式呈现，即 (timestamp, value) 对。异常可能表现为抖动（jitters）、缓慢攀升（slow ramp-ups）、突增（sudden spikes）和突降（sudden dips）等不同模式。

2.2 现有异常检测方法的困境

尽管多年来已有数十种异常检测器被提出，但将它们部署到特定服务仍然是一个巨大挑战，主要体现在两个方面：

2.2.1 定义挑战

操作人员难以精确定义异常。具体表现在：

  • 操作人员通常难以完整描述其领域知识
  • 无法定量定义异常，更难以将模糊定义转化为检测器的具体参数和阈值
  • 操作人员更倾向于定性描述异常，并以典型异常案例作为示例

2.2.2 检测器挑战

选择最合适的检测器需要同时具备算法专业知识和特定服务的领域知识：

  • 最佳参数和阈值高度依赖于实际数据
  • 可能需要组合多个检测器才能处理一个 KPI
  • 需要多轮耗时的迭代来寻找合适的检测器和参数
  • 许多操作人员在几轮迭代后放弃，转而使用简单的静态阈值

2.3 KPI 数据的特征分析

论文研究了三种不同类型的 KPI，它们具有不同的数据特征：

KPI数据间隔数据长度（周）季节性变异系数 Cv
PV1 分钟25强0.48
#SR1 分钟19弱2.1
SRT60 分钟16中等0.07

其中变异系数 \(C_v = \frac{\sigma}{\mu}\)，#SR 的 \(C_v = 209\%\) 最为分散，SRT 的 \(C_v = 7\%\) 最为集中。不同的数据特征意味着不同的检测器对不同的 KPI 可能有截然不同的效果。

三、论文目的与解决的问题

3.1 核心目标

Opprentice 旨在解决以下两个核心问题：

  1. 定义挑战：通过监督机器学习，从操作人员的标注中自动捕获异常定义
  2. 检测器挑战：自动组合和调优多种现有检测器，以满足操作人员的准确度偏好

3.2 准确度偏好

操作人员可以指定准确度偏好，例如 recall ≥ 0.66 且 precision ≥ 0.66。其中：

  • Recall（召回率）：\( \text{Recall} = \frac{\text{检测到的真实异常点数}}{\text{真实异常点总数}} \)
  • Precision（精确率）：\( \text{Precision} = \frac{\text{检测到的真实异常点数}}{\text{检测到的异常点总数}} \)

选择 precision 而非 FPR（假阳性率）来衡量检测准确性，是因为异常在数据中是稀少的。Precision 等价于 1 - FDR（假发现率）。

3.3 定性目标

除了准确度目标外，Opprentice 还有一个定性目标：足够自动化，使得操作人员无需参与选择和组合合适的检测器，也无需调优它们。操作人员唯一的手动工作是定期标注异常。

四、核心原理（含数学公式）

4.1 检测器统一模型

论文将不同检测器统一为以下模型：

数据点 → 带参数的检测器 → 严重度（severity） → 严重度阈值 sThld → {1, 0}

其中严重度是一个非负值，衡量数据点的异常程度。例如：

  • Holt-Winters 使用残差误差（实际值与预测值的绝对差）衡量严重度
  • 历史平均假设数据服从高斯分布，使用偏离均值的标准差倍数衡量严重度

4.2 检测器配置作为特征提取器

将每个带特定采样参数的检测器视为一个配置（configuration），即一个特征提取器：

配置（检测器 + 采样参数）→ 数据点 → 特征

在 Opprentice 中，共使用 14 个基础检测器，通过参数采样得到 133 个配置/特征。

4.3 随机森林分类

随机森林是一种集成分类器，由多棵决策树组成。其核心思想是：一组弱学习器可以共同组成一个强学习器。

决策树基础：决策树通过贪心算法自顶向下构建，在每个层级选择最佳特征和分裂点来最大化数据分离程度。纯度衡量使用信息增益或基尼指数：

\[ \text{Gini}(D) = 1 - \sum_{i=1}^{m} p_i^2 \]

其中 \(p_i\) 是类别 \(i\) 在数据集 \(D\) 中的比例。

随机森林的随机性：

  1. 每棵树在从原始训练集中采样的子集上训练
  2. 每次分裂时只考虑特征的随机子集
  3. 所有树完全生长，不进行剪枝

分类过程：给定新数据点，每棵树给出自己的分类。例如，100 棵树中有 40 棵将数据点分类为异常，则异常概率为 40%。

4.4 PC-Score：偏好感知的阈值选择指标

PC-Score（Preference-Centric Score）基于 F-Score 扩展，显式考虑操作人员的偏好：

\[ \text{PC-Score}(r, p) = \begin{cases} \frac{2rp}{r+p} + 1, & \text{if } r \geq R \text{ and } p \geq P \\ \frac{2rp}{r+p}, & \text{otherwise} \end{cases} \]

其中 \(R\) 和 \(P\) 来自操作人员的偏好"recall ≥ R 且 precision ≥ P"。

基本思路：PC-Score 衡量 F-Score 的大小，同时为满足操作人员偏好的点添加激励常数 1。由于 F-Score 不超过 1，此激励常数确保满足偏好的点必定具有更大的 PC-Score。选择 PC-Score 最大的点对应的 cThld 作为阈值。

4.5 EWMA 阈值预测

在线检测时，需要预测未来的 cThld。采用指数加权移动平均（EWMA）进行预测：

\[ cThld_p^i = \begin{cases} \alpha \cdot cThld_b^{i-1} + (1-\alpha) \cdot cThld_p^{i-1}, & i > 1 \\ \text{5-fold 交叉验证}, & i = 1 \end{cases} \]

其中 \(cThld_b^{i-1}\) 是第 \(i-1\) 周的最佳 cThld，\(cThld_p^i\) 是第 \(i\) 周的预测 cThld，\(\alpha \in [0,1]\) 是平滑常数。论文使用 \(\alpha = 0.8\) 以快速追踪 cThld 的变化。

4.6 Precision-Recall 曲线与 AUCPR

使用 Precision-Recall（PR）曲线评估二分类器的准确性，特别适用于不平衡数据。AUCPR（PR 曲线下面积）作为单一数值指标：

\[ \text{AUCPR} = \int_0^1 p(r) \, dr \]

AUCPR 范围为 [0, 1]，值越大表示检测性能越好。

六、关键算法详解

6.1 随机森林训练算法

算法：随机森林训练
输入：训练集 D = {(x_i, y_i)}，树的数量 T，特征子集大小 m
输出：随机森林模型 {h_1, h_2, ..., h_T}

1: for t = 1 to T do
2:     从 D 中有放回地采样，得到子集 D_t
3:     构建决策树 h_t：
4:     递归构建 TreeNode(S)：
5:         if S 中所有样本同类 then
6:             返回叶节点（该类标签）
7:         end if
8:         从所有特征中随机选取 m 个特征
9:         在 m 个特征中选择最佳分裂特征 j 和分裂点 θ
10:        将 S 分为 S_left 和 S_right
11:        左子树 = TreeNode(S_left)
12:        右子树 = TreeNode(S_right)
13:        返回节点(j, θ, 左子树, 右子树)
14:    end for
15: 返回 {h_1, h_2, ..., h_T}

复杂度分析：

  • 时间复杂度：\(O(T \cdot n \cdot m \cdot \log n)\)，其中 \(T\) 为树的数量，\(n\) 为训练样本数，\(m\) 为每次考虑的特征数
  • 空间复杂度：\(O(T \cdot n_{leaf})\)，其中 \(n_{leaf}\) 为叶节点数
  • 预测复杂度：\(O(T \cdot \text{depth})\)，其中 depth 为树的平均深度

6.2 PC-Score 阈值选择算法

算法：PC-Score 阈值选择
输入：训练好的随机森林 RF，验证集 V，偏好 (R, P)
输出：最佳分类阈值 cThld*

1: cThld_list ← [0.001, 0.002, ..., 1.000]  // 步长 0.001
2: best_score ← -∞
3: best_cThld ← 0.5
4: for each cThld in cThld_list do
5:     TP ← 0, FP ← 0, FN ← 0
6:     for each (x, y) in V do
7:         prob ← RF.predict_proba(x)  // 异常概率
8:         if prob ≥ cThld then
9:             if y == 1 then TP += 1 else FP += 1
10:        else
11:            if y == 1 then FN += 1
12:        end if
13:    end for
14:    recall ← TP / (TP + FN)
15:    precision ← TP / (TP + FP)
16:    f_score ← 2 * recall * precision / (recall + precision)
17:    if recall ≥ R AND precision ≥ P then
18:        pc_score ← f_score + 1
19:    else
20:        pc_score ← f_score
21:    end if
22:    if pc_score > best_score then
23:        best_score ← pc_score
24:        best_cThld ← cThld
25:    end if
26: end for
27: 返回 best_cThld

复杂度分析：

  • 时间复杂度：\(O(K \cdot |V| \cdot T \cdot d)\)，其中 \(K = 1000\) 为候选阈值数，\(|V|\) 为验证集大小
  • 空间复杂度：\(O(1)\)，仅需存储当前最佳结果

6.3 EWMA 阈值预测

算法：EWMA 在线阈值预测
输入：历史最佳 cThld 序列 {cThld_b^1, ..., cThld_b^{i-1}}
     平滑常数 α = 0.8
输出：预测阈值 cThld_p^i

1: if i == 1 then
2:     使用 5-fold 交叉验证在初始训练集上选择 cThld_p^1
3: else
4:     cThld_p^i ← α * cThld_b^{i-1} + (1-α) * cThld_p^{i-1}
5: end if
6: 返回 cThld_p^i

7.3 参数采样策略

7.3.1 扫描参数空间

对于参数具有直观意义的检测器，直接扫描参数空间。例如 EWMA 的 α 从 {0.1, 0.3, 0.5, 0.7, 0.9} 取值，产生 5 个特征。对于多参数检测器如 Holt-Winters（α, β, γ），从 {0.2, 0.4, 0.6, 0.8} 取值，产生 4³ = 64 个特征。

7.3.2 参数估计

对于参数不直观或参数空间过大的检测器（如 ARIMA），从数据中估计"最佳"参数，只生成一个配置。由于数据特征可能随时间变化，需要定期更新参数估计。

7.4 增量重训练策略

由于异常类型可能随时间出现新类型，Opprentice 采用增量重训练：

  • 使用所有历史标注数据训练分类器
  • 定期（如每周）加入新标注数据进行重训练
  • 增量重训练能够捕获训练集中未出现的异常类型

7.5 类别不平衡处理

异常数据点远少于正常数据点，导致分类器偏向多数（正常）类。Opprentice 通过调整随机森林的分类阈值（cThld）来处理此问题，而非简单的过采样或欠采样。

7.6 MAD 鲁棒性增强

对于易受脏数据影响的检测器，使用中位数绝对偏差（MAD）替代标准差：

\[ \text{MAD} = \text{median}(|X_i - \text{median}(X)|) \]

MAD 相比标准差对缺失数据和异常值更具鲁棒性。

十五、深度技术分析：特征工程与模型选择

15.1 检测器参数采样的理论分析

Opprentice 的参数采样策略在理论上需要平衡覆盖度和计算开销。对于参数空间连续且有限的检测器，采样策略可以保证覆盖主要的行为模式。

考虑一个检测器 D 具有参数集合 Θ = {θ_1, θ_2, ..., θ_k}，每个参数 θ_i 的取值范围为 [a_i, b_i]。采样策略 S 从 Θ 中选择有限子集 Θ_S ⊂ Θ。覆盖度定义为：

\[ ext{Coverage}(S) = rac{|Θ_S|}{|Θ|} \]

对于连续参数空间，理论上覆盖度为 0，因此需要关注的是行为覆盖率（behavioral coverage），即采样点覆盖不同异常检测行为的比例。

15.2 特征冗余分析

133 个配置产生的特征之间存在不同程度的冗余。例如，简单移动平均（MA）的不同窗口长度产生的特征高度相关：

\[ ext{Corr}(MA_{win=10}, MA_{win=20}) pprox 0.85 ext{ 至 } 0.95 \]

Holt-Winters 的 64 个配置中存在更多的冗余。随机森林的随机特征子集机制使其天然抗冗余，但在极端冗余情况下仍然可能受到影响。

15.3 不平衡数据的理论分析

假设异常比例为 r（在 Opprentice 数据中 r ≈ 2.8% 至 7.8%）。在默认阈值 0.5 下，随机森林的损失函数对多数类有偏。设训练集中正常类样本数为 N，异常类样本数为 A，r = A/(A+N)。

随机森林使用投票机制，每棵树的分类误差为 ε。如果异常比例 r 很小，则多数树倾向于预测正常类。调整 cThld 相当于在投票阶段引入偏好：

\[ ext{P(异常)} = rac{1}{T} \sum_{t=1}^{T} I(h_t(x) = 1) \]

当 cThld < 0.5 时，更多的树投"异常"票即可触发告警，从而提高召回率，但精确率可能下降。

15.4 PC-Score 的数学性质

PC-Score 在 F-Score 基础上引入偏好激励，具有以下数学性质：

  • 有界性：PC-Score ∈ [0, 2]（不满足偏好时为 [0, 1]）
  • 单调性：在满足偏好区域，PC-Score 相对于 recall 和 precision 是单调递增的
  • 偏好分离性：激励常数 1 确保满足偏好的点与不满足的点在 PC-Score 上有 ≥ 1 的间隔
  • 退化为 F-Score：当偏好设置为 (R=0, P=0) 时，PC-Score 退化为 F-Score

15.5 EWMA 预测的理论分析

EWMA 预测 cThld 的误差分析。设真实最佳 cThld 序列为 {c_t}，预测序列为 {p_t}，则：

\[ p_t = α·c_{t-1} + (1-α)·p_{t-1} \]

\[ p_t - c_t = α·(c_{t-1} - c_t) + (1-α)·(p_{t-1} - c_t) \]

如果 c_t 变化平滑（即 |c_t - c_{t-1}| 较小），EWMA 能够紧密追踪。论文选择 α = 0.8 意味着预测主要依赖最近一周的最佳阈值，对突变快速响应。

扩展分析 1：随机森林的理论基础与参数敏感性

随机森林的理论保证

随机森林的理论基础来自于 Bagging 和随机子空间方法。Breiman (2001) 证明了随机森林的泛化误差上界为：

\[ PE^* \leq \bar{\rho} \cdot (1-s^2) / s^2 \]

其中 \(\bar{\rho}\) 是树之间的平均相关系数，\(s\) 是单棵树的强度。这一结果表明：

  • 降低树之间的相关性（通过随机性）可以降低泛化误差
  • 增强单棵树的预测强度可以降低泛化误差
  • 这两个目标通常可以通过增加特征随机性同时实现

参数敏感性分析

随机森林的主要参数包括：

  • 树的数量 T：增加 T 不会过拟合，但收益递减。通常 100-500 棵树足够
  • 每次分裂考虑的特征数 m：默认 \(m = \sqrt{p}\)（分类）或 \(m = p/3\)（回归），p 为总特征数
  • 最小叶节点大小：影响树的深度和复杂度

在 Opprentice 中，随机森林的参数敏感性较低，这是选择它的原因之一。论文的实验也证实了这一点。

特征重要性评估

随机森林可以计算特征重要性，方法包括：

  • Gini 重要性：基于每个特征在分裂时带来的 Gini 增益总和
  • 排列重要性：随机排列某个特征的值，观察 OOB 误差的变化

在 Opprentice 中，不同 KPI 的特征重要性不同，例如 PV 中 TSD MAD 最重要，#SR 中简单阈值最重要，SRT 中 SVD 最重要。

扩展分析 2：类别不平衡的深度处理策略

类别不平衡的理论分析

假设正类（异常）占比为 r，负类（正常）占比为 1-r。在 Opprentice 中 r ≈ 2.8%-7.8%。

对于随机森林，类别不平衡导致的问题：

  • 每棵树在大数法则下偏向多数类
  • 投票机制中，少数类需要更多树的支持
  • 默认阈值 0.5 在不平衡数据下偏向多数类

Opprentice 的处理策略

Opprentice 通过调整 cThld 而非重采样来处理类别不平衡。设随机森林输出的异常概率为 p(x)：

\[ \hat{y} = \begin{cases} 1, & p(x) \geq cThld \\ 0, & p(x) < cThld \end{cases} \]

当 cThld < 0.5 时，更少的树投"异常"即可触发告警，从而提高召回率。

扩展分析 4：增量重训练的工程优化

增量重训练的必要性

由于异常类型可能随时间变化，固定训练集无法捕获新出现的异常模式。增量重训练使用所有历史数据训练，确保模型能够适应新异常。

增量重训练的实现策略

  • 全量重训练：每次使用所有历史数据重新训练。优点：简单；缺点：数据量大时训练时间长
  • 增量更新：在现有模型基础上加入新数据。优点：快速；缺点：可能积累偏差
  • 滑动窗口：仅使用最近 N 周的数据。优点：适应变化快；缺点：丢失早期异常

Opprentice 的选择

论文使用增量重训练（全量 + 新数据），因为在实际部署中：

  • 标注数据量增长缓慢（每周数分钟标注）
  • 训练时间可控（<5 分钟）
  • 全量数据确保不遗漏历史异常模式

扩展分析 5：EWMA 阈值预测的稳定性分析

EWMA 的数学性质

EWMA 的递推公式为：

\[ p_t = \alpha \cdot b_{t-1} + (1-\alpha) \cdot p_{t-1} \]

展开后：

\[ p_t = \alpha \sum_{i=0}^{t-1} (1-\alpha)^i b_{t-1-i} + (1-\alpha)^t p_0 \]

当 \(\alpha = 0.8\) 时，最近 1 周的权重为 0.8，2 周前为 0.16，3 周前为 0.032，因此 EWMA 主要关注最近 1-2 周的数据。

EWMA 的稳定性条件

EWMA 是稳定的当且仅当 \(|1-\alpha| < 1\)，即 \(0 < \alpha < 2\)。在 Opprentice 中 \(\alpha = 0.8\)，满足稳定性条件。

与 5-fold 交叉验证的对比

5-fold 交叉验证在 Opprentice 中效果较差的原因：

  • 最佳 cThld 在不同周之间变化很大
  • 交叉验证选择的 cThld 是所有历史周的平均最佳，可能不适合未来周
  • EWMA 更关注最近的数据，因此预测更接近未来周的最佳值

扩展分析 6：14 种检测器的详细分析

检测器分类与原理

14 种检测器可以分为以下类别：

基于阈值的方法

  • 简单阈值：直接对原始值设阈值。适用于稳定波动小的 KPI

基于差分的方法

  • Diff：计算当前值与历史值的差。三种配置：上一时隙、上一日、上一周
  • Diff 的 MA：对差值做移动平均，用于检测连续抖动

基于移动平均的方法

  • 简单 MA：对历史值做等权移动平均，残差为严重度
  • 加权 MA：对近期值赋予更高权重
  • EWMA：指数加权移动平均，\(\hat{x}_t = \alpha x_{t-1} + (1-\alpha) \hat{x}_{t-1}\)

基于时间序列分解的方法

  • TSD：将序列分解为趋势、季节和残差，残差为严重度
  • TSD MAD：使用 MAD 替代标准差，提高对脏数据的鲁棒性

基于统计的方法

  • 历史平均：用历史同时段的均值和标准差判断异常
  • 历史 MAD：使用 MAD 替代标准差

基于预测模型的方法

  • Holt-Winters：三次指数平滑，分别对水平、趋势、季节建模。三个参数 \(\alpha, \beta, \gamma\)
  • ARIMA：自回归积分滑动平均模型，参数从数据估计

基于矩阵分解的方法

  • SVD：奇异值分解，提取主要模式后用残差作为严重度

基于频域的方法

  • Wavelet：小波变换分解频域成分，高频成分为严重度
