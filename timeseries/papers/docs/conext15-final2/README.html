二、研究背景

2.1 软件变更的重要性与挑战

在搜索引擎、在线购物、社交网络等大型互联网服务中，软件变更是日常运维的重要组成部分：

  • 每天进行数十甚至上百次软件变更
  • 每次变更可能影响数千台服务器
  • 每次变更需要监控成百上千个关键性能指标（KPI）
  • 及时发现性能退化对保障用户体验和业务收入至关重要

然而，软件变更的影响评估面临巨大挑战：

  1. 规模挑战：每天数万次软件变更，每次变更影响数千个KPI，人工评估不可行
  2. 及时性挑战：需要在分钟级时间内发现问题，及时回滚
  3. 鲁棒性挑战：KPI具有多样性（季节性、波动性、非平稳性）
  4. 因果性挑战：需要区分软件变更的真实影响与其他因素（如流量变化、硬件故障）

2.2 灰度发布（Dark Launching）

现代互联网服务通常采用灰度发布策略进行软件变更：

  1. 首先将变更部署到一小部分服务器（处理组）
  2. 大部分服务器保持不变（控制组）
  3. 持续监控两组服务器的KPI差异
  4. 如果KPI表现正常，逐步扩大部署范围
  5. 如果发现问题，立即回滚变更

2.3 KPI的多样性

互联网服务监控的KPI具有高度多样性：

KPI类型示例特点
用户体验指标网页响应延迟、加载时间强季节性、非平稳
服务性能指标广告点击率、QPS高波动性、季节性
系统健康指标服务器CPU、内存利用率相对平稳、周期性
网络指标带宽、连接数、丢包率突发性、非平稳
业务指标交易量、用户数强季节性、趋势性

2.5 两个关键技术挑战

2.5.1 挑战1：快速鲁棒的行为变化检测

需要设计一个算法能够：

  • 在变化发生后尽快检测到
  • 对各种类型的KPI都具有鲁棒性
  • 计算效率高，可扩展性好

2.5.2 挑战2：因果推断

需要区分：

  • 真正由软件变更引起的变化
  • 由其他因素（如流量变化、硬件故障）引起的变化
  • 随机相关性导致的假阳性

四、核心原理

4.1 奇异谱变换（Singular Spectrum Transform, SST）

4.1.1 SST基本原理

SST是一种基于矩阵奇异值分解（SVD）的时序分析方法，核心思想是：

  1. 将时间序列构造成轨迹矩阵
  2. 对轨迹矩阵进行SVD分解
  3. 计算原始序列与重构序列的差异作为异常分数

4.1.2 轨迹矩阵构造

给定时序 $x = [x_1, x_2, ..., x_T]$，构造轨迹矩阵 $X \in \mathbb{R}^{L \times W}$：

$$X = \begin{bmatrix} x_1 & x_2 & \dots & x_W \\ x_2 & x_3 & \dots & x_{W+1} \\ \vdots & \vdots & \ddots & \vdots \\ x_L & x_{L+1} & \dots & x_T \end{bmatrix}$$

其中：

  • $L$ 是窗口长度（嵌入维度）
  • $W = T - L + 1$ 是窗口数量

4.1.3 SVD分解

对轨迹矩阵进行SVD分解：

$$X = U \Sigma V^T$$

其中：

  • $U \in \mathbb{R}^{L \times L}$ 是左奇异向量矩阵
  • $\Sigma \in \mathbb{R}^{L \times W}$ 是奇异值对角矩阵
  • $V \in \mathbb{R}^{W \times W}$ 是右奇异向量矩阵

4.1.4 序列重构与异常分数

使用前 $r$ 个主成分重构序列：

$$\hat{X} = U_r \Sigma_r V_r^T$$

其中 $U_r, \Sigma_r, V_r$ 是对应前 $r$ 个最大奇异值的子矩阵。

异常分数定义为原始矩阵与重构矩阵的残差：

$$S(t) = \|X(t) - \hat{X}(t)\|_2$$

当异常分数超过阈值时，判定为发生了变化。

4.1.5 SST的优势

  • 自适应：不需要假设数据分布
  • 多尺度：通过窗口长度捕捉不同时间尺度的模式
  • 鲁棒性：对噪声和异常值不敏感
  • 无监督：不需要标注数据

4.2 多尺度SST

为了检测不同类型的变化（突变、渐变、周期性变化），FUNNEL采用多尺度SST：

  1. 使用多个不同的窗口长度 $L_1, L_2, ..., L_k$
  2. 在每个尺度上计算异常分数
  3. 融合多个尺度的结果得到最终检测

多尺度SST的优势：

  • 小窗口适合检测突变
  • 大窗口适合检测渐变和趋势变化
  • 多尺度融合提高鲁棒性

4.3 双重差分（Difference-in-Differences, DiD）

4.3.1 DiD基本思想

DiD是计量经济学中常用的因果推断方法，用于评估政策干预的效果。在软件变更评估中：

  • 处理组（Treated）：部署了新软件的服务器
  • 控制组（Control）：未部署新软件的服务器
  • 变更前（Pre）：软件变更前的时间段
  • 变更后（Post）：软件变更后的时间段

4.3.2 DiD计算公式

DiD效应计算为：

$$DiD = (\bar{Y}_{T,Post} - \bar{Y}_{T,Pre}) - (\bar{Y}_{C,Post} - \bar{Y}_{C,Pre})$$

其中：

  • $\bar{Y}_{T,Post}$：处理组变更后的平均KPI值
  • $\bar{Y}_{T,Pre}$：处理组变更前的平均KPI值
  • $\bar{Y}_{C,Post}$：控制组变更后的平均KPI值
  • $\bar{Y}_{C,Pre}$：控制组变更前的平均KPI值

DiD的核心洞察是：控制组的变化代表了所有非变更因素（如流量变化、时间趋势）的影响，因此减去这部分影响后，剩余的就是软件变更的净效应。

4.3.3 DiD统计检验

为了检验DiD效应的统计显著性，使用t检验：

$$t = \frac{DiD}{SE(DiD)}$$

其中 $SE(DiD)$ 是DiD估计的标准误。当 $|t|$ 超过临界值（如1.96对应95%置信度）时，认为软件变更具有统计显著的影响。

4.4 FUNNEL的检测流程

FUNNEL的检测流程融合了SST和DiD：

  1. 数据收集：收集处理组和控制组的KPI数据
  2. 预处理：去除异常值、填充缺失值、标准化
  3. SST检测：对处理组和控制组分别进行SST变化检测
  4. DiD分析：计算DiD效应并进行统计检验
  5. 结果融合：综合SST检测和DiD分析结果
  6. 告警生成：对检测到的显著影响生成告警

六、关键算法详解

6.1 基本SST算法

算法描述

基本SST算法对时间序列进行滑动窗口分析，在每个时间点构造轨迹矩阵，进行SVD，计算残差作为异常分数。

伪代码

算法1：基本奇异谱变换（SST）

输入：
    - 时间序列 x = [x_1, x_2, ..., x_T]
    - 窗口长度 L
    - 主成分数 r
    - 阈值 τ

输出：
    - 变化点位置集合 C

函数 BasicSST(x, L, r, τ):
    T = len(x)
    W = T - L + 1
    anomaly_scores = zeros(T)

    // 1. 构造轨迹矩阵
    X = zeros(L, W)
    for i from 1 to L:
        for j from 1 to W:
            X[i, j] = x[i + j - 1]

    // 2. SVD分解
    U, Σ, V^T = SVD(X)

    // 3. 使用前r个主成分重构
    U_r = U[:, 1:r]
    Σ_r = Σ[1:r, 1:r]
    V_r = V[:, 1:r]
    X_hat = U_r * Σ_r * V_r^T

    // 4. 计算残差
    R = X - X_hat

    // 5. 计算每个时间点的异常分数
    for t from 1 to T:
        // 收集所有包含x_t的矩阵元素
        elements = []
        for i from max(1, t-W+1) to min(L, t):
            j = t - i + 1
            if 1 <= j <= W:
                elements.append(R[i, j])
        anomaly_scores[t] = mean(abs(elements))

    // 6. 阈值判断
    C = [t for t in 1:T where anomaly_scores[t] > τ]

    return C, anomaly_scores

复杂度分析

操作时间复杂度空间复杂度
轨迹矩阵构造$O(LW)$$O(LW)$
SVD分解$O(LW^2)$$O(L^2 + W^2)$
矩阵重构$O(LrW)$$O(LW)$
残差计算$O(LW)$$O(LW)$
总复杂度$O(LW^2)$$O(LW)$

基本SST的时间复杂度为 $O(LW^2)$，对于大窗口（如L=100, W=1000）计算量较大，需要优化。

6.2 增量SST算法

算法描述

增量SST利用相邻时间窗口的重叠性，避免每次重新计算完整的SVD，显著提高计算效率。

伪代码

算法2：增量奇异谱变换（Incremental SST）

输入：
    - 时间序列 x = [x_1, x_2, ..., x_T]
    - 窗口长度 L
    - 主成分数 r
    - 阈值 τ
    - 更新频率 Δt

输出：
    - 变化点位置集合 C

函数 IncrementalSST(x, L, r, τ, Δt):
    T = len(x)
    anomaly_scores = zeros(T)
    C = []

    // 初始窗口的SVD
    X_initial = construct_trajectory_matrix(x[1:L+Δt], L)
    U, Σ, V^T = SVD(X_initial)

    for t from L+Δt+1 to T step Δt:
        // 增量更新轨迹矩阵
        new_columns = construct_new_columns(x[t-Δt+1:t], L)
        X = append_columns(X, new_columns)
        X = remove_old_columns(X, Δt)

        // 增量SVD更新
        U, Σ, V^T = incremental_svd_update(U, Σ, V^T, new_columns)

        // 重构和残差计算
        U_r = U[:, 1:r]
        Σ_r = Σ[1:r, 1:r]
        V_r = V[:, 1:r]
        X_hat = U_r * Σ_r * V_r^T
        R = X - X_hat

        // 计算异常分数
        for i from t-Δt+1 to t:
            anomaly_scores[i] = compute_anomaly_score(R, i, L)

            if anomaly_scores[i] > τ:
                C.append(i)

    return C, anomaly_scores

函数 incremental_svd_update(U, Σ, V, new_cols):
    // 使用增量SVD算法更新
    // 参考：Brand, M. (2006). Fast low-rank modifications of the thin SVD.
    k = size(new_cols, 2)

    // 正交投影
    m = U^T * new_cols
    p = new_cols - U * m
    p_norm = norm(p, 'fro')

    if p_norm > ε:
        P = p / p_norm
        U_extended = [U, P]
        Z = zeros(size(U, 2), k)
        I = eye(k)
        Σ_extended = [[Σ, Z], [m', p_norm*I]]
    else:
        U_extended = U
        Σ_extended = [Σ, m]

    // 重新SVD
    U_new, Σ_new, V_new^T = SVD(Σ_extended)
    U = U_extended * U_new
    Σ = Σ_new
    V = [V, zeros(size(V, 1), k); zeros(k, size(V, 2)), eye(k)] * V_new

    return U, Σ, V

复杂度分析

操作时间复杂度说明
增量SVD更新$O(Lr^2)$r通常远小于L和W
残差计算$O(LΔt)$只计算更新的部分
每Δt时间总复杂度$O(L(r^2 + Δt))$远优于基本SST

增量SST将计算复杂度从 $O(LW^2)$ 降低到 $O(L(r^2 + Δt))$，计算速度提升数个数量级。

6.3 矩阵压缩优化

算法描述

利用轨迹矩阵的特殊结构（Hankel矩阵）进行压缩，减少存储和计算开销。

伪代码

算法3：矩阵压缩SST

输入：
    - 时间序列 x = [x_1, x_2, ..., x_T]
    - 窗口长度 L
    - 主成分数 r
    - 阈值 τ

输出：
    - 变化点位置集合 C

函数 CompressedSST(x, L, r, τ):
    T = len(x)
    W = T - L + 1

    // 利用Hankel矩阵性质，只需要存储原始序列
    // 轨迹矩阵元素 X[i,j] = x[i+j-1]

    // 1. 计算协方差矩阵（利用结构优化）
    C = compute_hankel_covariance(x, L)

    // 2. 特征值分解（替代SVD）
    eigenvalues, eigenvectors = EIG(C)

    // 3. 选择前r个主成分
    U_r = eigenvectors[:, 1:r]

    // 4. 隐式计算重构和残差
    anomaly_scores = zeros(T)

    for t from 1 to T:
        // 不构造完整矩阵，直接计算残差
        anomaly_scores[t] = implicit_residual(x, U_r, t, L)

    C = [t for t in 1:T where anomaly_scores[t] > τ]

    return C, anomaly_scores

function compute_hankel_covariance(x, L):
    T = len(x)
    C = zeros(L, L)
    for lag from 0 to L-1:
        autocorr = autocorrelation(x, lag)
        for i from 1 to L-lag:
            C[i, i+lag] = autocorr
            C[i+lag, i] = autocorr
    return C

function implicit_residual(x, U_r, t, L):
    // 隐式计算时间t处的残差
    // 利用Hankel结构避免构造完整矩阵
    window_start = max(1, t - L + 1)
    window_end = min(len(x), t)

    local_sum = 0
    count = 0

    for i from window_start to window_end:
        j = t - i + 1
        if 1 <= j <= len(x) - L + 1:
            // 投影到主成分空间
            projection = compute_projection(x, U_r, i, j)
            local_sum += (x[t] - projection)^2
            count += 1

    return sqrt(local_sum / count)

复杂度分析

操作时间复杂度空间复杂度
协方差矩阵计算$O(LT)$$O(L^2)$
特征值分解$O(L^3)$$O(L^2)$
隐式残差计算$O(LrT)$$O(1)$ （在线计算）
总复杂度$O(LT + L^3 + LrT)$$O(L^2 + Lr)$

矩阵压缩将空间复杂度从 $O(LW)$ 降低到 $O(L^2 + Lr)$，对于长序列特别有效。

6.4 多尺度融合算法

算法描述

融合多个尺度的检测结果，提高检测的鲁棒性和准确性。

伪代码

算法4：多尺度SST融合

输入：
    - 时间序列 x = [x_1, x_2, ..., x_T]
    - 尺度集合 scales = [L_1, L_2, ..., L_k]
    - 主成分数 r
    - 融合权重 w = [w_1, w_2, ..., w_k]

输出：
    - 融合异常分数 scores
    - 变化点集合 C

函数 MultiScaleSST(x, scales, r, w):
    T = len(x)
    k = len(scales)
    scale_scores = zeros(k, T)

    // 1. 在每个尺度上计算SST
    for i from 1 to k:
        L = scales[i]
        _, scale_scores[i, :] = CompressedSST(x, L, r, τ=inf)

    // 2. 多尺度融合
    scores = zeros(T)
    for t from 1 to T:
        // 加权平均
        weighted_sum = 0
        weight_sum = 0
        for i from 1 to k:
            if t >= scales[i]:  // 确保有足够数据
                weighted_sum += w[i] * scale_scores[i, t]
                weight_sum += w[i]

        if weight_sum > 0:
            scores[t] = weighted_sum / weight_sum
        else:
            scores[t] = 0

    // 3. 自适应阈值
    τ = mean(scores) + 3 * std(scores)
    C = [t for t in 1:T where scores[t] > τ]

    return C, scores

// 自适应权重学习
function learn_weights(x_train, labels, scales):
    k = len(scales)
    scale_scores = zeros(k, len(x_train))

    for i from 1 to k:
        _, scale_scores[i, :] = CompressedSST(x_train, scales[i], r, τ=inf)

    // 使用逻辑回归学习权重
    w = logistic_regression(scale_scores', labels)
    return w

6.5 DiD检测算法

算法描述

对检测到的疑似变化，使用DiD方法验证其是否真正由软件变更引起。

伪代码

算法5：双重差分（DiD）验证

输入：
    - 处理组KPI序列：treated = [T_1, T_2, ..., T_T]
    - 控制组KPI序列：control = [C_1, C_2, ..., C_T]
    - 变更时间点：t_change
    - 置信水平：α = 0.95

输出：
    - 是否有显著影响：has_effect
    - 影响大小：effect_size
    - 统计显著性：p_value

function DiD_Analysis(treated, control, t_change, α):
    T = len(treated)
    Pre_T = treated[1:t_change-1]
    Post_T = treated[t_change:T]
    Pre_C = control[1:t_change-1]
    Post_C = control[t_change:T]

    // 1. 计算各组均值
    mean_Pre_T = mean(Pre_T)
    mean_Post_T = mean(Post_T)
    mean_Pre_C = mean(Pre_C)
    mean_Post_C = mean(Post_C)

    // 2. 计算变化量
    delta_T = mean_Post_T - mean_Pre_T
    delta_C = mean_Post_C - mean_Pre_C

    // 3. 计算DiD效应
    DiD = delta_T - delta_C

    // 4. 计算标准误
    n_T_pre = len(Pre_T)
    n_T_post = len(Post_T)
    n_C_pre = len(Pre_C)
    n_C_post = len(Post_C)

    var_Pre_T = var(Pre_T)
    var_Post_T = var(Post_T)
    var_Pre_C = var(Pre_C)
    var_Post_C = var(Post_C)

    se_DiD = sqrt(var_Post_T/n_T_post + var_Pre_T/n_T_pre +
                   var_Post_C/n_C_post + var_Pre_C/n_C_pre)

    // 5. t检验
    t_stat = DiD / se_DiD
    df = n_T_pre + n_T_post + n_C_pre + n_C_post - 4
    p_value = 2 * (1 - t_cdf(abs(t_stat), df))

    // 6. 判断显著性
    critical_value = t_ppf(1 - α/2, df)
    has_effect = abs(t_stat) > critical_value

    // 7. 计算效应大小（标准化）
    pooled_std = sqrt((var_Pre_T + var_Post_T + var_Pre_C + var_Post_C) / 4)
    effect_size = DiD / pooled_std

    return has_effect, effect_size, p_value, t_stat, DiD

// 批量DiD分析
function BatchDiD_Analysis(kpi_list, treated_servers, control_servers, t_change):
    results = []

    for kpi in kpi_list:
        treated_data = aggregate_kpi(kpi, treated_servers)
        control_data = aggregate_kpi(kpi, control_servers)

        has_effect, effect_size, p_value, t_stat, DiD =
            DiD_Analysis(treated_data, control_data, t_change, α=0.95)

        results.append({
            'kpi': kpi,
            'has_effect': has_effect,
            'effect_size': effect_size,
            'p_value': p_value,
            't_statistic': t_stat,
            'did': DiD
        })

    return results

7.2 数据预处理

7.2.1 缺失值处理

KPI数据中常见缺失值，FUNNEL采用多层级处理策略：

  • 短期缺失（< 5个点）：线性插值
  • 中期缺失（5-30个点）：历史同时期平均值填充
  • 长期缺失（> 30个点）：标记为不可靠，跳过该时段

7.2.2 异常值处理

使用3σ原则识别并处理极端异常值：

$$\text{outlier} = |x_t - \mu| > 3\sigma$$

对识别出的异常值，使用前后邻居的中位数替换。

7.2.3 标准化

对每个KPI进行Z-score标准化：

$$x_t' = \frac{x_t - \mu}{\sigma + \epsilon}$$

其中 $\epsilon = 10^{-8}$ 防止除零。

7.3 控制组选择

DiD分析的关键是选择合适的控制组。FUNNEL采用以下策略：

  1. 硬件相同：与处理组服务器硬件配置完全相同
  2. 负载相似：变更前的KPI模式和负载水平相似
  3. 无干扰：控制组服务器未部署其他变更
  4. 大小合适：控制组大小为处理组的1-2倍

控制组选择的伪代码：

算法6：控制组智能选择

输入：
    - 所有可用服务器列表 servers
    - 处理组服务器列表 treated_servers
    - 历史KPI数据 kpi_history

输出：
    - 控制组服务器列表 control_servers

function SelectControlGroup(servers, treated_servers, kpi_history):
    candidates = [s for s in servers if s not in treated_servers]

    // 1. 硬件匹配过滤
    candidates = filter_same_hardware(candidates, treated_servers)

    // 2. 计算相似度
    treated_profile = compute_kpi_profile(treated_servers, kpi_history)
    similarities = []

    for s in candidates:
        s_profile = compute_kpi_profile([s], kpi_history)
        sim = cosine_similarity(treated_profile, s_profile)
        similarities.append((s, sim))

    // 3. 按相似度排序
    sort(similarities, key=lambda x: x[1], reverse=True)

    // 4. 选择Top-N作为控制组
    control_size = min(2 * len(treated_servers), len(similarities))
    control_servers = [s for s, sim in similarities[:control_size]]

    return control_servers

7.4 多KPI聚合

对于大量KPI的检测结果，FUNNEL采用层次化聚合策略：

  1. 单KPI检测：对每个KPI独立进行SST和DiD检测
  2. 服务器级聚合：聚合同一服务器的所有KPI结果
  3. 集群级聚合：聚合同一集群的所有服务器结果
  4. 服务级聚合：聚合整个服务的检测结果

聚合时使用加权投票：

$$\text{AggregateScore} = \frac{\sum_{i=1}^{N} w_i \cdot \text{Score}_i}{\sum_{i=1}^{N} w_i}$$

其中权重 $w_i$ 基于KPI的重要性确定。

十二、局限性与未来工作

12.1 当前局限性

  1. 依赖控制组：DiD需要合适的控制组，对于全量发布难以应用
  2. 静态窗口：窗口大小固定，无法自适应调整
  3. 线性假设：SST基于线性模型，对非线性模式捕捉能力有限
  4. 缺乏根因分析：只能检测是否有变化，无法定位变化的根本原因
  5. 对渐变的敏感性：对于非常缓慢的变化，检测延迟仍然较长

12.2.1 算法改进

12.2.2 功能扩展

14.2 关键公式汇总

  1. 轨迹矩阵构造： $$X_{i,j} = x_{i+j-1}, \quad 1 \leq i \leq L, 1 \leq j \leq W$$
  2. SVD分解： $$X = U \Sigma V^T$$
  3. 序列重构： $$\hat{X} = U_r \Sigma_r V_r^T$$
  4. 异常分数： $$S(t) = \|X(t) - \hat{X}(t)\|_2$$
  5. DiD效应： $$DiD = (\bar{Y}_{T,Post} - \bar{Y}_{T,Pre}) - (\bar{Y}_{C,Post} - \bar{Y}_{C,Pre})$$
  6. DiD t统计量： $$t = \frac{DiD}{SE(DiD)}$$
  7. 效应大小（Cohen's d）： $$d = \frac{DiD}{\sigma_{pooled}}$$

14.3.1 Python版本的SST实现

import numpy as np
from numpy.linalg import svd

def singular_spectrum_transform(x, L, r=3):
    """
    基本奇异谱变换实现

    参数:
        x: 输入时间序列, shape (T,)
        L: 窗口长度
        r: 主成分数

    返回:
        anomaly_scores: 异常分数序列, shape (T,)
    """
    T = len(x)
    W = T - L + 1

    # 1. 构造轨迹矩阵 (Hankel矩阵)
    X = np.zeros((L, W))
    for i in range(L):
        X[i, :] = x[i:i+W]

    # 2. SVD分解
    U, s, Vt = svd(X, full_matrices=False)

    # 3. 使用前r个主成分重构
    U_r = U[:, :r]
    s_r = s[:r]
    V_r = Vt[:r, :]
    X_hat = U_r @ np.diag(s_r) @ V_r

    # 4. 计算残差
    R = X - X_hat

    # 5. 计算每个时间点的异常分数
    anomaly_scores = np.zeros(T)
    counts = np.zeros(T)

    for i in range(L):
        for j in range(W):
            t = i + j
            anomaly_scores[t] += abs(R[i, j])
            counts[t] += 1

    anomaly_scores = anomaly_scores / counts
    return anomaly_scores

def multi_scale_sst(x, scales=[10, 20, 40, 80], r=3):
    """
    多尺度SST融合
    """
    T = len(x)
    scores_list = []

    for L in scales:
        if L >= T:
            continue
        scores = singular_spectrum_transform(x, L, r)
        scores_list.append(scores)

    # 简单平均融合
    if len(scores_list) == 0:
        return np.zeros(T)

    max_len = max(len(s) for s in scores_list)
    padded_scores = []
    for s in scores_list:
        pad_len = max_len - len(s)
        padded = np.pad(s, (pad_len, 0), mode='edge')
        padded_scores.append(padded)

    return np.mean(padded_scores, axis=0)

def did_analysis(treated, control, t_change, alpha=0.95):
    """
    双重差分分析

    参数:
        treated: 处理组序列, shape (T,)
        control: 控制组序列, shape (T,)
        t_change: 变更时间点
        alpha: 置信水平

    返回:
        has_effect: 是否有显著影响
        effect_size: 效应大小
        p_value: p值
    """
    from scipy import stats

    # 分割为变更前后
    pre_T = treated[:t_change]
    post_T = treated[t_change:]
    pre_C = control[:t_change]
    post_C = control[t_change:]

    # 计算均值
    mean_pre_T = np.mean(pre_T)
    mean_post_T = np.mean(post_T)
    mean_pre_C = np.mean(pre_C)
    mean_post_C = np.mean(post_C)

    # 计算DiD效应
    delta_T = mean_post_T - mean_pre_T
    delta_C = mean_post_C - mean_pre_C
    did = delta_T - delta_C

    # 计算标准误
    n_T_pre = len(pre_T)
    n_T_post = len(post_T)
    n_C_pre = len(pre_C)
    n_C_post = len(post_C)

    var_pre_T = np.var(pre_T, ddof=1)
    var_post_T = np.var(post_T, ddof=1)
    var_pre_C = np.var(pre_C, ddof=1)
    var_post_C = np.var(post_C, ddof=1)

    se_did = np.sqrt(var_post_T/n_T_post + var_pre_T/n_T_pre +
                     var_post_C/n_C_post + var_pre_C/n_C_pre)

    # t检验
    t_stat = did / se_did
    df = n_T_pre + n_T_post + n_C_pre + n_C_post - 4
    p_value = 2 * (1 - stats.t.cdf(abs(t_stat), df))

    # 判断显著性
    critical_value = stats.t.ppf(1 - (1-alpha)/2, df)
    has_effect = abs(t_stat) > critical_value

    # 效应大小
    pooled_std = np.sqrt((var_pre_T + var_post_T + var_pre_C + var_post_C) / 4)
    effect_size = did / pooled_std

    return has_effect, effect_size, p_value, t_stat, did

# 使用示例
if __name__ == "__main__":
    # 生成模拟数据
    np.random.seed(42)
    T = 200
    t_change = 100

    # 正常模式
    x_normal = np.random.normal(0, 1, T)

    # 处理组：在t_change处有变化
    treated = x_normal.copy()
    treated[t_change:] += 0.5  # 50%的均值漂移

    # 控制组：无变化
    control = x_normal.copy()

    # 多尺度SST检测
    scores = multi_scale_sst(treated, scales=[10, 20, 40], r=3)
    threshold = np.mean(scores) + 3 * np.std(scores)
    detected = np.where(scores > threshold)[0]
    print(f"SST检测到变化点: {detected}")

    # DiD验证
    has_effect, effect_size, p_value, t_stat, did = \
        did_analysis(treated, control, t_change)
    print(f"DiD结果: has_effect={has_effect}, "
          f"effect_size={effect_size:.2f}, p={p_value:.4f}")

14.4 常见问题（FAQ）

Q1：FUNNEL适用于什么场景？

FUNNEL特别适合以下场景：

  • 大规模互联网服务的软件发布监控
  • 灰度发布、A/B测试的效果评估
  • 需要快速检测软件变更影响的场景
  • KPI种类多、模式复杂的系统

Q2：如何选择合适的窗口大小？

窗口大小的选择取决于：

  • KPI的采样频率（频率越高窗口可越大）
  • 需要检测的变化类型（突变用小窗口，渐变用大窗口）
  • 计算资源约束（窗口越大计算量越大）

一般建议从 [10, 20, 40, 80] 开始，根据实际效果调整。

Q3：DiD对控制组有什么要求？

理想的控制组应该满足：

  • 与处理组硬件配置相同
  • 变更前的KPI模式相似
  • 未受到其他变更的干扰
  • 大小为处理组的1-2倍

Q4：FUNNEL的误报率如何？

在百度的真实部署中，FUNNEL的精确率达到98.2%，即每100次告警只有约2次误报。这主要得益于DiD方法有效排除了其他因素的干扰。

Q5：FUNNEL能检测哪些类型的变化？

FUNNEL可以检测：

  • 均值突变（如CPU突然升高）
  • 均值渐变（如性能缓慢下降）
  • 方差变化（如响应时间波动变大）
  • 周期性模式变化
  • 趋势变化

Q6：FUNNEL的计算资源需求是多少？

对于1000个KPI，每个KPI 1天数据（1分钟采样）：

  • 内存：约4GB
  • CPU：8核，处理时间约10秒
  • 存储：约10GB原始数据 + 1GB处理结果
