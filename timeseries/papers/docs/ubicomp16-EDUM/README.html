2. 研究背景

2.1 教育测量的传统困境

课堂教学作为高等教育的核心形式，其质量评估长期面临以下挑战：

  1. 数据采集成本高昂：传统方法依赖人工观察、问卷调查、志愿者招募，单门课程需要投入大量人力物力
  2. 样本规模受限：大多数教育学研究样本量在30-100人，统计显著性不足，结论难以推广
  3. 主观性偏差：观察者效应、回答偏差、记忆偏差严重影响测量结果的客观性
  4. 侵入性强：人工观察会干扰正常教学，传感器穿戴可能引起学生不适
  5. 时间粒度粗糙：传统问卷调查通常在课程结束后进行，无法捕捉分钟级的行为变化

2.2 WiFi基础设施的普适性

截至2016年，全球高校已基本完成校园WiFi全覆盖：

  • 清华大学拥有2786个AP，覆盖114栋建筑
  • 学生移动设备携带率接近100%，日均WiFi连接时长超过10小时
  • WiFi网络本身具备精细的位置感知能力（RSSI信号强度）
  • SNMP协议可实时采集设备连接状态与流量统计

2.3 研究机遇与挑战

3. 论文目的与解决的问题

3.1 研究目标

EDUM系统的设计目标是构建一个大规模、非侵入式、低成本、精细化的课堂教育行为测量平台：

  1. 出勤测量：自动检测学生是否按时上课、是否迟到、是否早退
  2. 专注度测量：通过移动设备交互状态推断学生课堂专注度
  3. 吸引力评估：量化不同课程、不同教师、不同时间段的课堂吸引力
  4. 行为模式挖掘：发现学生选课、出勤、学习行为的隐藏模式
  5. 教学改进支持：为教师和教学管理部门提供数据驱动的决策依据

3.2 需要解决的核心问题

问题1：课程基础信息的自动发现

校方的课程管理系统通常不对外开放，EDUM需要解决：

  • 如何在没有校方课程数据的情况下，自动发现课程的时间安排？
  • 如何自动推断课程所在的具体教室位置？
  • 如何识别哪些学生选修了某门课程？

问题2：基于WiFi的精细化位置推断

传统室内定位需要预先采集指纹库，EDUM需要：

  • 如何在无预先指纹训练的情况下，利用众包数据动态建立位置模型？
  • 如何处理RSSI信号波动、多径效应、AP负载变化带来的噪声？
  • 如何在保证隐私的前提下进行跨学生的位置聚合分析？

问题3：教育行为语义的映射

从WiFi连接数据到教育行为存在语义鸿沟：

  • 如何区分"在教室但没听课"与"在教室且认真听课"？
  • 如何定义和量化"课堂吸引力"这一抽象概念？
  • 如何验证测量指标的有效性与可靠性？

4. 核心原理（含数学公式）

4.1 AP邻近度计算

EDUM首先计算AP之间的邻近度矩阵，用于后续的位置推断。对于两个AP \(i\) 和 \(j\)，其邻近度定义为：

\[ \text{Proximity}(i, j) = \frac{N_{ij}}{\sqrt{N_i \times N_j}} \]

其中：

  • \(N_{ij}\) 表示同时检测到设备连接到AP \(i\) 和AP \(j\) 的时间窗口数量
  • \(N_i\) 和 \(N_j\) 分别表示AP \(i\) 和AP \(j\) 出现的总时间窗口数

该公式本质是Jaccard相似度的变体，衡量两个AP在空间上的接近程度。值越接近1，表示两个AP距离越近；值越接近0，表示距离越远。

4.2 基于RSSI的位置指纹匹配

对于特定教室，EDUM构建其RSSI指纹向量 \(F_c = (f_{c1}, f_{c2}, ..., f_{cM})\)，其中 \(f_{ci}\) 是教室 \(c\) 中AP \(i\) 的平均信号强度。对于待匹配的设备观测向量 \(O_t = (o_{t1}, o_{t2}, ..., o_{tM})\)，匹配度计算为：

\[ \text{MatchScore}(F_c, O_t) = \exp\left(-\frac{1}{K} \sum_{i \in S} |f_{ci} - o_{ti}|\right) \]

其中：

  • \(S\) 是指纹与观测中共同出现的AP集合
  • \(K = |S|\) 是共同AP的数量
  • 指数函数将距离转换为相似度，范围在(0, 1]之间

当匹配度超过预设阈值 \(\theta_{\text{match}}\) 时，判定设备在该教室内。

4.3 课程时间表的概率推断

EDUM采用概率方法推断课程时间表。对于候选时间段 \([t_s, t_e]\)，定义课程存在的后验概率：

\[ P(\text{class} \mid D) = \frac{P(D \mid \text{class}) \times P(\text{class})}{P(D)} \]

其中似然函数 \(P(D \mid \text{class})\) 基于以下特征计算：

  1. 空间聚集度：该时间段内设备在目标教室的聚集程度
  2. 时间规律性：该模式每周重复出现的次数
  3. 人员稳定性：每次出现的人员构成的相似度
  4. 时长合理性：课程时长符合高校排课常规（45分钟、90分钟、135分钟等）

4.4 设备使用状态检测

EDUM通过WiFi流量模式推断设备是否处于活跃状态。对于时间窗口 \(w\)，设备活跃度得分：

\[ \text{Activity}(w) = \alpha \cdot \frac{\text{Packets}(w)}{\mathbb{E}[\text{Packets}]} + \beta \cdot I(\text{AssocChanges}(w) > 0) \]

其中：

  • \(\text{Packets}(w)\) 是窗口内的数据包数量
  • \(\mathbb{E}[\text{Packets}]\) 是该设备的历史平均发包率
  • \(I(\cdot)\) 是指示函数，检测AP关联变化（表示设备可能在移动）
  • \(\alpha, \beta\) 是权重系数，实验中分别设为0.7和0.3

4.5 课堂吸引力量化模型

EDUM提出课堂吸引力指数（Class Attractiveness Index, CAI）：

\[ \text{CAI}(c, w) = 1 - \frac{1}{N} \sum_{i=1}^{N} \frac{\text{ActiveTime}_i(c, w)}{\text{PresentTime}_i(c, w)} \]

其中：

  • \(c\) 表示课程，\(w\) 表示时间窗口
  • \(N\) 是在场学生总数
  • \(\text{ActiveTime}_i\) 是学生 \(i\) 设备活跃的时长
  • \(\text{PresentTime}_i\) 是学生 \(i\) 在教室的总时长

CAI取值范围为[0, 1]，值越高表示课堂吸引力越强，学生使用手机的时间比例越低。

6. 关键算法详解（含伪代码、复杂度分析）

6.1 AP邻近度矩阵构建算法

算法描述

该算法构建全校AP的邻近度矩阵，为后续位置推断提供基础。邻近度基于设备在两个AP间的共现频率计算。

伪代码

算法1: AP邻近度矩阵构建
输入: WiFi连接日志集合 L = {(mac, ap, timestamp)}
      时间窗口大小 T = 5分钟
输出: AP邻近度矩阵 P ∈ R^{M×M}

1: 初始化空字典 ap_occurrences = {}
2: 初始化空字典 pair_occurrences = {}
3: 初始化空字典 ap_timestamps = {}

4: // 第一步：按时间窗口和MAC聚合
5: 将日志 L 按时间窗口分组，窗口大小 T
6: for 每个时间窗口 w do:
7:     mac_ap_map = {}
8:     for 每条日志 (mac, ap, t) in w do:
9:         if mac not in mac_ap_map:
10:            mac_ap_map[mac] = set()
11:        mac_ap_map[mac].add(ap)
12:
13:    // 统计该窗口内的AP共现
14:    for mac in mac_ap_map:
15:        aps = mac_ap_map[mac]
16:        for ap in aps:
17:            ap_occurrences[ap] = ap_occurrences.get(ap, 0) + 1
18:        // 生成所有AP对
19:        ap_list = list(aps)
20:        for i in range(len(ap_list)):
21:            for j in range(i+1, len(ap_list)):
22:                pair = tuple(sorted([ap_list[i], ap_list[j]]))
23:                pair_occurrences[pair] = pair_occurrences.get(pair, 0) + 1

24: // 第二步：计算邻近度
25: M = 所有AP的数量
26: 初始化矩阵 P 为 M×M 的零矩阵
27: for 每个AP对 (ap_i, ap_j) in pair_occurrences:
28:     n_ij = pair_occurrences[(ap_i, ap_j)]
29:     n_i = ap_occurrences[ap_i]
30:     n_j = ap_occurrences[ap_j]
31:     P[ap_i][ap_j] = n_ij / sqrt(n_i * n_j)
32:     P[ap_j][ap_i] = P[ap_i][ap_j]

33: return P

复杂度分析

复杂度维度 分析
时间复杂度 \(O(N + K^2 \cdot W)\)，其中 \(N\) 是日志条数，\(K\) 是单窗口内平均AP数，\(W\) 是窗口总数。实际中 \(K \ll M\)，因此复杂度可接受。
空间复杂度 \(O(M^2)\) 存储邻近度矩阵。对于清华的2786个AP，矩阵大小约7.76百万个元素，内存占用约30MB（浮点数存储）。
优化策略 使用稀疏矩阵存储，因为大部分AP对的邻近度为0；采用分布式计算处理大规模日志。

6.2 基于密度峰值的课程发现算法

算法描述

该算法在时空维度上发现课程模式。课程具有周期性（每周重复）、空间聚集性（固定教室）、人员稳定性（固定学生群）三大特征。

伪代码

算法2: 课程时间表自动发现
输入: 学生-位置-时间张量 X ∈ R^{S×L×T}
      每周周期 W = 7天
      最小课程时长 min_len = 45分钟
      聚类阈值 θ_cluster
输出: 发现的课程列表 classes

1: // 第一步：构建每周重复模式
2: 按周对齐数据，构建周模式张量 X_week ∈ R^{S×L×(T/W)}
3: 计算周模式相似度矩阵 Sim ∈ R^{(T/W)×(T/W)}

4: // 第二步：时空密度峰值聚类
5: for 每个位置 l ∈ L do:
6:     提取该位置的时间序列 X_l = X[:, l, :]
7:     计算每个时间点 t 的局部密度:
8:         ρ_t = Σ_{t'} exp(-||X_l[:,t] - X_l[:,t']||² / 2σ²)
9:     计算每个时间点 t 的距离 δ_t:
10:        δ_t = min_{t': ρ_t' > ρ_t} |t - t'|
11:
12:    // 识别密度峰值（候选课程中心）
13:    peaks = [t for t where ρ_t > θ_ρ and δ_t > θ_δ]
14:
15:    // 第三步：课程时间范围确定
16:    for 每个峰值 t_p in peaks:
17:        向前扩展 t_start = t_p
18:        while t_start > 0 and Sim[t_start-1, t_p] > θ_sim:
19:            t_start -= 1
20:        向后扩展 t_end = t_p
21:        while t_end < T/W-1 and Sim[t_end+1, t_p] > θ_sim:
22:            t_end += 1
23:
24:        // 验证课程时长
25:        duration = t_end - t_start + 1
26:        if duration ≥ min_len:
27:            class_info = {
28:                'location': l,
29:                'time_slot': (t_start, t_end),
30:                'students': 识别稳定学生群,
31:                'confidence': 计算置信度得分
32:            }
33:            classes.append(class_info)

34: // 第四步：去重与合并
35: classes = 合并重叠或高度相似的课程
36: classes = 过滤置信度过低的候选

37: return classes

复杂度分析

时间复杂度为 \(O(L \cdot T^2)\)，其中 \(L\) 是位置数，\(T\) 是时间点数。通过以下优化可大幅降低：

  • 只对有足够人员聚集的位置进行计算
  • 相似度计算采用近似最近邻算法
  • 密度峰值计算使用KD树加速

6.3 课堂出勤状态检测算法

伪代码

算法3: 学生出勤状态检测
输入: 课程信息 class = {location, start_time, end_time, students}
      学生位置观测位置序列 loc_seq = [(t, location)]
      迟到阈值 θ_late = 10分钟
      早退阈值 θ_early = 10分钟
      在场比例阈值 θ_present = 0.7
输出: 出勤状态 status ∈ {在场, 缺席, 迟到, 早退}

1: // 提取课程时间段内的观测
2: class_obs = [(t, loc) for (t, loc) in loc_seq
               if class.start_time ≤ t ≤ class.end_time]

3: if len(class_obs) == 0:
4:     return "缺席"

5: // 计算在正确教室的时间比例
6: correct_loc_time = 0
7: total_time = 0
8: first_arrival = ∞
9: last_present = -∞

10: for i in range(len(class_obs)):
11:     t, loc = class_obs[i]
12:     if loc == class.location:
13:         correct_loc_time += 时间间隔
14:         first_arrival = min(first_arrival, t)
15:         last_present = max(last_present, t)
16:     total_time += 时间间隔

17: present_ratio = correct_loc_time / total_time

18: // 判定出勤状态
19: if present_ratio < θ_present:
20:     return "缺席"
21: else:
22:     status = "在场"
23:     // 检测迟到
24:     if first_arrival > class.start_time + θ_late:
25:         status = "迟到"
26:     // 检测早退
27:     if last_present < class.end_time - θ_early:
28:         if status == "迟到":
29:             status = "迟到并早退"
30:         else:
31:             status = "早退"
32:     return status

7. 实现细节

7.1 数据采集方案

WiFi网络数据采集

  • 采集协议：SNMP v2c，通过AP的管理接口获取连接设备信息
  • 采集频率：每分钟轮询一次所有AP
  • 采集字段：设备MAC地址、关联AP ID、RSSI信号强度、连接时长、数据包统计
  • 数据量：每日约500万条日志，原始数据大小约20GB/天

众包数据采集

EDUM开发了两款移动应用进行众包数据采集：

应用名称 功能 用户规模
TUNet 校园网认证客户端，自动上报WiFi连接状态与设备信息 数万名活跃用户
TUNow 校园生活助手，用户主动上传课程表与位置信息 数千名活跃用户

7.2 隐私保护机制

隐私保护是EDUM系统设计的核心考量：

  1. 数据匿名化：MAC地址通过SHA-256加盐哈希处理，无法反向识别具体个人
  2. 知情同意：所有众包用户签署知情同意书，明确数据使用范围
  3. 聚合分析：所有分析结果在群体层面输出，不展示个体行为细节
  4. 访问控制：原始数据仅授权研究人员访问，严格的审计机制
  5. 数据脱敏：对外发布的数据集移除所有可识别个人的信息

发现4：课堂专注度的时间动态

以45分钟课程为例，设备活跃度（表示分心）的时间分布呈现以下规律：

  2. 5-25分钟：活跃度保持低位（专注听课阶段）
  3. 25-35分钟：活跃度开始上升（疲劳期开始）
  4. 35-45分钟：活跃度显著上升（等待下课）

中间休息后的第二个45分钟，整体活跃度比第一个高约15-20%，表明学生注意力随时间衰减。

12. 局限性与未来工作

12.1 当前系统的局限性

  1. 设备依赖：仅能检测携带WiFi设备的学生，若学生不带手机或关闭WiFi则无法检测。实验中约有3-5%的学生属于此类。
  2. 位置精度天花板：纯WiFi定位难以达到亚米级精度，无法区分教室内的具体座位。
  3. 专注度间接测量：通过设备使用推断专注度存在误差：学生可能用手机查学习资料（活跃但不分心），也可能发呆（不活跃但分心）。
  4. 冷启动问题：新学年、新学期需要重新积累数据，初期精度较低。
  5. 隐私边界：虽然采取了匿名化措施，但大规模行为数据的收集仍存在伦理争议。

12.2 未来工作方向

技术演进方向

  • 多源数据融合：结合蓝牙、声学信号、教室摄像头（隐私保护前提下）等多模态数据提升精度
  • 深度学习建模：利用Transformer等序列模型对学生行为轨迹进行更精细的建模
  • 实时分析能力：当前系统以离线分析为主，未来可实现课堂实时反馈与干预
  • 跨校迁移学习：利用已有的学校数据加速新学校的系统部署

应用拓展方向

  • 个性化学习支持：基于出勤和专注度模式，为学生提供个性化学习建议
  • 教学质量评估：将EDUM指标纳入教师教学评价体系（需谨慎设计）
  • 课程优化建议：基于课堂吸引力的时间动态，为教师提供教学节奏改进建议
  • 教育政策研究：为排课、选课、课程设置等政策提供大规模数据支持

伦理与隐私方向

  • 差分隐私：在算法层面引入差分隐私机制，确保个体行为不可识别
  • 联邦学习：数据不出校，模型跨校联合训练，进一步降低隐私风险
  • 学生知情权：让学生了解哪些数据被收集、如何使用，建立信任机制

13. 总结与启示

13.1 论文核心贡献总结

EDUM论文在以下三个方面做出了重要贡献：

  1. 系统贡献：首次设计并实现了基于现有WiFi基础设施的大规模教育行为测量系统，验证了非侵入式教育测量的技术可行性。

  2. 方法贡献：提出了众包辅助的无预训练室内定位方法、课程时空模式自动发现算法、课堂吸引力量化指标，形成了完整的方法学体系。

  3. 科学贡献：通过大规模真实世界实验，定量揭示了多项课堂行为规律，为教育学研究提供了新的实证依据和研究范式。

13.2 对智能教育的启示

教育测量范式的转变

EDUM展示了"无感知测量"范式的巨大潜力：从传统的人工、主观、小样本测量，转向自动化、客观、大规模测量。这种范式转变将：

  • 降低教育研究的成本，使大规模纵向研究成为可能
  • 减少测量偏差，提高研究结论的可靠性和可复现性
  • 支持精细化的教学反馈，实现真正的因材施教

校园数据资产的价值

校园WiFi、一卡通、选课系统等基础设施产生的数据是重要的教育数据资产。EDUM证明：

  • 现有校园数据蕴含丰富的教育语义，需要专业的挖掘方法
  • 跨系统数据融合能够产生远大于单系统的价值
  • 高校应建立专门的教育数据治理和研究机制

技术与人文的平衡

教育是高度人文化的领域，技术的应用需要谨慎：

  • 技术应作为辅助工具，不应替代教师的专业判断
  • 学生不是被"监控"的对象，而是技术服务的主体
  • 任何技术应用都应以促进学生成长为根本目标
