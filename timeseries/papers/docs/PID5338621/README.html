二、背景（Background）

互联网公司监控数千到数百万 KPI（CPU 利用率、QPS 等）以保证服务可靠。KPI 上的异常常预示潜在故障（服务器宕机、网络过载、外部攻击）。异常检测被广泛用于最小化损失。

多数异常检测算法（Opprentice、EGADS、DONUT）假设每个 KPI 需要一个独立模型，但在大规模场景下：模型选择、参数调优、训练、标注成本巨大；数千到数百万 KPI 无法逐一建模。

幸运的是，许多 KPI 因隐含关联性彼此相似。若能按相似度分组为少数聚类，每个聚类只需一个异常检测模型，便能大幅降低开销。

KPI 聚类面临两个新挑战：(1) 噪声、异常、相位偏移、振幅差异扭曲相似度；(2) 高维度——KPI 通常上万数据点。传统时间序列聚类方法（<1000 点、平滑曲线）难以应对。

三、目的（Problems Solved）

  • 大规模 KPI 异常检测训练开销高：通过聚类把 M 个模型减少到 K 个。
  • KPI 形状变化鲁棒性：对噪声、异常、相位偏移、振幅差异不敏感。
  • 高维时间序列聚类效率：处理上万点 KPI。
  • 无需预设聚类数：DBSCAN 自动确定密度半径。
  • 新 KPI 自动分配：用质心做 1-NN 分配。
  • 真实场景验证：在大型互联网公司 KPI 上验证。

四、核心原理（Principles）

系统总览：ROCKA 4 步：(1) 预处理：缺失值插填、标准化；(2) 基线提取：移除极值、移动平均；(3) 聚类：DBSCAN + SBD + 启发式 ε；(4) 分配：新 KPI 同样流程 + SBD 到质心。

关键概念：

  • KPI：关键绩效指标。
  • Baseline / Residual：基线 / 残差。
  • z-score：标准化。
  • SBD / NCC / CC_s：形状距离 / 归一化互相关 / 位移互相关。
  • DBSCAN / ε / minPts：密度聚类 / 半径 / 最小邻居数。
  • kdis：k 距离。
  • Cluster Centroid：簇内 SBD 平方和最小的对象。

数学原理：

  • 基线提取：

\[\]

x_t^* = (1/W) \sum_{i=1}^{W} x_{t-i+1}


\[\]
  • 互相关：

\[\]

CC_s(\vec{x}, \vec{y}) = \begin{cases} \sum_{i=1}^{m-s} x_{i+s} \cdot y_i, & s \ge 0 \\ \sum_{i=1}^{m+s} x_i \cdot y_{i-s}, & s < 0 \end{cases}


\[\]
  • 归一化互相关与 SBD：

\[\]

NCC(\vec{x}, \vec{y}) = \max_s CC_s / (\|\vec{x}\|_2 \cdot \|\vec{y}\|_2), \quad SBD = 1 - NCC


\[\]
  • 质心：

\[\]

\text{centroid} = \arg\min_{\vec{x} \in C} \sum_{\vec{y} \in C} SBD(\vec{x}, \vec{y})^2


\[\]

五、算法详解（Algorithm）

  1. 输入 / 输出：输入：M 条原始 KPI 时序；输出：K 个聚类与质心，新 KPI 分配到最近簇。
  2. 核心模块：预处理、极值移除、基线提取、SBD 计算、密度估计 ε、DBSCAN、质心计算、分配。

def preprocess(kpi):
    kpi = linear_interpolate_missing(kpi)
    kpi = z_score_standardize(kpi)
    return kpi

def extract_baseline(kpi, window=24):
    threshold = np.percentile(np.abs(kpi), 95)
    cleaned = np.where(np.abs(kpi) > threshold, np.nan, kpi)
    cleaned = linear_interpolate(cleaned)
    baseline = moving_average(cleaned, window)
    return z_score(baseline)

def sbd(x, y):
    ncc = max(np.correlate(x, y, mode='full')) / (np.linalg.norm(x) * np.linalg.norm(y))
    return 1 - ncc

def estimate_eps(kdis, max_radius=0.05, minPts=4):
    candidates = find_flat_parts(kdis, max_radius, minPts)
    return max(candidates) if candidates else 0.05

def cluster_kpis(baselines, minPts=4):
    dist_matrix = pairwise_sbd(baselines)
    eps = estimate_eps(kdis_from(dist_matrix, k=minPts))
    clusters = dbscan(dist_matrix, eps=eps, minPts=minPts)
    centroids = [argmin_sum_sbd(c) for c in clusters]
    return clusters, centroids

关键数学：见 §四。

复杂度分析：SBD O(m log m)；距离矩阵 O(N² m log m)；DBSCAN O(N log N)；分配 O(K m log m)。

训练与推理：离线采样训练 + 在线 SBD 分配。

示例：9000+ KPI → 抽 2000 → 50 簇 → 每簇 DONUT；训练时间减少 90%。

八、关键创新点（Key Innovations）

  • + 极值移除 + 基线提取：消除异常和噪声对聚类的影响。
  • + SBD 形状距离：NCC 处理相位偏移，FFT 优化 O(m log m)。
  • + 启发式 ε 估计：自动选 DBSCAN 密度半径。
  • + 聚类 + 共享异常检测模型：训练开销从 M 降到 K。
  • + 真实工业 KPI 验证：F-score>0.85，训练时间减少 90%。

九、实验与结果（Experiments）

  • 数据集：公开 UCR + 真实工业 KPI（>9000 条）。
  • Baseline：YADING、K-Means、K-medoids、DBSCAN（多距离）。
  • 主要指标：F-score、Rand Index、训练时间、异常检测精度损失。
  • 关键结果数字：F-score>0.85；训练时间降 90%；性能损失 15%；采样 2000 代表 9000+。
  • 消融实验：分别去掉极值移除、基线提取、SBD、密度估计。
  • 效率分析：单次 SBD <1ms；聚类分钟级。
