二、背景（Background）

大型在线服务系统（Google、亚马逊、微软、银行）由上千分布式组件组成。监控系统收集 KPIs（日活、错误率）、logs（关键字匹配）、events 等。工程师写大量规则触发告警（CPU > 90% 触发 P2 错误、warning/error/fail 关键字触发不同严重度）。P1 critical 告警优先处理，但 P1 仍可能每天上百条。

真实案例：某银行 P2 告警（10:14 响应时间 500ms）因工程师忙于 P1 而被忽略，10:45 用户投诉才发现，浪费 31 分钟。规则分类 P/R 仅 0.43/0.68。痛点：(1) 标注成本高；(2) 告警类型多；(3) 系统动态变化；(4) 类别不平衡 50:1。

三、目的（Problems Solved）

  • 多源特征：从告警文本、时间、KPI 提取可解释特征。
  • 类别不平衡：排序学习（pointwise ranking）替代二分类。
  • 自动打标：历史工单 + 解决记录聚类。
  • 动态适应：增量训练（每日/每周）。
  • 可解释性：每维特征物理意义清晰。
  • 减少人工：检查量约减少 31%，保持高 P/R。

四、核心原理（Principles）

系统总览：两阶段——离线训练和在线排序。离线：(1) 提取 40 维特征；(2) 解决记录 TF-IDF + K-means（k=7，silhouette）打严重度分数；(3) XGBoost 回归训练；(4) 增量更新。在线：15 分钟窗口聚合告警，预测分数，按阈值告警。

关键概念：

  • BTM：Biterm Topic Model 短文本主题模型。
  • FT-tree：动态增量日志/告警模板解析树。
  • ACF：自相关函数。
  • IDF：逆文档频率。
  • XGBoost pointwise ranking：排序学习。
  • KPI Univariate/Multivariate Anomaly：单/多变量异常。

  1. 输入 / 输出：告警流 + KPI 时间序列 → 严重度分数 + 严重/非严重标签。
  2. 核心模块：预处理 → 文本/时间/KPI 特征 → 自动打标 → XGBoost 训练 → 在线检测。

def extract_alert_features(alerts, btm, idf, ftree):
    feats = []
    for a in alerts:
        topic = btm.transform(a.tokens)             # 14 dim
        entropy = sum(idf[w] / count(w) for w in a.tokens)
        freq = frequency(a.template)
        seasonal = max_autocorr(time_series[a.template], 15)
        count_w = count_in_window(a, window=30*60)
        inter_arrival = a.time - prev_alert_time(a)
        feats.append([*topic, entropy, freq, seasonal,
                      count_w['P1'], count_w['P2'], count_w['P3'],
                      count_w['all'], inter_arrival])
    return feats

def extract_kpi_features(kpi, lstm):
    w = lstm.window
    pred = lstm.predict(kpi[-w:])
    uni_err = np.abs(kpi[-1] - pred)        # 11 dim
    multi_err = np.linalg.norm(kpi[-1] - pred)
    return np.concatenate([uni_err, [multi_err, multi_err]])

def auto_label(alerts, k=7):
    tfidf_vecs = tfidf(alerts.resolution_record)
    clusters = kmeans(tfidf_vecs, k=k)
    severity_map = {0: 0.0, 1: 0.1, 2: 0.2, 3: 0.4, 4: 0.6, 5: 0.8, 6: 1.0}
    return [severity_map[c] for c in clusters]

def train_ranker(features, severity_scores):
    model = xgb.XGBRegressor(objective='reg:squarederror', n_estimators=300)
    model.fit(features, severity_scores)
    return model

def detect_online(new_alert, kpi_data, model, threshold):
    f = np.concatenate([
        extract_alert_features([new_alert]),
        extract_kpi_features(kpi_data, lstm)
    ])
    scores = model.predict(f)
    return (scores >= threshold).astype(int), scores

八、关键创新点（Key Innovations）

  • + 自动打标：解决记录聚类（k=7）替代手工标注。
  • + 40 维可解释特征：每维都有物理意义。
  • + 排序学习：缓解 50:1 不平衡。
  • + 增量训练：日更/周更，应对系统变化。
  • + 实际部署：中国建设银行上线。

九、实验与结果（Experiments）

  • 数据集：银行 3 数据集 A/B/C，6 个月一段，374940/429768/390437 条告警。
  • Baseline：Rule-based、Bug-KNN。
  • 指标：P/R/F1、precision@k、训练/检测时间。
  • 结果：AlertRank F1 0.89/0.86/0.93，平均 0.89；Rule 0.43/0.68；Bug-KNN 0.74/0.70/0.64。消融：去告警特征 F1 0.76，去 KPI F1 0.36。增量训练 0.68→0.88。检查量减少约 31%。
