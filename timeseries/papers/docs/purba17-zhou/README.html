二、背景（Background）

密集城区的管理与规划面临挑战：数据收集耗时、分析过时、依赖主观判断。校园尤其如此——学生的活动很少被大规模测量和分析。

论文系统化分析清华校园 WLAN 数据：~4.4 km² 校园，45000 学生 + 12000 教职员工；2016 年 8 月统计 116 栋楼 2890 个 Cisco 企业 AP，峰值同时连接 ~20000 设备，每日独立设备 >60000。

论文聚焦四个问题：(1) 居民如何在校内移动并花时间？(2) 人群活动有何频繁 / 周期模式？(3) 空间上下文（建筑物）如何影响活动？(4) 公共设施使用是否合理？

三、目的（Problems Solved）

  • 缺乏大规模校园移动数据：通过 WLAN 推断设备移动。
  • 室内 vs 室外移动差异：证明室内移动呈 Log-mixed-normal。
  • 建筑物使用模式识别：基于人流序列相似度聚类，得到 8 类建筑。
  • WiFi 热点诊断：识别 pause、entrance、leaky、roaming 四类热点。
  • 室内定位成本高：使用基于 AP "closeness" 的免标注算法。
  • 访客 vs 常住者区分：用 MAC 地址出现天数做粗筛。

四、核心原理（Principles）

系统总览：通过 SNMP trap + 三类 RSSI 收集设备移动轨迹；用 AP closeness 做室内定位（粒度到房间级）；基于人流序列做建筑聚类；基于移动模式识别 4 类 WiFi 热点。

关键概念：

  • Pause Time / Pause Location：设备停留时长与位置。
  • AP Closeness：两 AP 在同一 scan group 同时出现的频率。
  • Fingerprint Snapshot：滑动窗口内 AP RSSI 集合。
  • SNMP Trap / Object Values：AP 关联事件与 RSSI 数据源。
  • Levy Walk：重尾随机游走。
  • Log-mixed-normal：两个对数正态混合。
  • Hierarchical Clustering：层次聚类。
  • Cross Correlation：时序相似度。
  • Visitor / Occupant：访客 / 常驻。
  • Hotspot Type：pause / entrance / leaky / roaming 四类。

数学原理：

AP Closeness：

\[ C(\text{AP}_i, \text{AP}_j) = \frac{|\{sg \mid \text{AP}_i \in sg \land \text{AP}_j \in sg \land sg \in SG\}|}{|\{sg \mid \text{AP}_i \in sg \land sg \in SG\}|} \]

Cross Correlation：

\[ \rho_{XY}(\tau) = \frac{E[(X_t - \mu_X)(Y_{t+\tau} - \mu_Y)]}{\sigma_X \sigma_Y} \]

Log-mixed-normal 模型：

\[ P \sim \exp(\lambda_v \mathcal{N}(\mu_v, \sigma_v) + \lambda_o \mathcal{N}(\mu_o, \sigma_o)) \]

用 EM 拟合：cafeteria λ_v=0.918 / μ_v=3.042；dorm λ_v=0.802 / μ_v=3.590；classroom μ_o=3.829 对应课程 95-155 分钟。

暂停 vs 短暂移动：停留 ≥5 分钟 = pause；其他 = transient。

与现有技术的差异：相比传统 GPS 室外移动模型（Lévy walk 的飞行时长随室外区域缩小而缩短），论文证明室内不再符合；相比 LiveLabs（NUS）仅用关联事件，论文用三类 RSSI，覆盖率提升 71%。

五、算法详解（Algorithm）

  1. 输入 / 输出：
  • 输入：SNMP trap + RSSI 测量；可选 App 端移动数据。
  • 输出：每设备在每时刻的（停留区间 + 指纹）；建筑聚类；4 类热点清单。
  1. 核心模块：
  • Data Collection：过滤 LA-MAC 与 <5 天的 visitor。
  • AP Closeness 计算：用 scan group 统计 AP 之间共同出现频率。
  • Fingerprint Snapshot 生成：用滑动窗口（5 分钟）剔除过期与"远离"AP。
  • Snapshot Merge：相邻"相似"fingerprint 合并。
  • Pause vs Transient 分类：停留 ≥5 分钟 = pause。
  • Building Flow Series：统计每栋楼每 10 分钟的到达 / 离开 / 总设备数。
  • Hierarchical Clustering：用 cross correlation 衡量建筑间相似度。
  • Log-mixed-normal Fitting：对每类建筑做 EM 拟合。
  • Hotspot Classification：把 AP 分到 pause / entrance / leaky / roaming。
  1. 伪代码：

def compute_ap_closeness(scan_groups):
    closeness = defaultdict(lambda: defaultdict(int))
    for sg in scan_groups:
        for ap_i in sg:
            for ap_j in sg:
                if ap_i != ap_j:
                    closeness[ap_i][ap_j] += 1
    for ap_i in closeness:
        total = closeness[ap_i][ap_i]
        for ap_j in closeness[ap_i]:
            if total > 0:
                closeness[ap_i][ap_j] /= total
    return closeness

def mobility_detection(device, scan_groups, closeness, window=5*60):
    snapshots = []
    for sg in device.scan_groups:
        snap = [r for r in sg if r.rssi is not None and not far_away(r.ap, current_aps, closeness)]
        if snap:
            snapshots.append(snap)
    merged = merge_similar(snapshots, threshold=0.75)
    intervals = [(s.t_start, s.t_end) for s in merged if s.duration >= 5*60]
    return intervals

  1. 复杂度分析：
  • AP closeness：O(|scan_groups| × |AP|²)；
  • Mobility detection：单设备 O(n_snapshots)；
  • Building clustering：O(|buildings|² × |flow_len|)。
  1. 训练与推理：
  • 训练：无监督；EM 算法拟合 Log-mixed-normal。
  • 推理：在线 AP 选择 / 异常检测复用 closeness 度量。
  1. 示例：FIT（系楼）周三到达/离开曲线——早上涌入、中午短暂离开、下午课程再进入；发现某些 AP 总是 entrance 模式，某些总是 pause 模式，某些白天没人用（cold-spot）。

八、关键创新点（Key Innovations）

  • + 大规模真实数据：2800+ AP / 116 楼 / 60000+ 设备的清华校园。
  • + 三类 RSSI 融合：覆盖率提升 71%。
  • + 免标注 mobility detection：基于 AP closeness。
  • + Log-mixed-normal：刻画室内停顿时长双峰。
  • + 建筑聚类：cross correlation + hierarchical clustering 自动得到 8 类建筑。
  • + 4 类热点诊断：自动发现 AP 部署问题。

九、实验与结果（Experiments）

  • 数据集：清华校园 WLAN 11 周（2015-11 至 2016-01）；209,723 个 MAC 地址；107,290 个非访客设备。
  • 暂停统计：日均 2.64 次暂停，93% 室内时间在 pause，停留 196.35 分钟，跨 3.11 个位置。
  • 建筑分类：8 类——admin / cafeteria / classroom / department / dorm / gym / library / others。
  • Log-mixed-normal 参数：cafeteria λ_v=0.918 / μ_v=3.042；dorm λ_v=0.802 / μ_v=3.590；classroom μ_o=3.829 对应课程 95-155 分钟。
  • 热点：A 食堂 8 AP（4p/4e/7l/7r）；D 教学楼 115 AP（106p/3e/29l/26r）；F 系楼 121 AP（103p/2e/16l/24r）。
  • 诊断：A 食堂 AP 数量不足且功率过高；D 教学楼空旷结构导致 leaky AP；F 系楼 1 楼为 entrance，6 楼为 cold-spot。
  • 验证：对比 5 台设备手工日志，WLAN 数据准确率 84.5%、冲突率 0.73%；移动数据 92.0% / 1.31%。

十、应用场景（Use Cases）

  • 校园 AP 部署优化：诊断 pause / entrance / leaky / roaming 热点。
  • 大型公共场所 WiFi 规划：商场、机场、医院。
  • 教学楼课程安排：根据学生移动模式优化课程时段。
  • 节能 / 资源调度：识别 cold-spot，关闭冗余 AP。
  • 城市人群流动研究：扩展到城市级 AP 数据。
