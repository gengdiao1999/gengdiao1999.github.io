二、背景（Background）

Named Data Networking (NDN) 是新型互联网架构，关注"内容是什么"。消费者发送 Interest 包，匹配 Data 可从任何地方返回。NDN 带来网络内缓存、组播、数据中心化安全。

NDN 独特之处是自适应转发平面：路由器在 FIB 中记录每个 prefix 的多个出接口，观察 Interest/Data 性能后动态选最佳路径、检测/恢复故障、负载均衡、缓解 prefix hijacking 和 DDoS。

接口排名（interface ranking）决定何时、如何更新 metrics。颜色分类：GREEN / YELLOW / RED。NDN 有 Best-Route 和 NCC 两种策略（NCC 是 CCNx 0.7.2 默认）。

NDN 转发平面与 IP 不同：能实时感知拥塞状态。但 in-network cache 让 Interest 满足位置高度不确定，导致两个 NDN 特有（IP 没有）问题：

  1. SRTT 慢收敛：低频测量下 SRTT 跟不上 RTT 突变。
  2. Probing Oscillation：多路径 + 共享瓶颈链路 + 包级触发探测，造成 GREEN/YELLOW 高频切换。

三、目的（Problems Solved）

  • SRTT 慢收敛：用 ASU 动态调整采样间隔。
  • Probing Oscillation：Proactive Probing + Multi-GREEN-Interface + PPF。
  • EOR 降低：ASU 把 EOR 降 46.7%。
  • 丢包率降低：r=0.01/0.1 时降 75%-94.75%。
  • 理论分析闭环：推导收敛时间与丢包率公式。

四、核心原理（Principles）

系统总览：(1) ASU：用 Changing Factor η 衡量 RTT 变化剧烈度，η 大时减小采样间隔 Δn；(2) Proactive Probing：Multi-GREEN-Interface 保持 MNGI 个 GREEN 备份路径；PPF 用 1/COST 加权探测概率。

关键概念：

  • NDN / Interest / Data / NACK。
  • PIT / FIB。
  • SRTT / Color Classification / Interface Ranking。
  • Best-Route / NCC。
  • ASU / Proactive Probing / Multi-GREEN-Interface / MNGI / PPF。
  • η / η_threshold / EOR。
  • COST = OSPF + RTT。

数学原理：

  • SRTT 更新：

\[\]

SRTT_i = α · SRTT_{i-1} + (1 - α) · RTT_i


\[\]
  • Changing Factor η：

\[\]

η = (RTT - SRTT) / (RTT + SRTT), 0 ≤ η < 1


\[\]
  • ASU 采样间隔：

\[\]

Δn_i = max(Δn_min, (1-η_i) Δn_{i-1}) if η ≥ η_threshold; else min(Δn_max, (1+β) Δn_{i-1})


\[\]
  • PPF：

\[\]

PPF(i) = (1/COST_i) / Σ_j (1/COST_j)


\[\]
  • 退出 oscillation 期望：

\[\]

E(n_q) = 1/p, p ≈ [(1-PPF(2)) + (1-PPF(3))]/2


\[\]

五、算法详解（Algorithm）

  1. 输入 / 输出：输入：每个 prefix 在每个出接口的 RTT 采样、当前 SRTT/Δn、COST；输出：更新后的 SRTT、调整后的 Δn、颜色与排名。
  2. 核心模块：SRTT 更新、Δn 调整、Multi-GREEN 检查、PPF 探测、颜色更新、COST 重计算。

def adaptive_srtt_update(srtt_prev, rtt_new, alpha=0.8):
    return alpha * srtt_prev + (1 - alpha) * rtt_new

def update_delta_n(delta_n_prev, rtt, srtt, delta_n_min=1, delta_n_max=100, eta_th=0.1, beta=0.1):
    eta = (rtt - srtt) / (rtt + srtt) if (rtt + srtt) > 0 else 0
    if eta >= eta_th:
        return max(delta_n_min, (1 - eta) * delta_n_prev)
    else:
        return min(delta_n_max, (1 + beta) * delta_n_prev)

def proactive_probe(interfaces, mngi=2):
    green = [i for i in interfaces if i.color == GREEN]
    yellow = [i for i in interfaces if i.color == YELLOW]
    if len(green) < mngi and yellow:
        weights = [1.0 / i.cost for i in yellow]
        total = sum(weights)
        probs = [w / total for w in weights]
        chosen = random.choices(yellow, probs)[0]
        if probe_success(chosen):
            chosen.color = GREEN

复杂度分析：SRTT O(1)；Δn O(1)；PPF 探测 O(K)；Multi-GREEN 检查 O(K)。

训练与推理：无离线训练，纯在线。

示例：R2 缓存命中时 RTT 从 150ms 降到 50ms；ASU 6 步收敛 SRTT；Multi-GREEN+MNGI=2 防 oscillation；PPF 选最优 YELLOW 升级；丢包率降 94.75%。

八、关键创新点（Key Innovations）

  • + ASU 动态采样间隔：SRTT 收敛到 6 步。
  • + Multi-GREEN-Interface：避免 oscillation。
  • + PPF 探测概率函数：$1/COST$ 加权。
  • + 理论分析：收敛步数、EOR、丢包率、E(nq) 全推导。
  • + 仿真验证：SRTT 收敛减 37.9%，丢包率降 75%-94.75%。

九、实验与结果（Experiments）

  • 平台：NDNSim 2.0 + ndn-cxx + NFD；清华校园网子拓扑。
  • 参数：tdata=100ms、tout=120ms、k=100、η_threshold=0.1。
  • Baseline：Best-Route、NCC、固定 Δn SRTT 测量。
  • 主要指标：SRTT 收敛时间、丢包率、EOR、收敛步数。
  • 关键结果数字：EOR 降 46.7%；SRTT 收敛减 37.9%；丢包率降 75%-94.75%；E(nq)=5。
  • 消融实验：分别关闭 ASU、Multi-GREEN、PPF。
  • 效率分析：算法 O(1) 每 Interest。
