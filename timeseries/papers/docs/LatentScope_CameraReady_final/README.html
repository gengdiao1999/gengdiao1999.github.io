1. 文档信息速览

LatentScope 是面向有限可观测性微服务系统的根因分析框架，发表于 KDD 2024。

核心创新：将根因候选（RCC）建模为隐变量，通过双空间图 + RLIR 算法实现快速推理。

2. 研究背景

微服务系统故障根因分析至关重要。eBay 经验表明，63.59% 的故障来自第三方服务，但第三方服务的可观测性有限。

现有 RCA 算法需要 RCC 有良好的可观测性，不适用于有限可观测场景。

3. 论文目的与解决的问题

  1. RCC可观测性有限（第三方服务无监控指标）
  2. 间接指标带来模糊信息
  3. 异构RCC需统一建模
  1. 隐变量建模RCC
  2. 双空间图统一表示
  3. RLIR快速推理+LatentRegressor增强

4. 核心原理（含数学公式）

双空间图建模：可观测空间（指标）和隐空间（RCC），多对多连接。

\( x_i^{(t)} = f_i(pa(x_i)^{(t)}, rcc(x_i)^{(t)}) \)

RLIR算法通过两步骤计算RCC得分：

\( L_{i,j} = \max_t \frac{L(\hat{x}_i, \hat{x}_j)^{(t)}}{\hat{x}_i^{(t)}} \)

\( M_{a,i} = \hat{x}_i \cdot (1 - \max_{j,\exists z_b \in rcc_j - rcc_i} L_{i,j}) \)

\( score_{rcc}(z_a) = \max_{i,z_a \in rcc_i} \max_t M_{a,i} \)

6. 关键算法详解

算法: LatentRegressor

输入: RCC关系图G_RCC, 指标关系图G_metric
输出: RCC根因得分

for b in V_r:
  x_m = argmax score(x_i) 其中b∈rcc_i且next(b)不在rcc_i
  b_scores[b][x_m] = score(x_m)

// 向上传播(Up Step)
for b in V_r:
  if b只有一个父节点a:
    a_scores[a] = b_scores[b]

// 向下消除(Down Step)
for b in V_r:
  消除与父节点不共享指标的RCC

return 最终RCC得分

数据存储层推荐使用列式时序数据库如 InfluxDB 或 TDengine，相比关系数据库可节省5-10倍存储空间。建议按指标+实体+时间三维分区，并设置合理的数据保留策略（如热数据7天、温数据30天、冷数据归档至对象存储）。

预处理层应作为独立微服务便于横向扩展。流水线包括：数据校验→缺失值填充→异常值裁剪→归一化→窗口化。每步应有指标埋点便于排查。使用 Apache Flink 或 Kafka Streams 实现流式处理。
