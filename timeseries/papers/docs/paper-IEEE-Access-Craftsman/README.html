二、背景（Background）

云服务提供商在数据中心部署大量网络设备，Microsoft 上万 [1]，每年 400+ 故障 [2]。故障检测/诊断/预测 [3-7] 受关注。syslog 是重要信息源 [4, 5, 8-12]，但需解析 [13-15]。当前解析方法学模板再匹配。

syslog 解析对故障管理至关重要，但 STE [9]、LogSimilarity [4] 模板学习准确率低、匹配慢。数据中心每天数千万条 syslog；低效匹配消耗资源大。

SignatureTree [8]、STE [9] 不支持增量学习——新增模板需重解析全部历史。系统通常基于模板库训练故障检测/诊断/预测模型，需定期重训，因此模板库必须支持增量重训。运维频繁软/固件升级 [22] 生成新子类型，需新模板。

论文提出 Craftsman：基于前缀树（prefix-tree）+ 频繁模式自动学习模板。模板定义为"syslog 中最长的频繁词组合"。

三、目的（Problems Solved）

  • 模板学习准确率低：基于频繁词组合，准确率 Rand index 接近 1。
  • 模板匹配效率低：前缀树 O(H·k) 匹配。
  • 不支持增量学习：插入新 syslog 只更新局部路径。
  • Syslog 解析通用性差：5 个公开数据集通用。
  • 故障预测精度低：F1 提升 13.07%-188%。

系统总览：Craftsman 3 部分：(1) 模板学习（前缀树构建）；(2) 增量学习；(3) 模板匹配。

五、算法详解（Algorithm）

  1. 输入 / 输出：输入：消息集 DM、阈值 k；输出：前缀树 T（每根叶路径 = 模板）。
  2. 核心模块：构建初始 L、建树、剪枝、增量学习、匹配。

def build_prefix_tree(DM, k):
    L = sorted_words_by_frequency(DM)
    T = root(label=message_type)
    for msg in DM:
        sorted_msg = sort_words(msg, key=L)
        insert_tree(sorted_msg, T)
    for C in T.children:
        if len(C.children) > k:
            C.children = []
    return T

def insert_tree([p | P], T):
    for child in T.children:
        if child.nword == p:
            if P:
                insert_tree(P, child)
            return
    new_node = Node(nword=p)
    T.children.append(new_node)
    if P:
        insert_tree(P, new_node)

def match_syslog(T, msg):
    sorted_msg = sort_words(msg, key=L)
    cur = T
    for word in sorted_msg:
        found = False
        for child in cur.children:
            if child.nword == word:
                cur = child
                found = True
                break
        if not found:
            return None
    return cur.template_id

关键数学：见 §四。

复杂度分析：学习 O(|DM|·H·k)；匹配 O(H·k)；增量 O(H·k)；空间 O(节点数)。

训练与推理：离线建树 + 在线匹配 + 增量更新。

示例：13 条 OSPF_NBR_CHG → L 排序 → 4 模板（Init/Exstart/Exchange/Loading）；新 syslog from ExStart to Down → insert "ExStart"→"Down"。

八、关键创新点（Key Innovations）

  • + 前缀树 + 频繁模式：自然支持增量学习。
  • + Rand index 接近 1。
  • + 匹配 O(H·k)：比 STE/LogSimilarity 快 6.88-10.25x。
  • + 解析总提速 730-6847x。
  • + 故障预测 F1 提升 13.07%-188%。
  • + 5 个公开数据集通用：96.25%。

九、实验与结果（Experiments）

  • 数据集：tier-1 云 2223 设备 2 年；5 公开（Blue Gene/L、HPC、HDFS、Zookeeper、Proxifier）。
  • Baseline：STE、LogSimilarity、SignatureTree、IPLoM、LKE、LogSig。
  • 主要指标：Rand index、匹配时间、解析时间、内存、F1。
  • 关键结果数字：Rand index≈1；匹配 6.88x/10.25x；解析 730x/6847x；F1 提升 13.07%-188%；5 数据集 96.25%；解析 4.98s/天；内存 32.4MB。
  • 消融实验：去掉增量、去掉剪枝。
  • 效率分析：O(H·k) 匹配；O(节点数) 空间。
