二、背景（Background）

异常检测在 IT 基础设施管理中始终重要。近年趋势是引入深度学习（DL）算法（OmniAnomaly、LSTM-VAE、DAGMM）替代规则算法。DL 算法具有自动化、鲁棒性、可操作性优势。

但 DL 算法面对"维度爆炸"挑战：

  1. 机器域：现代数据中心数百万台服务器。
  2. KPI 域：每机器 10X KPI（CPU/内存/网络），每 KPI 单变量时序；每机器 = MTS。
  3. 时间域：30 秒间隔，每天 2880 时间点。

典型 DL 算法为每台机器训练一个模型，用于百万机器不可扩展；单一模型又难以表达多样机器。

一些方案先聚类再每簇异常检测，但高维 KPI + 时间维使聚类低效（百万机器的 MTS pairwise 距离无法实用）。

观察到一类无监督 DL 算法（RNN-VAE）会把原始 MTS 压缩为低维潜在表示，论文据此用低维表示做聚类以解维度爆炸问题。

三、目的（Problems Solved）

  • 机器 × KPI × 时间三维爆炸：用 RNN-VAE 潜在表示压缩到 C 维。
  • 每机一模型太慢：聚类到 K 簇，每簇一模型。
  • 一模型太粗：fine-tuning 让每簇模型更准确。
  • RNN-VAE 训练与聚类循环依赖：粗粒度预训练再聚类再 fine-tune。
  • 时域高维：latent 分布代替时序做距离。
  • 设计选择合理化：聚类算法、距离度量、迁移方法的选择。

四、核心原理（Principles）

系统总览：CTF 4 步：(1) 预训练粗粒度模型 M₀；(2) 特征提取得 z_t 分布 P_i；(3) HAC 聚类到 K 簇；(4) 冻结 RNN 调 Dense 做 fine-tuning。

关键概念：

  • Machine Entity、KPI、MTS。
  • RNN-VAE、Encoder / Decoder、Latent Representation zt。
  • Reconstruction Probability、ELBO。
  • Wasserstein Distance、HAC、KL / JS Divergence。
  • Fine-tuning、RNN Layer Frozen / Dense Layer Tuned。
  • POT、Coarse-grained / Fine-grained Model。

数学原理：

  • 归一化：

\[\]

\hat{x}_{i,j,t} = (x_{i,j,t} - \mu_{i,j}) / \sigma_{i,j}


\[\]
  • ELBO 损失：

\[\]

L(x_t) = E_{q_φ(z_t|x_t)} [\log p_θ(x_t|z_t)] - D_{KL}[q_φ(z_t|x_t) \| p_θ(z_t)]


\[\]
  • Wasserstein 距离：

\[\]

W(P_1, P_2) = \inf_{\gamma \in \Gamma(P_1, P_2)} E_{(z_1, z_2) \sim \gamma} [\|z_1 - z_2\|]


\[\]

  1. 输入 / 输出：输入：M 台机器的 MTS；输出：K 个 fine-grained 模型与异常分数。

def cluster_machines(all_Pi, K=50, n_sample=10000):
    sample_idx = random.sample(range(M), n_sample)
    dist_matrix = pairwise_wasserstein([all_Pi[i] for i in sample_idx])
    clusters = HAC(dist_matrix, n_clusters=K)
    for i in range(M):
        if i not in sample_idx:
            dists = [mean_wasserstein(all_Pi[i], cluster_centers[c]) for c in range(K)]
            clusters[i] = argmin(dists)
    return clusters

def fine_tune_cluster(cluster_data, coarse_model):
    new_model = copy.deepcopy(coarse_model)
    for param in new_model.rnn.parameters():
        param.requires_grad = False  # 冻结 RNN
    for epoch in range(N):
        loss = elbo(new_model, cluster_data)
        loss.backward()  # 只更新 dense
    return new_model

复杂度分析：粗粒度预训练 O(M_sample·T·(L+C)²)；特征提取 O(M·T·(L+C))；HAC O(M² log M)；fine-tune O(K·T·(L+C)²)；在线 O((L+C)²)。

训练与推理：训练 4 步 / 4.40h；推理毫秒级 + POT 阈值。

示例：10^5 机器 49 KPI 13 天 → CTF → 50 簇 → F1=0.830（vs per-machine 0.842）。

八、关键创新点（Key Innovations）

  • + 粗-精模型迁移：打破 RNN-VAE 训练与聚类循环依赖。
  • + 潜在表示分布做聚类：避免原始 MTS pairwise。
  • + Wasserstein 距离：处理分布间距离，对非重叠仍稳定。
  • + 冻结 RNN + 调 Dense。
  • + 训练时间从 2 个月降到 4.40h。
  • + F1=0.830 性能仅损 0.012。
  • + 公开标注工具与数据集。

九、实验与结果（Experiments）

  • 数据集：字节跳动生产 10^5 机器 × 49 KPI × 13 天（37440 时间点）；5 类 KPI。
  • Baseline：per-machine 模型（F1=0.842）、单模型、naive 聚类。
  • 主要指标：训练时间、F1、Precision、Recall。
  • 关键结果数字：CTF 训练 4.40h；F1=0.830（损 0.012）；K=50；C=3；T0=100；Tm=315s；Tf=0.3s。
  • 消融实验：分别去掉聚类、fine-tuning、Wasserstein，换 JS/KL。
  • 效率分析：相比 per-machine 提速约 300 倍。
