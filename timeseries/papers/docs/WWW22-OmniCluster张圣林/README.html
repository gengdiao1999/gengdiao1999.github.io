二、背景（Background）

云原生与微服务架构让 Web 服务可以快速弹性扩展，但代价是"实例爆炸"：单家厂商可能拥有百万级别的 service instance / container / VM / 物理机 / 交换机 / 路由器。每个实例都被实时打上数十到上百个指标（CPU、内存、磁盘 I/O、网络吞吐、平均响应时间、错误率、PV 等），形成多元时间序列（Multivariate Time Series, MTS）。一旦某个实例发生异常，告警会立刻对全网服务可用性造成影响。

当前主流的深度 MTS 异常检测模型（USAD、OmniAnomaly、SDFVAE、InterFusion、DAGMM 等）通常要为"每个实例"训练一个专属模型，论文 Table 1 给出在高配服务器上的单实例训练耗时：USAD 8.63s、OmniAnomaly 2.99min、SDFVAE 4.08min、InterFusion 8.09min、DAGMM 17.81s。换算到 100 万实例，USAD ≈ 99.87 天，OmniAnomaly ≈ 5.70 年，InterFusion ≈ 15.40 年。训练成本"完全不可能承受"。

但如果"全网一个模型"，又会因为不同实例的指标形态差异大而降低精度。自然的折中：先把实例聚成若干簇，再"每簇一个模型"。OmniCluster 的目标就是给出这种"快速 + 鲁棒 + 任务无关"的聚类方法。

三、目的（Purpose / Problems Solved）

  • 痛点：实例数百万 + 时点数万 → 传统聚类的 pairwise 距离计算不可承受 → 方案：用 1D-CAE 把每条 MTS 在 temporal 维度做降维，同时自动规避噪声与异常点干扰。
  • 痛点：每个实例的指标里既存在冗余又存在非周期噪声，污染聚类结果 → 方案：三步特征选择——YIN 去除非周期特征 + 冗余矩阵 + 4 条 RULE 1~4 解决冗余/全相关等退化情形。
  • 痛点：实际生产中"哪些实例属于一个聚类"是动态的，需要在线分配 → 方案：离线 HAC 聚类得到簇心与 encoder 参数；在线只对新增实例做"特征子集 + 距离簇心"的快速分类，同时通过距离阈值检测离群实例。
  • 痛点：聚类本身是 unsupervised，缺少评价手段与人工标注工具 → 方案：开源 GUI 标注工具，把聚类结果直接渲染给运维工程师打标。
  • 痛点：MTS 含噪声、异常点，传统聚类方法表现差 → 方案：M 通道独立 1D-CAE（避免 2D-CAE 跨通道信息丢失）+ 滑窗平均平滑 + 异常剔除 + 标准化。

四、核心原理（Principles）

  1. 系统总览：图 2 把 OmniCluster 拆成两段：
    • 离线聚类：MTS → 预处理（极值剔除、缺失插值、移动平均平滑、归一化）→ 1D-CAE 降维到 z → 三步特征选择得到 z'' → HAC 聚类得到簇心 C。
    • 在线分类：新实例 → 同样的预处理 → 复用 encoder + 特征子集索引 → 与各簇心计算欧氏距离 → 距离小于阈值 τd 归入该簇；否则视为离群。
  2. 关键概念：
    • MTS \( x^s \in \mathbb{R}^{M \times T} \)，M 是指标数，T 是时点数。
    • 1D-CAE：M 条独立 1D-CNN 编码器 + 对应的反卷积解码器，重建损失 = MSE。
    • YIN 周期度：用于判断单条特征是否周期。
    • 冗余矩阵 \( R \in [0,1]^{M' \times M'} \)：基于归一化互相关 NCC 构造。
    • HAC + 平均 linkage：聚类簇数由阈值 τd 控制（无需预设 K）。
    • 特征子集索引 \( S_F \)：保存下来用于在线推理。
  3. 数学原理：
    • 预处理（归一化到 [0,1]）：\[ x'^{sm} = \frac{x^{sm}_t - \min_{t \in T} x^{sm}_t}{\max_{t \in T} x^{sm}_t - \min_{t \in T} x^{sm}_t} \]
    • 1D-CAE 重建损失（MSE）：\[ \mathcal{L} = \frac{1}{M} \sum_{m=1}^{M} \text{MSE}(x^{sm}, \hat x^{sm}) \]
    • 冗余矩阵：\[ R_{ij} = \mathbb{1}\big[ \text{NCC}(z'^s_i, z'^s_j) > \theta_s \big] \]
    • RULE 3(a) 选最不相关特征：\[ i = \arg\min_{i' \in F} \sum_{j \in S_F} R_{i'j} \]
    • 簇间距离（average linkage）：\[ D_{\text{clust}}(A, B) = \frac{1}{|A| \cdot |B|} \sum_{a \in A} \sum_{b \in B} D_{\text{MTS}}(a, b) \]，其中 \( D_{\text{MTS}}(a, b) = \sum_{m=1}^{M''} \|a_m - b_m\|_2 \)，M'' 是特征选择后保留下来的指标数。
  4. 与现有技术的差异：
    • 相对 TICC/Mc2PCA/FCFW/SPCA+AED：复杂度从 O(N²) 降至 11.69 分钟级（Table 2）；同时支持"先聚类再训练检测模型"的工程管线。
    • 相对 2D-CAE：保留各指标的物理含义独立编码，避免跨通道信息丢失。
    • 相对传统 HAC：MTS 先经 1D-CAE 降噪 + 三步特征选择，因此聚类更鲁棒，且能从数据本身学到"应该保留多少指标"。

五、算法详解（Algorithm）

1. 输入 / 输出

  • 输入：M 个实例的 MTS \( X = \{x^1, ..., x^M\} \)，每条 \( x^s \in \mathbb{R}^{M \times T} \)。
  • 输出：每个实例的簇标签 \( \hat y^s \in \{1, ..., K\} \cup \{outlier\} \)，簇心集合 \( C = \{c^1, ..., c^K\} \)，保存的 encoder 与特征索引 \( S_F \)。

2. 核心模块

  • 预处理模块
  • 1D-CAE（3 层 conv + 3 层 deconv，kernel=7，stride=2，通道数 (16,32,1)）
  • 特征选择三步法
  • HAC 聚类 + 离群判定

3. 伪代码

# 离线聚类
def OmniCluster_offline(X, M, T):
    # step 1: preprocess
    X = remove_extremes(X, top=5%)
    X = interpolate_missing(X)
    X = moving_average(X, w=12)
    X = normalize(X)  # to [0,1]

    # step 2: 1D-CAE reduction
    encoder, decoder = train_1dcae(X, kernel=7, stride=2,
                                   channels=(16, 32, 1),
                                   loss='mse')
    z = encoder(X)  # shape M x T' x M

    # step 3: feature selection
    # 3.1: non-periodic removal via YIN
    z_prime = remove_non_periodic(z, theta_y=0.5)

    # 3.2: redundancy matrix
    R = build_redundancy_matrix(z_prime, theta_s=0.93, ncc)

    # 3.3: apply 4 rules iteratively
    F, SF, DF = apply_rules(R)  # SF = selected indices
    z_double_prime = z_prime[:, :, SF]

    # step 4: HAC
    clusters, centroids = HAC(z_double_prime, linkage='avg', threshold=tau_d)

    save(encoder, SF, centroids, tau_d)
    return clusters, centroids

# 在线分类
def OmniCluster_online(new_x):
    new_x = preprocess(new_x)  # 同上
    z = encoder(new_x)[:, :, SF]
    best_cluster, best_dist = None, INF
    for k, c in enumerate(centroids):
        d = euclidean(z, c)
        if d < best_dist:
            best_dist = d
            best_cluster = k
    if best_dist < tau_d:
        return best_cluster  # assign
    else:
        return 'outlier'  # report to operators

4. 关键数学

上面已给出 5 个核心公式（归一化、重建损失、冗余矩阵、最小相关特征选取、平均 linkage）。

5. 复杂度分析

  • 1D-CAE 训练：\( O(M \cdot T' \cdot K) \) per epoch，K 为卷积核数。
  • 冗余矩阵：\( O(M'^2 \cdot S \cdot T') \)（S 是样本数）。
  • HAC 距离矩阵：\( O(N^2 \cdot M'' \cdot T'') \)。
  • 论文用 3175 个实例、19 个指标、2016 时点/周的数据，全流程 11.69 分钟，比 TICC 的 104.17 小时快约 500 倍。

6. 训练与推理

  • 1D-CAE：3 卷积 + 3 反卷积，kernel=7，stride=2，channel=(16,32,1)，batch=128，Adam 优化器。
  • HAC：average linkage，τd 自动取 7。
  • 在线：仅前向推理，距离计算为 \( O(K \cdot M'' \cdot T'') \)，单实例毫秒级。

7. 示例

论文 § 4.4：对 ByteDance 3175 个实例、19 维指标聚类，得到 19 个簇（ground truth 也是 19），NMI=0.9160、ACC=0.7990、F1=0.9057（outlier detection），聚类总耗时 11.69 分钟。

八、关键创新点（Key Innovations）

  • + 把 1D-CAE 引入 MTS 聚类：每个指标用独立 1D-CNN 通道编码，保留物理含义同时压缩时序维度，对噪声/异常鲁棒（论文 § 3.3）。
  • + 三步特征选择策略：YIN 周期度 + 归一化互相关冗余矩阵 + 4 条 RULE，把"哪些指标应该被聚类"这件事变成数据驱动的可解释步骤。
  • + 离线聚类 + 在线分类 + 离群检测：用 τd 阈值即可同时处理"分类"与"发现新类型实例"，直接对接运维侧告警。
  • + 任务无关（task-agnostic）：与 5 种 MTS 异常检测模型正交结合（OmniAnomaly、USAD、SDFVAE、InterFusion、DAGMM），训练时间平均下降 95.01%，F1 不掉。
  • + 标注工具 + 数据集 + 代码全部开源：github.com/omni-cluster 三个仓库，方便后续工作复现与扩展。

九、实验与结果（Experiments）

  • 数据集：ByteDance 真实 MTS（3175 个系统实例，19 个指标，7 天 × 5 分钟间隔 = 2016 时点），含 29 个类 + 28 个 outlier 实例；运行在 2× Intel Xeon Gold 5218（16C32T @ 2.30GHz）+ NVIDIA Tesla V100S + 192GB RAM。
  • Baseline：TICC、Mc2PCA、FCFW、SPCA+AED（均需预指定 K）。
  • 主要指标：NMI、ACC、outlier 检测的 F1、聚类时间、异常检测模型的训练时间。
  • 关键结果数字：
    • 聚类：NMI=0.9160、ACC=0.7990、Outlier F1=0.9057、#C=19（与 ground truth 完全一致）、聚类时间 11.69 min。最强 baseline NMI=0.6236、ACC=0.4117，且无法识别 outlier。
    • 异常检测训练开销：5 种模型平均降低 95.01%（E1 vs E4，从"每实例一模型"压缩为"每簇一模型"），F1* 几乎不下降。
    • 与 CTF 框架对比：CTF（专为 OmniAnomaly 设计的 coarse-to-fine 框架）耗时 5601.48s、F1* = 0.8323；OmniCluster + OmniAnomaly 耗时约一半、F1* 更高。
    • 消融（C1~C4）：换 2D-CAE NMI 降到 0.8511；去 CAE 耗时 12 倍但 ACC 略升；去非周期去除 NMI=0.7602、#C=117；去冗余消除 NMI=0.8724。说明 1D-CAE + 周期去除 + 冗余消除三者缺一不可。
  • 效率分析：用 OmniCluster 后，训练 19 个模型 vs 训练 3175 个模型，时间从 O(N·T) 降到 O(K·T)，N/K ≈ 167 倍压缩，与实测 95.01% 的下降吻合。

十、应用场景（Use Cases）

  • 微服务 / 容器化平台：百万级实例的异常检测前置聚类，节约训练资源。
  • AIOps 平台：在新实例接入时，自动分到现有簇，调度对应的异常检测模型。
  • 数据中心容量规划：聚类结果可用于识别"行为相似的服务器组"，支撑调度与灰度。
  • 业务侧异常归因：把异常范围缩小到具体的簇，便于按 SLA 评估。
  • 标签辅助：用开源 GUI 工具标注后，可以做半监督 / 监督的进一步分析。

十二、术语表（Glossary）

  • MTS (Multivariate Time Series)：多维时间序列。
  • 1D-CAE：一维卷积自编码器。
  • YIN：基频估计器，被论文借用来判断时序是否具有周期。
  • NCC (Normalized Cross-Correlation)：归一化互相关，用于度量两条曲线的形状相似度。
  • HAC (Hierarchical Agglomerative Clustering)：层次凝聚聚类。
  • Average linkage：簇间距离定义为所有样本对的平均距离。
  • NMI / ACC：聚类评估指标，分别基于互信息与匹配度。
  • F1 (outlier)：二分类 F1，用于评估 outlier 检测能力。
  • τd：距离阈值，决定新实例是否归入最近簇。
  • task-agnostic：与下游任务（异常检测）解耦，OmniCluster 的输出可对接多种模型。
