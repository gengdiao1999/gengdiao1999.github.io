二、背景（Background）

服务器 OS 灰色故障介于"健康"和"不健康"之间，包括性能下降、容量压力、I-O 抖动、内存抖动、随机丢包、非致命异常等。华为云故障工单显示从灰色故障到定位根因平均 8 小时。现有方法分侵入式（Panomara、OmegaGen，需改源码）和非侵入式（统计/特征学习/因果图）。

三、目的（Problems Solved）

  • 复杂因果关系：指标数百个，纯 PC/Granger 因果图过大。
  • 相关性利用不足：当前只用异常度，忽略与 KPI 的相关性。
  • 可解释性：缺乏传播路径信息。
  • 专家知识缺失：纯学习不能利用运维经验。
  • 非侵入式：不修改源码。
  • 多场景：GaussDB/Redis/Kafka/Tomcat。

四、核心原理（Principles）

系统总览：4 模块——数据采集、因果图学习、根因推断、传播路径推断。

关键概念：Gray Failure、KPI、Metric、Meta-metric、Granger、Skeleton Graph、Partial Correlation、Anomaly Score。

数学原理：

  • 异常度：

\[\]

\(\text{anomaly\_degree}(v_j) = \frac{\text{anomaly\_score}(v_j)}{\text{anomaly\_score}(v_i) + \text{anomaly\_score}(v_j)}\)


\[\]

\[\]

\(H'_{i,j} = \lambda \cdot \text{correlation}(v_j) + (1 - \lambda) \cdot \text{anomaly\_degree}(v_j)\)

五、算法详解（Algorithm）

  1. 输入 / 输出：OS 指标 + KPI → 根因排名 + 传播路径。
  2. 核心模块：gala-gopher 采集 → 骨架图 → 插入指标 → Granger 过滤 → 随机游走 → 最短路径。
  3. 伪代码：

def build_skeleton_graph(meta_metrics):
    edges = [
        ('network', 'CPU'), ('CPU', 'memory'),
        ('memory', 'disk'), ('disk', 'TCP'),
        ('TCP', 'network'), ('network', 'performance'),
    ]
    return edges

def plug_metrics(skeleton, metrics, m=2):
    structure = skeleton.copy()
    for meta in skeleton.nodes:
        top = sorted(metrics[meta], key=lambda x: -x.anomaly_score)[:m]
        for ind in top:
            structure.add(meta, ind)
            for ind2 in top:
                if ind != ind2:
                    structure.add_full_connect(ind, ind2)
    return structure

def granger_filter(structure, kpi_metric, time_series, w=36, p=0.05, maxlag=2):
    G = nx.DiGraph()
    for u, v in structure.edges:
        ts_u = time_series[u][-w:]
        ts_v = time_series[v][-w:]
        _, pval, _ = grangercausalitytests(
            np.vstack([ts_v, ts_u]).T, maxlag=maxlag, verbose=False)
        if pval[maxlag] < p:
            G.add_edge(u, v)
    nodes_reach_kpi = nx.ancestors(G, kpi_metric) | {kpi_metric}
    return G.subgraph(nodes_reach_kpi).copy()

def random_walk_infer(G, kpi_metric, anomaly_scores, lam=0.2, rho=0.2, N=1000):
    H = build_transition_matrix(G, kpi_metric, anomaly_scores, lam, rho)
    visits = {n: 0 for n in G.nodes}
    cur = kpi_metric
    for _ in range(N):
        nxt = np.random.choice(G.nodes, p=H[cur])
        visits[nxt] += 1
        cur = nxt
    return sorted(visits.items(), key=lambda x: -x[1])

def shortest_path_infer(G, root, kpi, anomaly_scores):
    G_undir = G.to_undirected()
    dist = {n: np.inf for n in G.nodes}
    prev = {n: None for n in G.nodes}
    dist[root] = 0
    pq = [(0, root)]
    while pq:
        d, u = heappop(pq)
        if u == kpi:
            break
        for v in G_undir.neighbors(u):
            w = anomaly_scores[u]**2 + anomaly_scores[v]**2
            alt = d + w
            if alt < dist[v]:
                dist[v] = alt
                prev[v] = u
                heappush(pq, (alt, v))
    return reconstruct_path(prev, kpi)
  1. 关键数学：见 §四。
  2. 复杂度：Granger $O(|E| \cdot w \cdot \text{maxlag})$；Dijkstra $O(|E| \log |V|)$；8.74 s/case。
  3. 示例：GaussDB CPU 耗尽 → cpu_user_msec↑ → disk_util↓ → sli_tps↓。

八、关键创新点（Key Innovations）

  • + 专家知识 + Granger 融合：减少误判。
  • + 偏相关 + 异常度联合：随机游走。
  • + 传播路径推断：可解释。
  • + 非侵入式：不需改源码。
  • + 多应用：GaussDB/Redis/Kafka/Tomcat。

九、实验与结果（Experiments）

  • 数据集：16 台 EulerOS，4 应用 × 4 类故障，1241 注入 + 135 工业。
  • Baseline：CauseInfer、MicroCause、TS-InvarNet、CIRCA。
  • 指标：AC@k、Avg@k、解释精度。
  • 结果：GrayScope 0.86/0.90/0.82；CauseInfer 0.23/0.25/0.21；MicroCause 0.68/0.75/0.64；TS-InvarNet 0.68/0.80/0.63；CIRCA 0.51/0.64/0.50。工业：网络 AC@3=0.83、磁盘 0.98、内存 0.94。解释精度 81.5%。8.74 s/case。
  • 消融：去骨架 0.82；用 PC 0.34；仅偏相关 0.74；仅异常度 0.80；DFS 0.73。
