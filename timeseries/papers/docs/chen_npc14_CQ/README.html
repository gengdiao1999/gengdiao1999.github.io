二、研究背景

2.1 高速网络发展的挑战

随着互联网应用的不断丰富，视频流媒体、音频流媒体、文件共享、实时语音通话等内容密集型应用日益普及，对骨干网络带宽的需求呈现爆发式增长。在这种背景下，核心路由器中的交换矩阵面临着前所未有的挑战：

  • 链路速率不断提升，从10Gbps向100Gbps甚至400Gbps演进
  • 64字节小包在100Gbps链路上的传输时间仅为5.12纳秒
  • 调度决策时间窗口急剧缩小，对调度算法的实时性提出了极高要求

传统的交换机架构如输入队列（IQ）交换机、组合输入交叉点队列（CICQ）交换机等，将主要缓冲区放置在线卡而非交换矩阵上。为了避免数据包在交换矩阵处发生冲突和损坏，这些架构在每个调度周期都要求线卡与交换模块之间进行一轮往返通信。这种通信往返带来的时延成为了高速交换机的性能瓶颈。

2.2 交叉点队列（CQ）交换机的兴起

为了克服上述局限性，学术界和工业界对交叉点队列（CQ）交换机的兴趣日益增长。CQ交换机的核心创新在于：

将数据包缓冲在每个交叉点，使用片上内存。每个输出端口的调度器可以独立地做出调度决策，仅基于同一列中缓冲区的状态，无需线卡与交换模块之间的通信，从而大幅降低调度时延。

然而，CQ交换机在多年前被认为难以实现，主要原因是片上存储器资源稀缺。随着半导体集成技术的发展，现代FPGA芯片已经能够实现具有较大交叉点缓冲区的CQ交换结构。研究表明，对于具有上百个端口的交换机，交叉点缓冲区可以存储超过3兆比特的数据包。

四、核心原理

4.1 CQ交换机模型

考虑一个 $N \times N$ 的CQ交换机，其中第 $i$ 个输入和第 $i$ 个输出分别记为 $I_i$ 和 $O_i$。$XB_{ij}$ 表示 $I_i$ 与 $O_j$ 之间的交叉点缓冲区，其中 $i, j = 1, \dots, N$。

基本假设：

  • 时间被划分为等长的时隙
  • 所有数据包在进入交换机前被分割为固定长度的信元
  • 交换机的所有内部和外部链路具有相同的容量，每个时隙传输一个信元
  • $XB_{ij}$ 的容量为 $L_{ij}$ 个信元

4.2 工作机制

在每个时隙开始时，每个输入端口可能有一个信元到达或没有信元到达。如果在时隙开始时有一个信元到达输入 $i$ 并去往输出 $j$，则该信元以先进先出（FIFO）方式缓存在 $XB_{ij}$ 中（如果缓冲区未满）。如果 $XB_{ij}$ 已满，则该信元被丢弃。

在同一时隙内，每个输出端口的调度器独立地根据某种调度算法从其列中的一个缓冲区中选择，并通过输出端口将队首（HOL）信元发送出交换机（如果所选缓冲区非空）。

4.3 基本定义

定义1（吞吐量）：交换机的吞吐量定义为当时间趋于无穷时，通过交换机的信元数量与到达交换机的信元数量之比。记为 $TP$。

定义2（丢失率）：交换机的丢失率定义为当时间趋于无穷时，被交换机丢弃的信元数量与到达交换机的信元数量之比。记为 $LR$。

命题1：对于具有有限缓冲区的交换机，如果交换机的平均信元到达率大于零，则交换机的吞吐量等于1减去交换机的丢失率。

证明：假设交换机的总缓冲区可以容纳 $L$ 个信元。令 $\lambda$ 表示当时间趋于无穷时交换机所有输入的平均到达率，$L^*(n)$ 表示时隙 $n$ 时交换机缓冲区中的信元总数，因此 $L^*(n) \leq L$。

定义 $C_a, C_l, C_t$ 分别为当时间趋于无穷时到达、丢失和通过交换机的信元总数。显然有：

$$C_a = \lim_{n \to \infty} \lambda n$$ $$C_l = \lim_{n \to \infty} \lambda n \cdot LR$$ $$C_t = \lim_{n \to \infty} \left( \lambda n - \lambda n \cdot LR - L^*(n) \right)$$

因此，吞吐量为：

$$TP = \frac{C_t}{C_a} = \lim_{n \to \infty} \left( 1 - LR - \frac{L^*(n)}{\lambda \cdot n} \right) = 1 - LR$$

定义3（时延）：交换机的时延定义为当时间趋于无穷时，所有通过交换机的数据包的平均时延。记为 $DL$。

定义4（工作保守调度）：如果使用某种调度算法，只要至少有一个去往该输出的缓冲区非空，交换机的任何输出就总是处于忙碌状态，则称该调度算法为工作保守的。否则，称为非工作保守的。

定义5（静态调度）：如果调度规则不随系统状态而变化，则称该调度算法为静态的。否则，称为动态的。

4.4 流量模型

假设每个输入的信元到达服从独立的伯努利过程，并且去往每个输出的概率固定。使用以下符号：

  • $\rho_i$：输入 $I_i$ 中信元到达过程的伯努利参数
  • $a_{ij}^k$：在给定时隙内有 $k$ 个信元到达 $XB_{ij}$ 的概率，$k = 0, 1$
  • $d_{ij}$：到达 $I_i$ 的任一信元去往输出 $O_j$ 的概率，满足 $\sum_{j=1}^{N} d_{ij} = 1$ 且 $0 \leq d_{ij} \leq 1$
  • $s_{ij}$：输出 $O_j$ 选择交叉点缓冲区 $XB_{ij}$ 的概率，满足 $\sum_{i=1}^{N} s_{ij} = 1$ 且 $0 < s_{ij} < 1$

根据上述条件，$XB_{ij}$ 的到达概率为：

$$a_{ij}^0 = P\{A_{ij} = 0\} = 1 - \rho_i \cdot d_{ij}$$ $$a_{ij}^1 = P\{A_{ij} = 1\} = \rho_i \cdot d_{ij}$$ $$a_{ij}^k = P\{A_{ij} = k\} = 0, \quad k \neq 0, 1$$

4.5 排队模型

定义随机变量 $Q_{ij}(m)$ 为时隙 $m$ 结束时 $XB_{ij}$ 中的信元数。根据上述条件，$Q_{ij}(m)$ 可以建模为一个离散时间的准生灭（Quasi-Birth-Death）过程。

4.6 稳态分布推导

令 $Q_{ij}$ 表示 $XB_{ij}$ 的稳态队列长度。根据离散时间准生灭过程的稳态概率公式，可以得到稳态队列长度分布如下：

$$\eta_{ij}^0 = \frac{1}{1 + \sum_{l=1}^{L-1} \left( \frac{(1 - s_{ij}) a_{ij}^1}{s_{ij} a_{ij}^0} \right)^l + \frac{1}{a_{ij}^0} \left( \frac{(1 - s_{ij}) a_{ij}^1}{s_{ij} a_{ij}^0} \right)^L}$$ $$\eta_{ij}^l = \eta_{ij}^0 \left( \frac{(1 - s_{ij}) a_{ij}^1}{s_{ij} a_{ij}^0} \right)^l, \quad l = 1, \dots, L-1$$ $$\eta_{ij}^L = \eta_{ij}^0 a_{ij}^0 \left( \frac{(1 - s_{ij}) a_{ij}^1}{s_{ij} a_{ij}^0} \right)^L$$

其中 $\eta_{ij}^l$ 定义为 $XB_{ij}$ 长度等于 $l$ 的稳态概率，即 $Q_{ij} = l$。

4.7 吞吐量分析

显然，到达 $XB_{ij}$ 的信元被丢弃的概率等于 $XB_{ij}$ 满的概率，即稳态下的 $\eta_{ij}^L$。

定义随机变量 $D_{ij}, D_i, D$ 分别为稳态下给定时隙内 $XB_{ij}$、输入 $I_i$ 和整个交换机丢弃的信元数。显然，$D_{ij}, D_i$ 只能取0或1。

在给定时隙内，到达输入 $I_i$ 的信元被丢弃的概率为：

$$P\{D_i = 1 | A_j = 1\} = \sum_{j=1}^{N} d_{ij} \eta_{ij}^L$$

进一步，在一个时隙内 $I_i$ 处丢弃信元的期望为：

$$E(D_i) = \rho_i \cdot P\{D_i = 1 | A_j = 1\} = \rho_i \left( \sum_{j=1}^{N} d_{ij} \eta_{ij}^L \right)$$

因此，在给定时隙内整个交换机丢弃信元的期望为：

$$E(D) = E\left( \sum_{i=1}^{N} D_i \right) = \sum_{i=1}^{N} \rho_i \left( \sum_{j=1}^{N} d_{ij} \eta_{ij}^L \right)$$

由此可得CQ交换机的丢失率：

$$LR = \frac{E(D)}{E(A)} = \frac{\sum_{i=1}^{N} \rho_i \left( \sum_{j=1}^{N} d_{ij} \eta_{ij}^L \right)}{\sum_{i=1}^{N} \rho_i}$$

其中随机变量 $A$ 表示在一个时隙内到达交换机的信元数，$E(A)$ 表示 $A$ 的期望。

因此，根据命题1，可以得到交换机吞吐量的闭式公式：

$$TP = 1 - LR = 1 - \frac{\sum_{i=1}^{N} \rho_i \left( \sum_{j=1}^{N} d_{ij} \eta_{ij}^L \right)}{\sum_{i=1}^{N} \rho_i}$$

4.8 时延分析

接下来分析CQ交换机的平均时延。同样，我们从特定的交叉点缓冲区 $XB_{ij}$ 开始分析。

令随机变量 $W_{ij}, W_i$ 分别表示稳态下信元在 $XB_{ij}$ 和输入 $I_i$ 中花费的时隙数（即时延）。假设在一个信元到达 $XB_{ij}$ 时，缓冲区长度 $Q_{ij} = l$（$0 \leq l < L$），并且该信元在 $XB_{ij}$ 中花费了 $n$ 个时隙。

我们只考虑未被交换机丢弃的信元的时延，因为被丢弃的信元的时延没有意义。因此有：

$$P\{W_{ij} = n | Q_{ij} = l\} = C_{n}^{n-l} (1 - s_{ij})^{n-l} (s_{ij})^{l+1}$$

其中 $n = l, l+1, \dots, \infty$。这个等式表示信元的时延 $W_{ij} = n$ 的概率等于在 $n$ 个时隙中缓冲区被选择 $l$ 次以将信元移动到队首，并且在 $n$ 个时隙后缓冲区被选择以发送信元的概率。

利用负二项分布的期望公式，可以得到：

$$E[W_{ij} | Q_{ij} = l] = \frac{l+1}{s_{ij}}$$

因此，信元在 $XB_{ij}$ 中的平均时延为：

$$E[W_{ij}] = \sum_{l=0}^{L-1} E[W_{ij} | Q_{ij} = l] \cdot P\{Q_{ij} = l | Q_{ij} < L\}$$ $$= \frac{1}{1 - \eta_{ij}^L} \sum_{l=0}^{L-1} \frac{l+1}{s_{ij}} \eta_{ij}^l$$

六、关键算法详解

6.1 静态随机非工作保守调度算法

算法描述

静态随机非工作保守调度算法是本论文理论分析的基础。该算法的核心特点是：

  • 每个输出调度器以固定概率选择其列中的每个交叉点缓冲区
  • 即使所选缓冲区为空，也不重新选择
  • 调度决策不依赖于当前缓冲区状态

伪代码实现

算法1：静态随机非工作保守调度算法

输入：
    - N：交换机端口数
    - s[1..N]：每个交叉点缓冲区的选择概率
    - buffer_state[1..N]：各缓冲区的状态（空/非空）

输出：
    - selected_index：选中的缓冲区索引

函数 StaticNonWorkConservingScheduling(N, s, buffer_state):
    // 1. 生成[0,1)均匀分布随机数
    r = RandomUniform(0, 1)

    // 2. 轮盘赌选择
    cumulative_prob = 0
    for i from 1 to N:
        cumulative_prob = cumulative_prob + s[i]
        if r < cumulative_prob:
            selected_index = i
            break

    // 3. 即使缓冲区为空也不重新选择（非工作保守）
    return selected_index

// 主调度循环
for each time_slot:
    for each output_port j from 1 to N:
        selected = StaticNonWorkConservingScheduling(
            N, s[:, j], buffer_state[:, j])

        if buffer_state[selected, j] is not empty:
            transmit_head_cell(selected, j)
        else:
            // 输出端口空闲，不重新调度
            output_idle(j)

6.2 工作保守随机调度算法

算法描述

工作保守随机调度算法在实际应用中更为常用。该算法的核心特点是：

  • 仅从非空缓冲区中进行随机选择
  • 保证只要有非空缓冲区，输出端口就不会空闲
  • 调度决策依赖于当前缓冲区状态

伪代码实现

算法2：工作保守随机调度算法

输入：
    - N：交换机端口数
    - buffer_state[1..N]：各缓冲区的状态（空/非空）

输出：
    - selected_index：选中的缓冲区索引，-1表示全部为空

函数 WorkConservingRandomScheduling(N, buffer_state):
    // 1. 收集所有非空缓冲区索引
    non_empty_buffers = []
    for i from 1 to N:
        if buffer_state[i] is not empty:
            non_empty_buffers.append(i)

    // 2. 如果无非空缓冲区，返回空闲
    if len(non_empty_buffers) == 0:
        return -1

    // 3. 从非空缓冲区中随机选择
    r = RandomInteger(0, len(non_empty_buffers) - 1)
    selected_index = non_empty_buffers[r]

    return selected_index

// 主调度循环
for each time_slot:
    for each output_port j from 1 to N:
        selected = WorkConservingRandomScheduling(
            N, buffer_state[:, j])

        if selected != -1:
            transmit_head_cell(selected, j)
        else:
            output_idle(j)

6.3 轮询（Round-Robin）调度算法

算法描述

轮询调度算法是实际应用中最常用的调度算法之一。该算法的核心特点是：

  • 以循环方式依次选择每个交叉点缓冲区
  • 如果所选缓冲区为空，则跳过并选择下一个
  • 保证公平性，每个缓冲区获得平等的服务机会

伪代码实现

算法3：轮询（Round-Robin）调度算法

输入：
    - N：交换机端口数
    - buffer_state[1..N]：各缓冲区的状态（空/非空）
    - last_selected：上一次选中的缓冲区索引

输出：
    - selected_index：选中的缓冲区索引
    - new_last_selected：更新后的指针位置

函数 RoundRobinScheduling(N, buffer_state, last_selected):
    // 1. 从上一次选中的下一个位置开始扫描
    start_pos = (last_selected % N) + 1
    selected_index = -1

    // 2. 循环扫描N个缓冲区
    for offset from 0 to N-1:
        current = ((start_pos + offset - 1) % N) + 1
        if buffer_state[current] is not empty:
            selected_index = current
            new_last_selected = current
            break

    // 3. 如果未找到非空缓冲区，保持指针位置
    if selected_index == -1:
        new_last_selected = last_selected

    return selected_index, new_last_selected

// 主调度循环（初始化）
last_selected = [0] * (N + 1)  // 每个输出端口独立维护指针

for each time_slot:
    for each output_port j from 1 to N:
        selected, last_selected[j] = RoundRobinScheduling(
            N, buffer_state[:, j], last_selected[j])

        if selected != -1:
            transmit_head_cell(selected, j)
        else:
            output_idle(j)

空间复杂度$O(1)$只需存储上一次选择的索引

6.4 算法性能对比定理

定理1：对于相同的输入流量和缓冲区大小，工作保守调度算法的吞吐量不小于非工作保守调度算法的吞吐量。

证明概要：工作保守调度算法从不浪费输出带宽，只要有信元可发送就一定会发送。而非工作保守调度算法可能在存在可发送信元时选择空缓冲区，导致输出带宽浪费。因此，工作保守调度算法的吞吐量总是大于或等于非工作保守调度算法的吞吐量。

这个定理具有重要的实际意义：

基于静态非工作保守调度算法推导的理论吞吐量可以作为实际工作保守调度算法性能的保守下界。这意味着如果理论分析表明某一缓冲区大小可以满足性能要求，那么实际使用工作保守算法时的性能只会更好。

7.2 缓冲区设计考量

7.2.1 存储资源计算

对于一个 $N \times N$ 的CQ交换机，假设每个信元大小为64字节（512比特），每个交叉点缓冲区大小为 $L$ 个信元，则总片上存储需求为：

$$Total\_Memory = N \times N \times L \times 512 \text{ bits}$$

例如：

  • 32 × 32交换机，L = 4：总存储 = 32 × 32 × 4 × 512 = 2,097,152 bits ≈ 256 KB
  • 64 × 64交换机，L = 4：总存储 = 64 × 64 × 4 × 512 = 8,388,608 bits ≈ 1 MB
  • 64 × 64交换机，L = 8：总存储 = 64 × 64 × 8 × 512 = 16,777,216 bits ≈ 2 MB

这些存储需求在现代FPGA芯片（如Xilinx Virtex Ultrascale+或Intel Stratix 10）中是完全可行的。

7.2.2 缓冲区大小选择策略

根据论文的理论分析和仿真结果，推荐以下缓冲区大小选择策略：

  1. 低负载场景（$\rho < 0.3$）：L = 1 或 L = 2 即可满足性能要求
  2. 中等负载场景（$0.3 \leq \rho < 0.7$）：L = 4 可以实现接近100%的吞吐量
  3. 高负载场景（$\rho \geq 0.7$）：L = 8 可以保证良好的时延性能

7.3 调度器实现

7.3.1 硬件实现考量

CQ交换机的调度器设计需要考虑以下硬件实现细节：

  • 并行性：N个输出端口的调度器完全独立，可以并行实现
  • 速度要求：调度决策必须在一个时隙内完成，对于100Gbps链路，时隙约为5ns
  • 状态访问：每个调度器需要访问N个缓冲区的状态信息
  • 仲裁逻辑：工作保守算法需要简单的仲裁逻辑来选择非空缓冲区

7.3.2 FPGA实现示例

使用Verilog HDL实现轮询调度器的核心逻辑：

module round_robin_scheduler #(
    parameter N = 32,          // 端口数
    parameter PTR_WIDTH = 5    // 指针位宽
) (
    input wire clk,
    input wire rst_n,
    input wire [N-1:0] buffer_not_empty,  // 缓冲区非空标志
    output reg [PTR_WIDTH-1:0] selected,  // 选中的缓冲区索引
    output reg valid                      // 有效输出标志
);

reg [PTR_WIDTH-1:0] last_ptr;  // 上一次选择的指针
reg [N-1:0] search_mask;       // 搜索掩码

// 循环优先级编码器
always @(*) begin
    integer i;
    selected = 0;
    valid = 0;
    search_mask = {N{1'b1}};

    // 从last_ptr + 1开始循环搜索
    for (i = 0; i < N; i = i + 1) begin
        if (buffer_not_empty[(last_ptr + 1 + i) % N] &&
            search_mask[(last_ptr + 1 + i) % N]) begin
            selected = (last_ptr + 1 + i) % N;
            valid = 1'b1;
            search_mask = {N{1'b0}};  // 停止搜索
        end
    end
end

// 更新最后选择的指针
always @(posedge clk or negedge rst_n) begin
    if (!rst_n) begin
        last_ptr <= 0;
    end else if (valid) begin
        last_ptr <= selected;
    end
end

endmodule

7.4 流量控制机制

虽然CQ交换机本身不需要输入-输出之间的流量控制，但为了防止丢包，仍需要考虑以下流量控制机制：

  1. 反压信号：当交叉点缓冲区接近满时，向上游发送反压信号
  2. 信用管理：基于信用的流量控制，每个输入维护各输出的可用信用
  3. 拥塞避免：通过ECN（显式拥塞通知）标记拥塞的信元

十三-补充、深入分析与扩展讨论

13b.1 CQ交换机的演进历史

CQ交换机的概念可以追溯到上世纪70年代的电话交换网络，但由于早期半导体技术的限制，长期未能在数据网络中得到广泛应用。本节回顾CQ交换机的演进历史：

13b.1.1 早期阶段（1970s-1990s）

在电话交换网络中，CQ结构以电子矩阵或机械开关的形式存在。每个交叉点都是一个独立的开关，可以连接或断开相应的输入输出。这种结构的优点是控制简单，缺点是需要大量的硬件资源。

13b.1.2 探索阶段（1990s-2000s）

随着数据网络的兴起，研究人员开始探索将CQ结构应用于分组交换。然而，由于片上存储器容量有限，初期研究主要关注小规模CQ交换机（如4×4或8×8）。这一时期的代表性工作包括Karol等人的初步分析。

13b.1.3 复兴阶段（2008年至今）

随着FPGA和ASIC技术的飞速发展，片上存储器容量大幅提升，CQ交换机重新受到关注。Rojas-Cessa等人2008年的工作重新评估了CQ架构的可行性，并展示了其在现代半导体技术下的潜力。

13b.2 性能瓶颈深度剖析

13b.2.1 存储瓶颈

CQ交换机最显著的瓶颈是片上存储器需求与端口数的平方关系。具体计算：

  • 对于 $N \times N$ 交换机，需要 $N^2$ 个交叉点缓冲区
  • 每个缓冲区大小为 $L$ 个信元
  • 每个信元大小为 $S$ 字节
  • 总存储需求：$M_{total} = N^2 \times L \times S$

对于实际的100Gbps 64端口交换机（L=8, S=64字节）：

$$M_{total} = 64^2 \times 8 \times 64 = 2,097,152 \text{ bytes} = 2 \text{ MB}$$

这在现代FPGA（如Xilinx Virtex Ultrascale+ VU13P，拥有94.5Mb BRAM）和ASIC（拥有数十MB片上SRAM）中完全可行。但对于超大规模交换机（如512×512），存储需求将达到128MB，超出了当前主流芯片的能力。

13b.2.2 调度瓶颈

虽然CQ交换机的每个输出调度器是独立的，但调度器内部仍需要在一个时隙内完成以下操作：

  1. 读取N个缓冲区的状态信息
  2. 执行选择算法（如轮询或加权随机）
  3. 更新选择指针和缓冲区状态

对于100Gbps链路（5.12ns时隙），这些操作必须在一个时钟周期内完成。考虑到现代FPGA和ASIC的主频通常为500MHz-1GHz，每个时钟周期为1-2ns，这意味着调度逻辑必须高度并行。

13b.2.3 功耗瓶颈

片上存储器是CQ交换机的主要功耗来源。根据典型FPGA的功耗模型：

  • BRAM动态功耗：约0.1W/Mb
  • BRAM静态功耗：约0.01W/Mb

对于64×64 CQ交换机（2MB BRAM）：

$$P_{BRAM} = 2 \times 8 \times (0.1 + 0.01) = 1.76 \text{ W}$$

加上调度逻辑和I/O的功耗，整个交换机的功耗约为5-10W，与传统IQ/CICQ交换机相当。

13b.3 流量模型对性能的影响详析

13b.3.1 独立伯努利模型

独立伯努利模型是论文理论分析的基础。在该模型下，每个时隙的到达事件相互独立，到达率为 $\rho$。这种模型适用于：

  • 聚合后的网络流量（中心极限定理）
  • 低负载场景
  • 短时间窗口内的流量

13b.3.2 突发流量模型

实际网络流量通常具有突发性。突发流量可以用以下参数描述：

  • 突发强度 $\beta$：突发期间到达率与平均到达率之比
  • 突发持续时间 $T_b$：单个突发的持续时长
  • 静默时间 $T_s$：突发之间的静默时长

突发流量下的性能分析更加复杂，因为状态不再满足马尔可夫特性。仿真结果显示，当突发强度 $\beta = 4$ 时，为了维持相同的吞吐量，缓冲区大小需要从L = 4增加到L = 8。

13b.3.3 自相似流量模型

自相似流量具有长程相关性，其自相关函数随延迟呈幂律衰减。这种流量模型更符合互联网骨干网和数据中心的真实流量特征。

自相似流量的关键参数是赫斯特指数H（0.5 < H < 1）。H越接近1，流量的长程相关性越强，突发性越显著。

13b.4 服务质量（QoS）支持

13b.4.1 优先级调度

CQ交换机可以通过扩展支持优先级调度。每个交叉点可以配置多个优先级队列，调度器按优先级顺序选择。

优先级调度的理论分析更加复杂，因为不同优先级之间存在相互影响。一般来说，高优先级流量的性能优于低优先级流量，但低优先级流量的性能下降幅度也需要控制。

13b.4.2 带宽预留

带宽预留是QoS支持的另一种重要机制。在CQ交换机中，可以为每个输出端口的特定输入预留服务时隙，从而保证带宽。

带宽预留的实现相对简单，只需要修改调度概率 $s_{ij}$。例如，如果为输入 $i$ 预留输出 $j$ 的20%带宽，只需将 $s_{ij}$ 设置为0.2即可。

13b.4.3 时延保证

在静态非工作保守调度下，信元的最大排队时延可以理论计算：

$$D_{max} = \frac{L}{s_{min}} + 1$$

其中 $s_{min}$ 是最小的调度概率。这个公式来自于最坏情况下，信元需要等待最多 $L$ 个调度才能被选中。

对于典型的均匀调度（$s_{ij} = 1/N$）：

$$D_{max} = L \times N + 1$$

对于32×32交换机（L = 4），最大排队时延为129个时隙，约660ns（5.12ns/时隙），远小于大多数应用的时延要求。

13b.5 可靠性与容错分析

13b.5.1 单故障容忍

CQ交换机具有天然的容错能力，因为各个交叉点缓冲区是独立的。如果某一个交叉点出现故障：

  • 只影响对应的输入-输出对
  • 其他输入输出对完全不受影响
  • 可以通过软件配置绕过故障交叉点

这种故障隔离特性是CQ架构的重要优势之一。

13b.5.2 冗余设计

为了提高可靠性，可以在CQ交换机中引入冗余设计：

  1. 交叉点冗余：为每个输出端口预留1-2个额外的交叉点缓冲区，当主缓冲区故障时切换到备用
  2. 调度器冗余：每个输出端口配备主备两个调度器，主调度器故障时自动切换
  3. 链路冗余：使用链路聚合（LAG）技术，提高输入输出链路的可靠性

13b.5.3 故障检测与恢复

故障检测机制包括：

  • 缓冲区溢出检测（统计异常）
  • 调度器空闲检测（长期未选中任何缓冲区）
  • ECC纠错码检测存储器错误
  • 周期性诊断测试

故障恢复时间通常小于50ms，满足电信级可靠性要求（99.999%可用性）。

13b.6 与新兴技术的结合

13b.6.1 光子交换

CQ架构与光子交换技术具有良好的兼容性：

  • 光子交叉点矩阵可以直接映射为CQ结构
  • 电子缓冲区可以放置在光-电-光转换节点
  • 调度器可以使用高速电子电路实现

目前已有研究展示了基于CQ架构的16×16光子交换原型，吞吐率达到1.28Tbps。

13b.6.2 机器学习辅助调度

虽然论文中的调度算法是确定性的，但可以引入机器学习技术进一步优化性能：

  • 使用强化学习在线调整调度概率
  • 使用监督学习预测流量模式并预配置缓冲区
  • 使用无监督学习检测异常流量模式

初步实验表明，机器学习辅助调度可以在高负载场景下进一步降低5-10%的平均时延。

13b.6.3 软件定义网络（SDN）集成

CQ交换机可以作为SDN架构中的基础交换单元：

  • SDN控制器可以通过OpenFlow协议配置每个交叉点的调度权重
  • 可以动态调整缓冲区大小以适应流量需求
  • 可以收集详细的队列统计信息用于网络管理

14.4 详细推导补充

14.4.1 准生灭过程的状态转移矩阵

对于交叉点缓冲区 $XB_{ij}$，其状态空间为 $\{0, 1, 2, \dots, L\}$，状态转移概率矩阵 $P$ 的元素如下：

对于状态 $l$（$0 \leq l \leq L$）：

  • 状态 $0$ 到状态 $0$ 的转移概率：$P_{0,0} = a_{ij}^0 + s_{ij} a_{ij}^1$（无到达，或到达且被立即调度）
  • 状态 $0$ 到状态 $1$ 的转移概率：$P_{0,1} = (1 - s_{ij}) a_{ij}^1$（到达且未被调度）
  • 对于 $1 \leq l \leq L-1$：
    • $P_{l,l-1} = s_{ij} a_{ij}^0$（无到达且被调度）
    • $P_{l,l} = (1 - s_{ij}) a_{ij}^0 + s_{ij} a_{ij}^1$（无到达且未被调度，或到达且被调度）
    • $P_{l,l+1} = (1 - s_{ij}) a_{ij}^1$（到达且未被调度）
  • 对于状态 $L$（缓冲区满）：
    • $P_{L,L-1} = s_{ij}$（被调度，无论是否有到达，因为新到达的会被丢弃）
    • $P_{L,L} = 1 - s_{ij}$（未被调度）

14.4.2 平衡方程求解

令 $\pi = (\pi_0, \pi_1, \dots, \pi_L)$ 表示稳态分布，则有以下平衡方程：

$$\pi_l \cdot P_{l,l+1} = \pi_{l+1} \cdot P_{l+1,l}, \quad l = 0, 1, \dots, L-1$$

结合归一化条件 $\sum_{l=0}^{L} \pi_l = 1$，可以求解出唯一的稳态分布。

定义比率 $r_{ij} = \frac{(1 - s_{ij}) a_{ij}^1}{s_{ij} a_{ij}^0}$，则有：

$$\pi_l = \pi_0 \cdot r_{ij}^l, \quad l = 1, 2, \dots, L-1$$ $$\pi_L = \pi_0 \cdot a_{ij}^0 \cdot r_{ij}^L$$

通过归一化条件可以得到：

$$\pi_0 = \frac{1}{1 + \sum_{l=1}^{L-1} r_{ij}^l + a_{ij}^0 \cdot r_{ij}^L}$$

14.4.3 渐近行为分析

当缓冲区大小 $L \to \infty$ 时（假设 $r_{ij} < 1$）：

$$\pi_0 \to \frac{1}{\frac{1}{1 - r_{ij}}} = 1 - r_{ij}$$ $$\pi_l \to (1 - r_{ij}) r_{ij}^l, \quad l \geq 1$$

此时系统的稳态分布退化为几何分布，与经典的M/M/1队列类似。

当缓冲区大小 $L \to \infty$ 时，吞吐量趋于：

$$TP \to \min\left(1, \sum_{i=1}^{N} \rho_i \cdot d_{ij}\right) / \sum_{i=1}^{N} \rho_i \cdot d_{ij}$$

这表明在无限缓冲区情况下，CQ交换机可以达到100%的吞吐量（在不超载的情况下）。

14.11 常见问题（FAQ）

Q1：为什么CQ交换机不需要线卡与交换模块之间的通信？

因为每个输出端口的调度器只需要查看其同一列的缓冲区状态，这些信息都在交换芯片内部。而IQ和CICQ架构需要将输入端口的队列状态发送给中央调度器，然后接收调度结果，导致额外的通信开销。

Q2：CQ交换机的主要缺点是什么？

主要缺点是片上存储需求为 $O(N^2)$，限制了交换机规模。对于大型交换机（如1024×1024），需要将CQ与其他架构（如Clos）结合使用。

Q3：本论文的理论结果对其他流量模型适用吗？

理论分析严格基于独立伯努利到达假设。对于其他流量模型（如自相似流量、突发流量），理论结果可能不再精确，但可作为参考。仿真结果显示，理论值通常是实际性能的下界。

Q4：如何选择交叉点缓冲区的大小？

根据论文的分析，对于典型的数据中心和骨干网应用，L = 4 是一个良好的折衷点。如果应用对时延极其敏感或负载非常高，可以考虑 L = 8 或更大。

Q5：CQ交换机适合多大规模的交换矩阵？

受限于片上存储，CQ交换机适合中小规模（不超过128×128）的交换矩阵。对于更大规模，可以使用CQ作为基本构建块，构造多级Clos网络。
