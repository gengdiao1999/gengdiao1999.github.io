二、背景（Background）

微服务架构的灵活可扩展性让大型在线服务系统能快速响应业务变化，但也让系统维护变得极其复杂。故障一旦发生，会沿着服务间调用链迅速传播。微服务故障诊断包含两个子任务：

  • Failure Detection（故障检测）：识别系统异常行为。
  • Failure Triage（故障分诊）：判断故障类型并派给对应工程团队。

运维人员通常持续收集三类可观测数据：metrics（指标）、logs（日志）、traces（追踪），每类数据各有特点：

  • Metrics：时序数据，能快速反映性能变化（如 CPU/内存/网络/响应时间/错误率）。但过度敏感，容易误报（如内存被有效释放时反而"低"）。
  • Logs：半结构化文本，记录系统事件。依赖开发者的日志规范；细微性能问题（如内存利用率异常）可能无日志反映，导致"资源不足"类故障难诊断。
  • Traces：调用链树，记录服务间调用。不提供实例内部信息，"系统组件损坏"类故障难单独诊断。

Table I 列出 10 类典型故障 + 各模态表现 + 推荐修复措施，揭示一个关键事实：没有任何单一模态能覆盖所有故障。例如：

  • "CPU 资源不足"只在 metric 体现；
  • "文件系统配置错误"只在 log/trace 体现；
  • "网络设备故障"在 metric + trace 体现；
  • "系统配置错误"三模态都有。

现有方法分两类问题：

  • 单模态方法：覆盖率低。
  • 多模态融合方法：要么映射到统一表示空间破坏模态结构（CloudRCA、DiagFusion），要么用监督学习无法泛化到新故障（Eadro、MicroCBR）。

UniDiag 提出用时序知识图谱（TKG） 把三模态融合，把故障诊断重新定义为"基于 TKG 的图分类/检索"——既保留模态内结构，又支持半监督的图嵌入学习。

三、目的（Purpose / Problems Solved）

论文给出三大挑战对应方案：

  • 挑战 1：微服务监控数据量巨大 + 多模态异构。微服务实例多、metric/log/trace 三模态格式完全不同，构造一个统一 KG 极其困难。解决方案：Stream-based Anomaly Detection——先用流式方法检测 metric/log 的异常，把"异常"才作为 KG 实体、正常数据不进 KG，从源头压低 KG 规模。
  • 挑战 2：KG 中关系异构 + 时序动态。不同类型的关系（metric 间、log 间、trace 间、跨模态）信息不同；实例状态随时间变化；实例数量动态增减。传统图嵌入方法难以处理。解决方案：MOGE = R-GCN + GRU + SOPOOL——R-GCN 捕捉异构关系结构、GRU 捕捉时序动态、SOPOOL 把动态节点嵌入汇聚为稳定图表示。
  • 挑战 3：故障模式不可枚举。监督学习方法难以泛化到未见故障。解决方案：用 TKG + 图嵌入，让"未见故障"如果和历史失败模式在图结构上相似也能被识别。

四、核心原理（Principles）

UniDiag 框架（Fig.2）包含 offline training 和 online diagnosis 两部分。

1) Multimodal TKG Construction（V-B）

  • Stream-based Anomaly Detection：对 metric 时序用流式异常检测（如基于显著性检验 / 残差 / 阈值）减少正常数据点；对 log 用流式模板聚类，把"突增/突减"的模板视为异常。
  • TKG 四元组 (h, r, t, τ)：把 (head, relation, tail, timestamp) 作为 TKG 的基本单元。例如：
    • 同一实例内：metric → log（"当 CPU 飙升时，error log 增加"）。
    • 跨实例：instance_A → instance_B（"调用延迟增加"）。
    • 不同时刻：t-1 → t（"状态演化"）。
  • Temporal stacking：把 TKG 在时间轴上切片成多个 KG snapshot，每个 snapshot 对应一个时间窗的图。

2) Microservice-Oriented Graph Embedding (MOGE, V-C)

  • Relation Embedding：把每种关系类型 r 编码为独立的向量。
  • R-GCN（Relation-aware Graph Convolutional Network）：对不同类型的关系分别做图卷积，捕捉异构结构。论文将其扩展为 R-GCN 编码 TKG 节点。
  • Entity Embedding：得到每个实体（实例/事件）的高阶表示。
  • GRU（Gate Recurrent Unit）：对每个实体的多时刻嵌入做时序聚合，捕捉状态演化。
  • SOPOOL（Second-Order Pooling）：把动态变化的节点嵌入汇聚为稳定的图级表示。SOPOOL 比 mean/max pooling 更强，因为它捕获节点间二阶统计量。

3) Failure Case Clustering（V-D）

  • 把训练集 TKG 嵌入后聚类（如 K-means）成"典型故障 case cluster"。

4) Online Diagnosis（V-E）

  • 新故障事件的 TKG 用同一 MOGE 编码；
  • 计算与各 cluster 中心的距离；
  • 取最近 cluster 的标签为故障类型；同时计算 anomaly score（基于 stream-based detection）判为是否真故障。

与现有方法差异：

  • vs 单模态方法：UniDiag 融合三模态，覆盖 Table I 中所有故障类型。
  • vs 特征融合方法（CloudRCA、DiagFusion）：UniDiag 用 TKG 保留模态间复杂关联结构，不是简单映射。
  • vs 知识图谱方法（MicroCBR）：UniDiag 用时序 KG + 图嵌入，MicroCBR 是案例检索。
  • vs 监督方法：UniDiag 通过 KG 嵌入 + 聚类实现半监督，对未见故障类型有泛化。

数学核心：

  • TKG 四元组：(h, r, t, τ)
  • R-GCN 更新：hi(l+1) = σ(Σr∈R Σj∈Nir (1/ci,r) Wr(l) hj(l) + W0(l) hi(l))
  • SOPOOL：图级表示 G = SOPOOL({h1, h2, ..., hN})
  • 故障分类：ŷ = argminc d(Encode(TKGnew), c) 其中 c 是 cluster 中心。

五、算法详解（Algorithm）

1. 输入 / 输出

  • 输入：metric、log、trace 三模态数据 + 训练时的 TKG 标签。
  • 输出：故障检测 (yes/no) + 故障分类（10 类）。

2. 核心模块

  • Stream-based Anomaly Detection：metric 流式异常 + log 模板聚类异常。
  • TKG Construction：构造 (h, r, t, τ) 四元组并时间分片。
  • MOGE：R-GCN + GRU + SOPOOL。
  • Failure Case Clustering：K-means 聚类。
  • Online Diagnosis：MOGE 编码 + 距离最近 cluster。

3. 伪代码

# === Offline Training ===
def build_tkg(metrics, logs, traces):
    # 1) 流式异常检测
    metric_anomalies = stream_anomaly_detect(metrics)
    log_anomalies = log_template_anomaly(logs)
    # 2) 构造 TKG 四元组
    quadruples = []
    for t in time_windows:
        for m in metric_anomalies[t]:
            quadruples.append((m_instance, 'has_metric_anom', m_metric, t))
        for l in log_anomalies[t]:
            quadruples.append((l_instance, 'has_log_anom', l_template, t))
        for span in traces[t]:
            quadruples.append((span.caller, 'calls', span.callee, t))
        # 跨模态关联
        for (m_inst, l_inst) in correlated(metric_anomalies[t], log_anomalies[t]):
            quadruples.append((m_inst, 'correlates_with', l_inst, t))
    return TKG(quadruples)

def moge_encode(tkg):
    # R-GCN 编码异构结构
    node_emb = RGCN(tkg)  # 每个节点嵌入
    # GRU 编码时序
    sequence_emb = GRU(node_emb)  # 每个节点的多时刻表示
    # SOPOOL 池化为图级表示
    graph_emb = SOPOOL(sequence_emb)
    return graph_emb

def train_unidiag(tkg_dataset):
    embeddings = [moge_encode(tkg) for tkg in tkg_dataset]
    clusters = KMeans(n_clusters=N).fit(embeddings)
    return clusters

# === Online Diagnosis ===
def diagnose(new_metrics, new_logs, new_traces, clusters):
    new_tkg = build_tkg(new_metrics, new_logs, new_traces)
    emb = moge_encode(new_tkg)
    # 与 cluster 中心距离
    dists = [||emb - c|| for c in clusters.cluster_centers_]
    return clusters.labels_[argmin(dists)]

4. 关键数学

TKG 四元组：

$$Q = \{(h, r, t, \tau) \mid h, t \in E, r \in R, \tau \in \mathbb{R}^+\}$$

R-GCN 更新：

$$h_i^{(l+1)} = \sigma\left(\sum_{r\in R}\sum_{j\in N_i^r}\frac{1}{c_{i,r}}W_r^{(l)}h_j^{(l)} + W_0^{(l)}h_i^{(l)}\right)$$

SOPOOL 图级表示：

$$G = \mathrm{SOPOOL}(\{h_1, h_2, ..., h_N\}) = \left[\bar{h}, \frac{1}{N}\sum_i h_i h_i^\top\right]$$

K-means 聚类：

$$\min \sum_{i=1}^{K}\sum_{x\in C_i}||x - \mu_i||^2$$

5. 复杂度分析

论文未给严格复杂度公式，强调：

  • 流式异常检测：减少 TKG 实体规模，复杂度近线性于"异常点"数量（远小于全量数据）。
  • R-GCN：每层 O(|E|·d2)（E 边数、d 嵌入维度）。
  • 在线诊断：0.6 秒（论文实测）。

6. 训练与推理

  • 训练：用历史故障 TKG 数据训练 MOGE 参数 + K-means 聚类中心。
  • 推理：新数据 → TKG → MOGE → 找最近 cluster → 故障类型。
  • 超参：嵌入维度、R-GCN 层数、GRU 隐藏层维度、K-means 簇数、Stream 异常阈值。

八、关键创新点（Key Innovations）

  • + 第一次把 TKG 应用于三模态微服务故障诊断：把 metric/log/trace 异常 + 跨模态关联 + 时间戳统一为 TKG 四元组，让异构数据有统一表示但保留各自结构。
  • + Stream-based Anomaly Detection 预处理：在构造 TKG 前先用流式方法过滤"正常"数据，让 TKG 实体规模可控、聚焦"真正异常"，避免 KG 爆炸。
  • + MOGE = R-GCN + GRU + SOPOOL：R-GCN 捕捉异构关系、GRU 捕捉时序动态、SOPOOL 池化为稳定图级表示——三件套解决"动态异构图"嵌入难题。
  • + 半监督 + 跨故障泛化：用 K-means 聚类作为故障类型判别器，新故障如果和某 cluster 中心在嵌入空间接近就被归到该类；对未见故障也有泛化能力。
  • + 在线 0.6s 故障诊断：R-GCN + GRU 都是高效的图/序列模型，能在亚秒级返回诊断结果。
  • + 跨数据集 12.7% / 4% F1 提升：相比最佳 baseline，平均 F1 从 0.752/0.683 提升到 0.869/0.723。

九、实验与结果（Experiments）

  • 数据集：2 个公开微服务 benchmark（164 个 failure case），由 §VI-A 详述。
  • Baseline：单模态（iSQUAD、PatternMatcher、DéjàVu、LogCluster、Yuan et al.、MEPFL）+ 多模态（CloudRCA、MicroCBR、DiagFusion、Eadro、TAD、HeMiRCA、MULAN）。
  • 评估指标：Precision、Recall、F1-score。
  • 关键结果数字（Table II / §VI-B）：
    • Failure Diagnosis F1：数据集 1 = 0.869、数据集 2 = 0.723。
    • 相对最佳 baseline 提升：+0.117（数据集 1）、+0.04（数据集 2）。
    • 在线诊断时间：≤ 0.6s。
    • Failure Detection F1：UniDiag 在两个数据集上均超越 baseline。
    • Failure Triage F1：UniDiag 在两个数据集上均超越 baseline。
    • 泛化性：对未见故障类型，UniDiag 表现稳定，baseline 普遍下降明显。
  • 消融实验：逐个去掉 R-GCN、GRU、SOPOOL、stream-based detection 模块，验证每个组件的贡献。
  • 效率分析：在线 0.6s、离线训练时间。
  • 超参分析：嵌入维度、cluster 数量等对 F1 的影响。

十、应用场景（Use Cases）

  • 微服务故障自动检测 + 分诊：覆盖 10 类典型故障（Table I）。
  • AIOps 平台核心模块：作为"故障诊断引擎"嵌入。

  • 新故障类型泛化：通过图嵌入空间相似度识别"没见过但类似"的故障。

  • 故障回溯：用 TKG 时间分片 + 跨实例关联，重现故障传播路径。

十二、术语表（Glossary）

  • TKG（Temporal Knowledge Graph）：时序知识图谱，把 (h, r, t) 扩展为 (h, r, t, τ)。
  • KG Snapshot：TKG 在某一时间窗的切片。
  • Stream-based Anomaly Detection：流式异常检测，用于 KG 构造前的预处理。
  • MOGE（Microservice-Oriented Graph Embedding）：UniDiag 提出的图嵌入方法。
  • R-GCN（Relation-aware Graph Convolutional Network）：关系感知图卷积网络。
  • SOPOOL（Second-Order Pooling）：二阶池化，捕获节点间二阶统计量。
  • GRU（Gate Recurrent Unit）：门控循环单元，LSTM 简化版。
  • Failure Detection：故障检测，判 yes/no。
  • Failure Triage：故障分诊，判故障类型。
  • K-means Clustering：聚类算法，用于典型故障 case 聚类。
  • EDA / CloudRCA / DiagFusion / MicroCBR：相关多模态故障诊断方法。
  • HipsterShop / Online Boutique：常用微服务 benchmark。
  • Instance / Service：微服务的实例 / 服务。
