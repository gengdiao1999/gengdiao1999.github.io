二、研究背景

2.1 排序在推荐系统中的重要性

排序是推荐系统的核心任务，旨在为用户提供有序的物品列表。排序质量直接影响用户满意度和系统收入。

  • 传统排序方法对每个物品独立评分，未考虑物品间的相互影响
  • Pairwise 和 listwise 方法虽尝试解决，但未显式建模物品间的影响

2.2 重排序的必要性

  • 物品间的相互影响在推荐中很常见（如价格比较、类别替代）
  • 重排序可以在初始排序列表的基础上进一步优化
  • RNN 方法（DLCM, GlobalRerank）存在编码距离衰减问题

三、论文目的与解决的问题

  • 提出个性化重排序问题：不同用户对物品间相互影响的感知不同
  • 使用 Transformer 的自注意力机制建模物品间全局交互
  • 引入个性化嵌入学习用户特定的编码函数
  • 发布电商重排序数据集

四、核心原理

4.1 传统排序损失函数

\[ L = \sum_{r \in R} \ell \{y_i, P(y_i|x_i; \theta) | i \in I_r\} \]

4.2 重排序损失函数

\[ L = \sum_{r \in R} \ell \{y_i, P(y_i|X, PV; \hat{\theta}) | i \in S_r\} \]

其中 X 是特征矩阵，PV 是个性化矩阵，\(\hat{\theta}\) 是重排序模型参数。

4.3 输入层嵌入

\[ E' = [x_i ; pv_i] \]

\[ E'' = E' + PE \]

\[ E = E'' W_E + b_E \]

4.4 自注意力机制

\[ \text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d}}\right) V \]

4.5 多头注意力

\[ S' = \text{MH}(E) = \text{Concat}(\text{head}_1, ..., \text{head}_h) W^O \]

\[ \text{head}_i = \text{Attention}(EW_i^Q, EW_i^K, EW_i^V) \]

4.6 输出层

\[ \text{Score}(i) = P(y_i|X, PV; \hat{\theta}) = \text{softmax}(F(N_x)W^F + b^F) \]

4.7 训练损失

\[ L = -\sum_{r \in R} \sum_{i \in S_r} y_i \log(P(y_i|X, PV; \hat{\theta})) \]

4.8 预训练模型损失

\[ L = -\sum_{i \in D} [y_i \log(P(y_i|H_u, u; \theta')) + (1-y_i) \log(1-P(y_i|H_u, u; \theta'))] \]

六、关键算法详解

6.1 PRM 前向传播算法

算法：PRM 前向传播
输入：初始列表特征 X, 个性化矩阵 PV, 位置嵌入 PE
输出：重排序分数列表

1: // 输入层
2: E' ← Concat(X, PV)  // [x_i; pv_i]
3: E'' ← E' + PE  // 加入位置嵌入
4: E ← E'' @ W_E + b_E  // 线性投影到 d 维

5: // 编码层 (Nx blocks)
6: for b = 1 to Nx do
7:     // 多头自注意力
8:     Q ← E @ W_Q, K ← E @ W_K, V ← E @ W_V
9:     scores ← Q @ K^T / sqrt(d)
10:    attn_weights ← softmax(scores)
11:    attn_output ← attn_weights @ V
12:    // 残差连接 + LayerNorm
13:    E ← LayerNorm(E + attn_output)
14:    // 前馈网络
15:    FFN_output ← FFN(E)
16:    // 残差连接 + LayerNorm
17:    E ← LayerNorm(E + FFN_output)
18: end for

19: // 输出层
20: logits ← E @ W^F + b^F
21: scores ← softmax(logits)
22: 返回 scores

复杂度分析：

  • 自注意力计算：O(n^2 * d)，其中 n 是列表长度，d 是维度
  • 前馈网络：O(n * d * d_ff)，其中 d_ff 是 FFN 隐藏维度
  • 总复杂度：O(Nx * n * (n*d + d*d_ff))
  • 相比 RNN 的 O(n * d)，Transformer 可并行化，实际推理更快

6.2 预训练模型生成 PV

算法：生成个性化向量 PV
输入：用户行为历史 Hu, 用户侧信息 u
输出：个性化向量 pv_i

1: // 用户行为嵌入
2: h_emb ← EmbeddingLayer(Hu)
3: // 用户侧信息嵌入
4: u_emb ← EmbeddingLayer(u)
5: // 物品特征嵌入
6: i_emb ← EmbeddingLayer(item_features)
7: // 拼接并过隐藏层
8: hidden ← ReLU(Concat(h_emb, u_emb, i_emb) @ W_h + b_h)
9: // 输出点击概率
10: prob ← Sigmoid(hidden @ W_out + b_out)
11: // 提取隐藏层向量作为 PV
12: pv_i ← hidden  // 在 Sigmoid 之前的向量
13: 返回 pv_i

1. Transformer 在推荐系统中的适用性分析

Transformer 的自注意力机制在推荐重排序中具有独特优势。

  • O(1) 编码距离：任意两个物品之间的交互距离为 1
  • 可并行化：所有物品的注意力可以同时计算
  • 长距离依赖：列表头部和尾部的物品也能直接交互
  • 相比 RNN 的 O(n) 编码距离，自注意力消除了信息衰减
  • 但在推荐场景中，物品高度同质化，多头注意力提升有限

2. 个性化重排序的理论基础

不同用户对物品间相互影响的感知不同，这是个性化重排序的理论基础。

  • 价格敏感用户：相似价格物品间的影响更大
  • 品牌偏好用户：同品牌物品间的影响更强
  • 无明显意图用户：列表应更注重多样性
  • 全局编码函数无法捕捉这些差异
  • 个性化向量 PV 提供了用户特定的视角

3. 预训练 vs 端到端学习 PV 的权衡

11. 与学习排序方法的关系

重排序与学习排序（LTR）方法既有联系又有区别。

  • LTR 中的 pointwise/pairwise/listwise 方法关注标签空间优化
  • 重排序关注特征空间中物品间的交互
  • PRM 在特征空间建模交互，与标签空间方法互补

扩展分析 1：Transformer 在推荐系统中的理论基础

自注意力机制的数学分析

自注意力机制的核心计算：

\[ \text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d}}\right) V \]

在推荐重排序中，Q=K=V=E（特征矩阵），因此：

\[ A_{ij} = \frac{\exp(e_i^T e_j / \sqrt{d})}{\sum_k \exp(e_i^T e_k / \sqrt{d})} \]

\(A_{ij}\) 表示物品 i 对物品 j 的注意力权重，即物品 j 对物品 i 的影响程度。

与 RNN 编码的理论对比

RNN 的编码过程：

\[ h_t = f(h_{t-1}, x_t) \]

物品 i 对物品 j 的交互需要经过 \(|i-j|\) 步传递，信息衰减与距离成正比。

而自注意力中，任意两个物品的交互距离为 1，不存在衰减。

扩展分析 2：个性化嵌入的深度分析

PV 的数学表示

个性化矩阵 PV 的每一行 \(pv_i \in \mathbb{R}^{d_{pv}}\) 对应列表中物品 i 的个性化嵌入。

PV 由预训练模型生成，该模型的输入为：

  • 用户行为历史 \(H_u = [item_1, item_2, ..., item_m]\)
  • 用户侧信息（性别、年龄、购买力等级等）
  • 候选物品特征

预训练模型的隐藏层向量作为 PV，捕捉了用户的通用偏好。

PV 的信息论分析

PV 携带了关于用户偏好的信息量：

\[ I(PV; Y|X) = H(Y|X) - H(Y|X, PV) \]

当 \(I(PV; Y|X) > 0\) 时，PV 提供了原始特征 X 之外的额外信息。

实验结果证实了这一点：PRM-Personalized 相比 PRM-BASE 在 MAP 上提升 4.5%。

候选架构

  • DeepFM：结合 FM 和 DNN 的混合模型

  • FNN (Factorization-supported Neural Network)：FM 初始化的 DNN

扩展分析 4：损失函数的设计分析

Pointwise 交叉熵损失

PRM 使用负对数似然损失：

\[ L = -\sum_{r \in R} \sum_{i \in S_r} y_i \log(P(y_i|X, PV; \hat{\theta})) \]

这是标准的 pointwise 损失，每个物品独立计算损失。

Pairwise/Listwise 损失的可能性

论文提到未来可以探索 pairwise 或 listwise 损失：

  • Pairwise：比较同一列表中两个物品的相对顺序
  • Listwise：直接优化整个列表的排序质量
  • 结合特征空间（Transformer）和标签空间（Listwise）的优化可能是更好的方向

扩展分析 5：在线推理的工程优化

推理延迟分析

PRM 的推理延迟由以下部分组成：

  • 特征提取：从初始列表提取特征（~1ms）
  • PV 查找：从预训练模型获取 PV（~1ms）
  • Transformer 前向传播：O(Nx * n^2 * d)（~5-10ms for n=30, d=64）
  • Softmax + 排序：O(n log n)（<1ms）

总延迟约 7-12ms，满足推荐系统的严格延迟要求（通常 <50ms）。

并行化策略

  • 多头注意力可以并行计算
  • FFN 对每个位置独立计算
  • 相比 RNN 的顺序计算，Transformer 天然支持并行
