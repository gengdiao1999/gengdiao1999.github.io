1. 文档信息速览

Label-Less 是面向 KPI 时间序列的半自动标注工具，发表于 INFOCOM 2019。

核心思想：使用 Isolation Forest 进行无监督异常检测 + DTW 加速的异常相似度搜索，减少 90% 以上标注时间。

属性 值
方法 Label-Less
会议 INFOCOM 2019 (CCF-A)
核心 iForest + DTW + UCR Suite
数据集 30条真实KPI(6个月)
最佳F1 0.95
响应时间 <0.5秒/KPI
标注减负 >90%

2. 研究背景

KPI 异常检测对互联网服务可靠性至关重要，但缺乏大规模高质量标注数据集。

手工标注一个 6 个月长度的 KPI 需要数小时，5 家大公司最终只能标注 30 条 KPI。

3. 论文目的与解决的问题

  1. 人工标注耗时: 6个月KPI需数小时
  2. 一致性难: 不同时段模式不同
  3. 专家稀缺: 需要领域知识
  1. iForest无监督检测候选异常
  2. 基于模板的DTW相似度搜索
  3. UCR Suite加速DTW到<0.5秒

4. 核心原理（含数学公式）

DTW距离: \( D_{i,j} = d(i,j) + \min\{D_{i,j-1}, D_{i-1,j}, D_{i-1,j-1}\} \)

LB_Keogh: \( LB_{Keogh}(q,c) = \sum_i (c_i-U_i)^2 I(c_i>U_i) + (c_i-L_i)^2 I(c_i

iForest分数: 基于随机森林的路径长度计算异常分数

5. 系统/算法整体架构

6. 关键算法详解

算法: 异常相似度搜索

输入: 模板q, 候选异常C
输出: Top-k相似异常

for c in C:
  best_sofar = h.root
  if LB_Kim(q,c) > best_sofar: prune
  if LB_Keogh(q,c) > best_sofar: prune
  if LB_Keogh_Rev(q,c) > best_sofar: prune
  else:
    计算cDTW
    if cDTW + LB_Keogh > best_sofar: prune
    h.push(c, dist)
return h.top_k()

flowchart TD A["原始KPI"] --> B["iForest检测"] B --> C["候选异常"] C --> D["选模板"] D --> E["DTW搜索"] E --> F["Top-k结果"] F --> G["人工确认"] 
算法: iForest特征提取

输入: KPI序列X
输出: 异常分数

features = []
for model in [Diff, MA, WMA, EWMA, ARIMA, HW]:
    pred = model.forecast(X)
    error = |X - pred|
    features.append(error)

X_feat = stack(features)  # n×6
scores = iForest.fit_predict(X_feat)
return scores
