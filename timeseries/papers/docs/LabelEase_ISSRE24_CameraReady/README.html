1. 文档信息速览

LabelEase 是面向微服务架构的追踪数据（Trace）半自动标注工具，发表于 ISSRE 2024。

核心创新：使用图神经网络（GAT）进行 Trace 表示学习，结合主动学习和混合代表性选择策略，显著减少人工标注量。

2. 研究背景

微服务系统每天产生海量 Trace 数据（eBay 约 1500亿/天）。人工标注这些 Trace 需要大量时间和专家知识。

现有无监督 Trace 异常检测方法在有大量异常数据时无法准确区分正常和异常模式。

3. 论文目的与解决的问题

  1. 海量数据: 每天1500亿条Trace
  2. 复杂结构: Trace有复杂的调用图
  3. 根因难定: 需多Trace综合分析
  1. GNN编码Trace表示
  2. 聚类选择代表性Trace供标注
  3. SBFL计算可疑度+可视化辅助根因标注
  • 首个半自动Trace标注工具
  • 主动学习减少99.9%标注量
  • F1达0.99/0.98
  • 发布高质量Trace数据集

4. 核心原理（含数学公式）

余弦相似度: \( d(x,y) = 1 - \frac{x \cdot y}{\|x\|\|y\|} \)

三元组损失: \( L(A,B,C) = \max\{d(G_A,G_B)-d(G_A,G_C)+margin, 0\} \)

SBFL可疑度: \( Score_O = \frac{O_{ef}}{\sqrt{(O_{ef}+O_{ep})(O_{ef}+O_{nf})}} \)

F1分数: \( F_1 = 2 \times \frac{P\times R}{P+R} \)

6. 关键算法详解

算法: LabelEase Trace标注

1. Trace预处理: 提取语义向量+时间特征+状态码
2. 构建Trace图(GAT+GCL)
3. 三元组训练GNN
4. GNN编码所有Trace为64维向量
5. K-means聚类(k=标注预算)
6. 选择聚类中心供人工标注
7. 基于标注结果自动标注其余Trace
8. SBFL计算服务可疑度
9. 可视化服务拓扑供根因标注

7. 实现细节

  • Python 3.7 + PyTorch 1.5.0
  • GAT + 图对比学习
  • Batch=32, lr=0.0001, epoch=15
  • 2×16C32T Xeon Gold 5218, V100S

输入: 向量化Trace, k=标注预算

centroids = KMeans(Trace_vecs, k)
for c in centroids:
    label = human_label(c)
    cluster = assign_to_cluster(c)
    for t in cluster:
        auto_label(t, label)
return labeled_traces
