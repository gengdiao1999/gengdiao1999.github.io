二、背景（Background）

随着移动技术发展，WiFi 在手机、平板、便携游戏机中普遍存在。借助随身设备，企业 WiFi 网络能在室内层面追踪人，比 GPS 或蜂窝基站更细粒度。这种移动数据集有巨大价值，可用于基于位置的社交网络、近邻营销、移动建模、智能交通等。

然而，人类移动轨迹非常敏感。研究 [3] 显示一个人访问最多的两个位置最可能是家和公司。如果轨迹数据在发布前未妥善脱敏，用户隐私会被严重侵犯。早期研究 [4] 表明，即使身份被匿名，仅用 4 个随机时空点就能重新识别 95% 的用户。

为解决这一风险，已有大量工作试图发布隐私保护的轨迹数据集。其中，k-匿名 [15] 被广泛使用来防止重新识别攻击。然而已有定性认识：k-匿名在敏感属性多样性低时仍存在风险（即属性链接攻击 [16]）。l-多样性 [16] 要求 k-匿名集合中敏感属性的多样性足够大。

三、目的（Problems Solved）

  • k-匿名的 attribute linkage 漏洞：当敏感属性多样性低时，k-匿名仍泄露周期性时间表。
  • 缺乏定量研究：首次给出 schedule-leakage 的量化指标。
  • 校园周期性行为场景：以 16 周 WiFi 轨迹为代表场景。
  • 真实数据驱动：用清华校园 2,670 AP、111 楼、10,126 学生 16 周数据。
  • 隐私 vs 实用性权衡：在隐私保护与数据可用性之间寻找平衡。

四、核心原理（Principles）

系统总览：论文提出 schedule-leakage 攻击模型：(1) 把每天分成 T=30 时段，统计 16 周的访问位置；(2) 提取最频繁位置形成 periodic schedule；(3) 用 M 个随机时空点作 QID 执行 4-匿名攻击；(4) 用 prob-m 和 ssr 两指标量化攻击成功率。

关键概念：

  • Trajectory：个体的时空序列。
  • Periodic Schedule：周期性时间表。
  • k-anonymity：k-匿名。
  • QID：准标识符。
  • Record / Attribute Linkage：记录链接 / 属性链接。
  • l-diversity：l-多样性。
  • prob-m：最大攻击概率。
  • ssr：敏感时段率。
  • ntr：非空时段率。

数学原理：

  • 攻击概率（某时段）：

\[\]

P_n = A_n / (k \cdot C)


\[\]
  • 最大攻击概率：

\[\]

\text{prob-m} = \max_{n=1}^{T} P_n


\[\]
  • 敏感时段率：

\[\]

\text{ssr} = \#(\text{sensitive time slots}) / T


\[\]
  • 非空时段率：

\[\]

\text{ntr} = \#(\text{non-empty time slots}) / T


\[\]

五、算法详解（Algorithm）

  1. 输入 / 输出：输入：清华校园 16 周 WiFi 轨迹、721 志愿者 4,412 门课课表。输出：每个个体在 4-匿名下的 prob-m 与 ssr 分布。
  2. 核心模块：轨迹预处理；周期表提取；QID 构造；k-匿名集构造；prob-m / ssr 计算。

def extract_periodic_schedule(traj_i, weeks=16, T=30, threshold_weeks=9):
    counts = [[0] * N_BUILDINGS for _ in range(T)]
    for (t, b) in traj_i:
        slot = time_to_slot(t)
        counts[slot][b] += 1
    schedule = []
    for slot in range(T):
        max_b = argmax(counts[slot])
        if counts[slot][max_b] > threshold_weeks:
            schedule.append((slot, max_b))
        else:
            schedule.append((slot, None))
    return schedule

def build_anonymous_set(target_i, all_trajs, M=3):
    qid = random_sample(target_i.traj, M=M)
    S = [j for j, tr in enumerate(all_trajs) if contains_all(tr, qid)]
    return S

def compute_prob_m(S_i, all_schedules, k=4, weeks=16):
    L = stack_periodic_schedules([all_schedules[i] for i in S_i])
    max_p = 0.0
    for n in range(L.shape[0]):
        counter = Counter(L[n])
        top_count = counter.most_common(1)[0][1]
        p = top_count / (k * weeks)
        if p > max_p:
            max_p = p
    return max_p

复杂度分析：周期表提取 O(|traj|)；匿名集构造 O(N·M)；prob-m 计算 O(T·k)。

训练与推理：纯数据分析，无训练。

示例：学生 ID1 每周二 13:30-15:05 都在 building a（14/16 周）；ID2/3 类似；P_n=0.64；prob-m 可达 0.85；ssr=0.33。

八、关键创新点（Key Innovations）

  • + 首次量化周期性时间表泄露：从定性到定量（prob-m、ssr 两指标）。
  • + 大规模真实校园数据：10,126 学生、16 周、2,670 AP、111 building。
  • + 4-匿名仍 28% 完全泄露：揭示 k-匿名在周期性数据上的不足。
  • + 与 721 志愿者课表对齐：用 4,412 门课验证周期表提取的准确性。
  • + Privacy vs Utility 讨论：l-diversity 会显著降低数据可用性。

九、实验与结果（Experiments）

  • 数据集：清华校园 16 周 WiFi 轨迹；10,126 学生；2,670 Cisco AP；111 building。
  • 真实课表：721 TUNow 志愿者 4,412 门课。
  • 设置：T=30 时段；M=3 QID；k=4 匿名；threshold 60%。
  • 关键结果数字：56% 匿名集大小=1（直接重识别）；4-匿名下 28% 完全泄露；14% prob-m>0.8；平均 prob-m=0.65；周期表提取 3,548 vs 真实 4,412（覆盖 80%+）。
  • 基线方法：top-2 location 等 k-匿名方案。
  • 可视化：prob-m、ssr、ntr 三个 CDF；ssr/ntr 接近 1。
