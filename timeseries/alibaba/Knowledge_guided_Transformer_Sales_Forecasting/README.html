2. 研究背景

时间序列预测（TSF）在金融投资、天气预报、销售预测等领域有重要应用。在电商场景中，时间序列销售预测（TSSF）对商品选品、库存管理、营销策略有极大价值。

2020 年天猫双 11 的 GMV 达 4982 亿元，背后是数百万商家、数千工厂、电商平台等供应链的密切合作。其中，选品与营销活动的协调是关键，因为消费者对促销（如价格折扣、优惠券、广告投放）极其敏感。

传统 TSF 方法分为三类：

  • 经典时间序列：Brockwell & Davis、Seger、Salinas、Flunkert，仅基于历史观测值预测。
  • 深度学习方法：LSTNet、DeepAR，结合时序特征与生成能力。
  • 注意力模型：Transformer 衍生出 LogTrans、Informer、Autoformer 等。

这些方法有一个共同限制：只利用历史信息，无法利用未来已知信息。在 TSSF 场景中，许多未来信息是可提前获取的：

  • 商品价格：未来 15 天的定价已确定。
  • 营销活动：是否参加双 11、是否打折、是否参加聚划算等都是预先规划好的。
研究空白：现有 TSSF 方法完全忽略未来已知信息，无法在促销期间准确预测销售爆发。先前一些工作如 Qi 等 2019 试图显式建模"目标商品-邻近商品的关联"；Xin 等 2019 修改 GRU 引入预销阶段信息；Ekambaram 2020 利用稀疏注意力跨商品学习。但这些方法部分泄露未来信息或不真正利用未来信息。缺乏公开的电商销售预测数据集。

3.论文目的与解决的问题

本文提出 Aliformer，一个知识引导的 Transformer，把未来已知信息（产品相关：价格、广告；平台相关：促销级别、级别、时间、品类）注入到预测模型。

核心贡献：

  1. Aliformer 架构：在 vanilla Transformer 基础上加入 AliAttention 层，用未来知识引导 attention 修正；同时提出未来强调训练策略。
  2. TMS 数据集：首个大规模电商销售预测公开数据集，120万商品，86 维特征，覆盖 200 天。
  3. 实际部署：自 2021-05 部署阿里电商，验证 GMV 覆盖率提升 4.73%。

4.核心原理

4.1 问题形式化

给定历史统计与知识信息：

\[ \mathbb{S} = \{s_1^{(n)}, s_2^{(n)}, ..., s_T^{(n)}\}, \quad \mathbb{K} = \{k_1^{(n)}, k_2^{(n)}, ..., k_T^{(n)}\} \]

对未来 \(L\) 步：

\[ \{y_{T+1}^{(n)}, ..., y_{T+L}^{(n)}\} = f_h(x_1^{(n)}, ..., x_T^{(n)}) \]

其中 \(x_t^{(n)} = \text{Emb}(\{s_t^{(n)}, k_t^{(n)}\})\)。对未来时段，\(k\) 已知（知识信息）。

未来知识分类：

  • 商品相关：商品本身、价格、广告投入、是否参加活动。
  • 平台相关：促销活动级别、时间、品类等。

4.2 Vanila Self-Attention

vanilla attention（来自 BERT/Vaswani 2017）计算：

\[ \text{Att}(i,j) = \frac{(x_i^h W_Q)(x_j^h W_K)^T}{\sqrt{d}}, \quad x_i^{h+1} = \sum_j \text{Softmax}(\text{Att}(i,j)) \cdot (x_j^h W_V) \]

每层：

\[ x^h = \text{Emb}(\{s_1, k_1\}, ..., \text{Emb}(u_{T+L}, k_{T+L})), \quad x^{h+1} = \text{VSA}(x^h) \]

局限：直接对整个输入做双向 self-attention 可能导致未来信息泄露——因为所有 token 都参与 attention，模型可能"作弊"看到未来。

4.3 AliAttention 层

AliAttention 同时利用集成信息 \(x_t^{(n)}\) 和知识 \(\tilde{x}_t^{(n)}\)。对集成输入 \(x_t^{(n)} \in \mathbb{R}^{B \times T \times C}\)，知识 \(\tilde{x}_t^{(n)} \in \mathbb{R}^{B \times T \times C}\)，AliAttention 计算：

\[ \text{Att}(i,j) = \frac{(x_i^h W_Q^I)(x_j^h W_K^I)^T}{\sqrt{2d}}, \quad \widetilde{\text{Att}}(i,j) = \frac{(x_i W_Q^{\overline{I}})(x_j W_K^{\overline{I}})^T}{\sqrt{2d}} \] \[ \text{Att}(i,j)^* = \text{Att}(i,j) + \widetilde{\text{Att}}(i,j) \] \[ x_i^{h+1} = \sum_j \text{Softmax}(\text{Att}(i,j)^*) \cdot (x_j^h W_V^I) \]

核心思想：\(\widetilde{\text{Att}}\) 是"知识引导分支"，将未来知识作为"明确"信息注入到 attention 中；而原始 Att 仍然只基于历史。

4.4 未来强调训练策略

朴素训练会让模型对"可学习 token"过拟合，难以利用未来知识。Aliformer 引入span masking（来自 Joshi 2020）：

  • 在训练中，序列中段被 mask（替换为可学习 token）。
  • 模型必须依赖未来知识来预测 mask 段。

span masking 概率 \(p_2\) 控制。

6.关键算法详解

6.1 AliAttention 完整流程

AliAttention 的计算步骤：

  1. 对集成输入 \(x_t^{(n)}\)，分别用 \(W_Q^I, W_K^I, W_V^I\) 投影。
  2. 对未来知识 \(\tilde{x}_t^{(n)}\)，分别用 \(W_Q^{\bar{I}}, W_K^{\bar{I}}\) 投影。
  3. 计算两条 attention：\(\text{Att}\)（仅基于集成输入）和 \(\widetilde{\text{Att}}\)（仅基于知识）。
  4. 两条 attention 相加：\(\text{Att}^* = \text{Att} + \widetilde{\text{Att}}\)。
  5. Softmax + 乘以 V，得到下一层表示。

6.2 未来强调训练策略

在训练阶段：

  • 对每个序列，以概率 \(p_1\) 选择 naive 策略（不 mask），以概率 \(p_2\) 选择 span masking。
  • span masking：在序列中段 mask 一段连续 token，让模型只能用未来知识预测。

未来强调训练策略让模型"被迫"利用未来知识而非作弊。

6.3 伪代码

伪代码：Aliformer 前向推理

Algorithm 1: Aliformer Forward Pass
Input: 历史统计 s_1..s_T, 知识 k_1..k_{T+L}
Output: 未来预测 y_hat_{T+1..T+L}

1. // Embedding
2. x ← []  // 集成输入
3. for t in 1..T:
4.    x.append(Emb(s_t, k_t))
5. x_tilde ← []  // 知识输入
6. for t in 1..T+L:
7.    x_tilde.append(Emb(k_t))  // 仅知识
8. // AliAttention stack
9. for h in 1..12:
10.   // 计算集成 attention
11.   Q_I = x @ W_Q_I
12.   K_I = x @ W_K_I
13.   V_I = x @ W_V_I
14.   Att = (Q_I @ K_I.T) / sqrt(2d)
15.   // 计算知识 attention
16.   Q_K = x_tilde @ W_Q_K
17.   K_K = x_tilde @ W_K_K
18.   Att_bar = (Q_K @ K_K.T) / sqrt(2d)
19.   // 融合
20.   Att_star = Att + Att_bar
21.   x = Softmax(Att_star) @ V_I
22. // 输出预测
23. y_hat = x[T+1..T+L]
24. return y_hat

11. 工程落地与部署

下图给出 Aliformer 在阿里电商的部署：

flowchart LR A[商品过去 200 天销售数据] --> B[Aliformer 推理] C[未来 15 天促销计划] --> B B --> D[预测未来 15 天销售值] D --> E{排序 top 100 万} E --> F[平台促销候选商品] F --> G[商家使用] G --> H[流量 + 销售] H --> I[GMV 统计] I --> J{效果评估} J -- 提升 --> K[继续使用 Aliformer]

11.1 部署清单

  1. 模型：12 层 Aliformer，12 头 attention。
  2. 数据：阿里 ODPS 系统，过去 200 天 + 未来 15 天计划。
  3. 推理：8 × V100 GPU，每商品 ~10ms。
  4. 集成：阿里电商平台商品选品流程。

11.2 实际效果

自 2021-05 部署以来，Aliformer 用于天猫商品选品。具体流程：从亿级商品中预选 top 100 万参加平台促销活动。结果显示：Aliformer 选品的商品 GMV 覆盖率 74.96%，相比 Informer 的 70.23% 提升 4.73%，带来显著电商平台利润。

12.局限性与未来工作

  1. 仅依赖商品/平台相关未来知识：未考虑供应链、宏观市场等外部因素。
  2. 静态知识表示：未来知识用 one-hot 或数值嵌入，未做动态推理。
  3. 训练与推理的分布偏移：促销期间 vs 平时数据分布差异大。
  4. 数据集规模：TMS 已发布但商品范围有限。

12.1未来工作

  • 扩展到更多电商场景（直播、跨境）。
  • 结合强化学习做动态选品。
  • 结合大语言模型（LLM）做商品/活动语义理解。
  • 多商品关联建模（graph + transformer）。

13.总结与启示

Aliformer 是首个把"未来知识"显式注入时间序列预测的工业级 Transformer，在阿里电商平台部署并验证有效。核心贡献：

  1. 问题重新定义：从"基于历史"扩展到"基于历史 + 未来已知"，这是 TSSF 的本质。
  2. AliAttention 机制：通过 attention 残差连接，把知识作为 attention bias 注入。
  3. 未来强调训练：span masking 防止模型作弊，提高泛化。
  4. 实际业务价值：GMV 覆盖率 +4.73%，是 AIOps 落地的标杆。

对实践者的启示：

  1. 领域知识是 AI 模型的胜负手——纯数据驱动方法忽视业务规则。
  2. 防止未来泄露是 TSSF 的关键——双向 attention + 朴素训练容易让模型作弊。
  3. span masking 是训练时强制利用特定信息的有效手段。

对研究者的启示：

  1. 将"先验知识"显式注入到 attention 是 Transformer 的新方向。
  2. 领域数据公开（TMS）能极大推动研究社区进步。
  3. 实际部署效果（GMV 提升）是检验 AIOps 工作的最终标准。
