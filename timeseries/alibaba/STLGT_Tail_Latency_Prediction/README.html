2. 研究背景

云原生计算已成为现代软件交付的基础范式，微服务和容器技术是其核心架构。IBM、阿里云等大型云提供商利用 Kubernetes 容器编排管理大规模生产系统。

与许多稳定的企业工作负载不同，教育平台工作负载通常包含调度驱动的突发：

  • 在线考试
  • 截止日期驱动的提交
  • 同步课堂活动

这些突发在短时间内急剧增加流量。弹性（elasticity）是云服务的关键特性——基于运行时指标（CPU、内存利用率）提供/回收资源，以满足 QoS 要求并控制运营成本。

典型的主动自动扩缩系统包含两个核心组件：

  1. 预测器（Predictor）
  2. 扩缩控制器（Scaling Controller）

虽然扩缩控制器的设计已被广泛研究（强化学习、actor-critic 等），但预测器建模能力在复杂微服务环境中仍受限。三个关键挑战：

  • C1：全局依赖建模受限：现有预测器要么忽略微服务调用依赖，要么只捕获局部邻域。
  • C2：突发感知时间建模不足：现有方法隐式假设负载平稳性，无法应对突发流量模式。
  • C3：可扩展性差、忠实度低：基于图神经网络或全局注意力机制的方法在大规模系统中计算开销巨大。

3.论文目的与解决的问题

本文提出 STLGT（Scalable Trace-based Linear Graph Transformer），目标是为每个 API 流构建一个 per-API 预测器，对 multi-step p95 尾延迟进行精准预测。

核心贡献：

  1. 基于追踪的跨度图抽象（Trace-based span graph abstraction）：从分布式追踪构建 API-specific span graph 与轻量级 span-to-graph 特征聚合。
  2. 结构感知线性图 Transformer（Structure-aware Linear Graph Transformer）：在线性图 Transformer 编码器中线性推理，捕获全局依赖传播。
  3. 解耦时空建模（Decoupled spatiotemporal modeling）：全局依赖编码与时序动态解耦，避免耦合的时空注意力。

4.核心原理

4.1 问题形式化

定义 1：微服务调用图（MCG）：

\[ \mathcal{G} = (\mathcal{V}, \mathcal{E}) \]

节点为 \(M\) 个微服务，边 \((s_i, s_j) \in \mathcal{E}\) 表示 \(s_i\) 可能调用 \(s_j\)。

定义 2：追踪与 Span：将时间离散化为固定长度 \(\Delta\) 的窗口。窗口 \(t\) 内观察到追踪 \(\mathcal{T}_t^{\mathcal{A}}\)，由一组 span \(\Sigma(t) = \{\sigma_{\tau,1}, \sigma_{\tau,2}, ..., \sigma_{\tau;|t|}\}\) 组成。每个 span 捕获一次调用事件：

\[ \sigma = (\text{traceID}, \text{spanID}, p, s, t_1, t_2) \]

span 通过父子关系连接形成根树。

定义 3：API 诱导 Span 图：

\[ \mathcal{G}^{\mathcal{A}} = (\mathcal{V}^{\mathcal{A}}, \mathcal{E}^{\mathcal{A}}) \]

对目标 API \(\mathcal{A}\)，节点为执行阶段的 stage 节点，\(\mathcal{E}^{\mathcal{A}}\) 表示调用关系。

问题：尾延迟预测：给定历史长度 \(L\) 和预测 horizon \(H\)，目标是学习预测器 \(f_\theta\)：

\[ \hat{y}^A_{t+1:t+H} = f_\theta(\Theta^{\mathcal{A}}_t, \mathbf{X}^{\mathcal{A}}_{t-L+1}, \mathbf{c}^{\mathcal{A}}_{t-L+1}) \]

4.2 跨度图构建

Algorithm 1 描述了从追踪构建跨度图的过程。对每条 span \(\sigma \in \Sigma(\tau)\)，将其起始节点 \((p, s)\) 添加到 \(\mathcal{V}^{\mathcal{A}}\)。对于同 trace 内的连接，保留调用者-被调用者对的 stage 节点。给所有从 \((p, s)\) 到 \((s, u)\) 的边添加反向边，形成 \(\mathcal{E}^{\mathcal{A}}\)。

4.3 特征构造

尾延迟标签：

\[ y^{\mathcal{A}}_{t} = \rho_q(\mathcal{T}^{\mathcal{A}}_t) \]

其中 \(\rho_q(\cdot)\) 返回 \(q\)-分位数（\(q=0.95\)）。

追踪公共向量：

\[ \mathbf{c}^{\mathcal{A}}_t = \phi_{\text{common}}(\mathcal{T}^{\mathcal{A}}_t) \]

汇总 API 级工作负载和性能信号：吞吐量、响应时间百分位、失败率。

span-specific 向量：

\[ \mathbf{u}^{\mathcal{A}}_{t,t} = \phi_{\text{span}}(\mathcal{T}^{\mathcal{A}}_t, v^{\mathcal{A}}_t) \]

捕获 span 内的细粒度时序结构：归一化的开始/结束时间、归一化的持续时间。

4.4 空间编码器：结构感知线性注意力

单次稀疏预混合：

\[ \tilde{\mathbf{A}} = \mathbf{A} + \mathbf{I}, \quad \tilde{\mathbf{D}} = \text{diag}(\tilde{\mathbf{A}}\mathbf{1}), \quad \mathbf{P} = \tilde{\mathbf{D}}^{-1} \tilde{\mathbf{A}} \] \[ \mathbf{K} = (1-\rho) \mathbf{I} + \rho \mathbf{P} \mathbf{K}, \quad \mathbf{V} = (1-\rho) \mathbf{I} + \rho \mathbf{P} \mathbf{V} \]

线性混合：

\[ \mathbf{D}_t = \text{diag}\left(\frac{1}{N}\tilde{\mathbf{Q}}(\tilde{\mathbf{K}}^T)\mathbf{1}\right) \in \mathbb{R}^{N \times N} \] \[ \mathbf{Z}^{\mathcal{A}}_t = \mathbf{D}^{-1}_t \left[ \mathbf{Y} + \frac{1}{N}\tilde{\mathbf{Q}}(\tilde{\mathbf{K}}^T) \mathbf{V} \right] \]

本地传播（GCN 分支）：

\[ \mathbf{Z}^{\mathcal{A}}_{f,t} = \tilde{\mathbf{D}}^{-\frac{1}{2}} \tilde{\mathbf{A}} \tilde{\mathbf{D}}^{-\frac{1}{2}} \mathbf{H}^{(0)}_t \mathbf{W}_L \]

融合与读取：通过两层 MLP 融合全局和局部表示，再通过 attention pooling 读取图嵌入。

4.5 时序解码器：TimesBlock

TimesBlock 来自 TimesNet：

  • 对输入做 FFT，识别 \(K\) 个主频。
  • 对每个频率做 2D 卷积。
  • 残差堆叠 B 个 TimesBlock。

4.6 分位数损失（Quantile Loss）

对 p95 尾延迟，使用分位数回归损失：

\[ \ell_q(y, \hat{y}) = \psi_q(y - \hat{y}), \quad \psi_q(u) = \max(qu, (q-1)u) \]

6.关键算法详解

6.1 跨度图构建（Algorithm 1）

核心步骤：

  1. 对追踪 \(\mathcal{T}_t^{\mathcal{A}}\)，为每条 span 添加 stage 节点到 \(\mathcal{V}^{\mathcal{A}}\)。
  2. 为每个 stage 节点 \(v = (p, s)\)，添加从 \((p, s)\) 到其连接子节点的边。
  3. 为每条边 \((p, s)\)，添加反向边。
  4. 对每个 stage 节点 \(v\)，用 \(\phi_{\text{span}}\) 聚合 span 特征，得到 \(\mathbf{u}^{\mathcal{A}}_{t, v}\)。
  5. 对每个 service 节点 \(v\)，用服务级指标 \(\mathbf{m}^{\mathcal{A}}_t\) 填充 \(\mathbf{m}^{\mathcal{A}}_{t, v}\)。

6.2 伪代码：STLGT 预测

Algorithm 2: STLGT Tail-Latency Prediction
Input: MCG G = (V, E), service metrics {m_t}, traces of A in window t: T_t^A
Output: 未来 H 步尾延迟预测 y_hat_{t+1:t+H}

1. // 跨度图构建
2. G^A, X_t^A, c_t^A ← SpanGraphConstruction(G, {m_t}, T_t^A)
3. // 空间编码
4. H^(0) ← MLP_in(X_t^A)              // 投影
5. Z_t^G ← LinearAttn(H^(0))           // 全局
6. Z_t^L ← GCN(H^(0))                  // 局部
7. g_t^node ← Fusion(Z_t^G, Z_t^L)     // 融合
8. g_t^code ← AttentionPool(g_t^node)  // 读取
9. c_bar_t ← Fusion(g_t^code, c_t^A)  // 上下文融合
10. // 时序解码
11. G_t ← stack(g_t^code, c_bar_t) over last L windows
12. U_t^(0) ← MLP_in(G_t)
13. for b in 1..B:
14.    U_t^(b) ← U_t^(b-1) + TimesBlock(U_t^(b-1))
15. // 预测
16. y_hat_{t+1:t+H} ← MLP_out(U_t^B)
17. return y_hat

7.2 训练设置

  • 优化器：AdamW (lr=1e-3, weight_decay=1e-5)。
  • Batch size：32。
  • 训练轮数：至 temporal leakage 出现时停止。
  • Early stopping：基于验证集 MAE。
  • 数据划分：时序切分 70%/15%/15%。

7.3 模型配置

  • 隐藏维度 d：32（与基线一致）。
  • Spatial encoder：1 个 Sec. 4.1 + Sec. 4.2 block。
  • TimesBlock：B=2 堆叠层，K=3 主频。

  • STLGT 在 4 个数据集上一致优于所有基线。
  • 相比 PERT-GNN，MAPE 改善 8.5%（Social Network）到 9.5%（Edu Platform）。
  • Edu Platform 改善最显著（教育场景突发模式），平均改善 8.5%。

8.2 消融实验（RQ2）

变体Social Network MAPEHotel Reservation MAPEAlibaba MAPEEdu Platform MAPE
STLGT w/ Standard Attention10.30%9.86%10.24%10.21%

关键发现：

  • 去掉 Local (GCN) 性能下降最大（11.88-12.16%），说明局部依赖关键。
  • 用 Standard Attention 略差于线性注意力（10.30 vs 10.12），说明线性化在大规模图上有效。
  • 用 Linear Decoder 替换 TimesBlock 性能显著下降（12.41%），说明时序建模对突发模式至关重要。

9.消融实验

9.1 隐藏维度 d 的影响

d 从 16 → 128 的扫描：

  • DSB 上：d=32/64 时 MAPE 稳定，d=16 表现较差。
  • Edu Platform 上：d=32/64 最优，d=16 性能差，d=128 没有提升。
  • 结论：d=32 在精度和效率间平衡最佳。

9.2 工作负载模式的影响

Edu Platform 在周期性工作负载中，STLGT 优于 FastPERT 0.16%（MAPE）。但 STLGT 的 MAPE 更低，说明在 burst-sensitivity 方面更强。

12.局限性与未来工作

  1. 依赖调用路径稳定性：假设 API 的主导调用路径在评估期内稳定。如果调用路径切换（服务插拔、路由变更），需要重训。
  2. 未结合 trace / log / metric 三源：未来可融合多模态信息。
  3. 无强化学习闭环：未与扩缩控制器联合优化。

12.1未来工作

  • 引入拓扑漂移在线适应。
  • 结合 trace / log / metric 三源数据。
  • 与扩缩控制器的强化学习联合优化。
  • 支持亚秒级粒度的更细预测。

13.总结与启示

STLGT 是首个面向微服务尾延迟预测的可扩展线性图 Transformer，在 4 个数据集上全面超越 PERT-GNN/FastPERT。核心贡献：

  1. 追踪感知的跨度图抽象：从分布式追踪构建 API-specific span graph。
  2. 结构感知线性图 Transformer：避免 \(O(N^2)\) 全局注意力，扩展到大规模微服务。
  3. 解耦时空建模：全局依赖 + 时序动态分开建模，避免耦合注意力。

对实践者的启示：

  1. 线性注意力是大规模图的必选——\(O(N^2)\) softmax attention 不可扩展。
  2. 解耦时空建模比耦合时空注意力更高效。
  3. 追踪数据蕴含丰富结构信息，是优于纯 metric 的输入。

对研究者的启示：

  1. 把"per-API 单独建模"作为大系统的可扩展手段。
  2. TimesBlock 等周期感知时序模块对突发负载建模更鲁棒。
