核心问题在保留 DTW/k-NN 分类精度的同时，构造一种 指数级加速、可处理超长/近似重复/非等长 序列的距离度量。

2. 研究背景

距离度量是时序分类/聚类/异常检测的基石。常用距离的痛点：

距离优点缺点
Euclidean最快、O(m)对相位、噪声极敏感，分类精度差
DTW分类精度 SOTAO(m²)，长序列爆栈；不支持非等长；warping 难以下界
ERP / EDR / LCSS支持非等长常数项巨大，速度慢
Shapelet 距离 / SAX 距离可解释、维度低信息损失严重，分类不如 DTW

Gharghabi 等人观察到一个关键现象：在很多 真实工业/医疗/科学数据 中，同一类别的两条序列往往在很多"局部子序列"上都能精确匹配——DTW 找到的是"少数最佳匹配点"，而这些 非最佳 但同样相关的子序列反而能提供更鲁棒的判别信号。MPdist（Matrix Profile distance）正是利用 Matrix Profile（MP） 把这种"局部匹配数"转化为距离。

3. 论文目的与解决的问题

  1. 精度对齐 DTW：在大多 UCR/UEA 任务上 1-NN MPdist 与 1-NN DTW 精度持平或更好。
  2. 速度远超 DTW：利用 STOMP/STAMP 等线性时间 MP 算法，对 m = 10⁴ 长序列仍在秒级。
  3. 支持非等长：MP 自然对"长度不等的两条序列"定义良好，无需补零或剪裁。
  4. 对相位/重复/缺失稳健：在近似重复 (Self-similar)、含缺失片段的工业数据集上明显优于 DTW。

4. 核心原理（含数学公式）

4.1 Matrix Profile 与 PAB（Profile of Arrivals）

对单条时序 \(T = (t_1, \dots, t_m)\) 选取一个 子序列长度 \(\ell\)，在所有滑动窗口 \(T_{i:\,i+\ell-1}\) 上两两计算 z-normalised 欧氏距离，得到 Matrix Profile：

\[ P_T = \bigl( P_1, P_2, \dots, P_{m-\ell+1} \bigr), \quad P_i = \min_{j \ne i} \, \| \hat{T}_{i:\,i+\ell-1} - \hat{T}_{j:\,j+\ell-1} \|_2 \]

对应还有一个 Profile Index \(I_T(i) = \arg\min_{j\ne i} \dots\)。\(P_i\) 小说明 \(T_{i:\,i+\ell+1}\) 在 \(T\) 别处有近重复，\(P_i\) 大说明该子序列独特。

在 MP 基础上定义 Profile of Arrivals (PAB)：把长度为 \(\ell\) 的 query \(Q\) 滑过 \(T\)，记录每次最小距离的"到达位置"曲线：

\[ \mathrm{PAB}_T(Q) = \bigl( a_1, a_2, \dots \bigr), \quad a_k = \text{the } k\text{-th index in } T \text{ where the sliding distance to } Q \text{ ranks low} \]

4.2 MPdist 定义

对两条待比较时序 \(A, B\)（长度可不等），核心思想是：

用 \(A\) 滑过 \(B\)，记录每次最近距离；把"k 个最近距离的中位数"再对调一次（用 \(B\) 滑过 \(A\)），最后取最大。这种"最大中位距离"就是 MPdist。

形式化定义：

\[ \mathrm{MPdist}(A, B) = \max\Bigl( \mathrm{med}_{k}\!\bigl( \min\text{-dist}(A \text{ in } B) \bigr),\; \mathrm{med}_{k}\!\bigl( \min\text{-dist}(B \text{ in } A) \bigr) \Bigr) \]

其中 \(\min\text{-dist}(A \text{ in } B)\) 给出 \(A\) 每次滑到 \(B\) 不同位置上的最近距离集合，\(\mathrm{med}_k\) 取该集合中第 \(k\) 小的元素。论文取 k = ⌈0.05 × (|A| - ℓ + 1)⌉——即"前 5% 最近匹配"的中位数。这个 5% 的设计来自经验：太大会被噪点支配，太小对相位敏感。

4.3 关键性质

  • 非对称性：MPdist 定义天然非对称（先 \(A\) in \(B\)，再 \(B\) in \(A\)），取 max 后强制对称，论文 §3.2 证明该对称化满足三角不等式（实证上偏离极小）。
  • 支持非等长：只要滑动合法即可，\(A, B\) 长度差几十倍也能算。
  • 线性复杂度：MP 单序列 O(m²)（STOMP）或 O(m² log m) → 大数据集上比 DTW 加速 10³ – 10⁴ 倍。

6. 关键算法详解

6.1 子程序：滑动最小距离（naive）

def sliding_min_dist(A_sub, B):
    m = len(A_sub)
    n = len(B)
    dists = []
    for j in range(0, n - m + 1):
        d = euclid(A_sub, B[j:j+m])   # 配合 z-norm
        dists.append(d)
    return dists          # 一个 list

6.2 MPdist 主算法

def mpdist(A, B, m, k_frac=0.05):
    # 阶段 1: A 滑过 B
    d_AB = []
    for i in range(len(A) - m + 1):
        a_sub = z_norm(A[i:i+m])
        dists = sliding_min_dist(a_sub, B)
        d_AB.append(min(dists))
    k = max(1, int(k_frac * len(d_AB)))
    med_AB = sorted(d_AB)[k]            # 5% 分位

    # 阶段 2: B 滑过 A
    d_BA = []
    for j in range(len(B) - m + 1):
        b_sub = z_norm(B[j:j+m])
        dists = sliding_min_dist(b_sub, A)
        d_BA.append(min(dists))
    k = max(1, int(k_frac * len(d_BA)))
    med_BA = sorted(d_BA)[k]

    return max(med_AB, med_BA)

直接实现仍然是 O(m n²)；论文用 STOMP 一次性把 sliding_min_dist 替换成矩阵化计算，把"枚举所有子序列两两距离"压到 O(m n)。

6.3 STOMP 加速核心

STOMP 利用滑动窗口下相关函数的增量更新：

\[ Q_{i+1} = Q_i - T_{i} \cdot T_{i} + T_{i+\ell} \cdot T_{i+\ell} \]

其中 \(Q_i = \sum_{u=0}^{\ell-1} T_{i+u} \cdot T_{i+u}\)（同序列自相关）。这一递推把原来 \(O(m^2)\) 的内积降到 \(O(m)\)，整体 MP 计算由 \(O(m^2 \ell)\) 降到 \(O(m^2)\)。

7. 实现细节

  1. 子序列长度 ℓ：作者建议按数据自适应。论文里 m 默认 = max(10, ⌊min(|A|,|B|)/4⌋)。
  2. k_frac：5% 是经验值；当数据严重重复 (Self-similar) 时可调到 1%–2%，极端相位漂移时调到 10%。
  3. z-normalise：强制要求；不归一化时振幅会污染 min 距离。
  4. 对称化：始终取 max 而非 avg，对长序列差异显著的 A,B 表现更稳。
  5. 下界剪枝 (LB_MPdist)：和 LB_Keogh 类似，可剪掉不可能的最近邻，加速 1-NN。
  6. 硬件/语言：作者公开了 C++ 实现 mpdist，可在单核 1 秒内完成两条 1M 点序列距离。

11. 工程落地与部署

MPdist 在 2018 年后逐步进入工业界：

  • 工业异常检测：旋转机械振动信号里"自相似段"众多，MPdist 替代 DTW 后 100× 加速且更准。
  • 长时序检索：金融分钟级 K 线、脑电 EEG 等分钟-小时级信号，搭配 LB_MPdist 下界可以做亚秒级 top-k 检索。
  • 边缘嵌入式：作者公开的 C++ 实现 mpdist 静态库仅 ~50 KB，可在 ARM Cortex-M4 实时运行。

12. 局限性与未来工作

  1. 子序列长度 ℓ 选择：对 ℓ 敏感，作者建议自适应 sweep，但仍是经验调参。
  2. 三角不等式偏离：对称 max 不严格满足三角不等式，对基于 KD-Tree 的检索需要 LB_MPdist 兜底。
  3. 内存占用：STOMP 的 O(m²) 距离矩阵对 m > 10⁵ 的极长序列仍需外存策略。
  4. 多变量扩展：原始 MPdist 仅单变量，MMP / MPMulti 等多通道扩展到 2019 年后才逐步成熟。
  5. 学习化：后续工作尝试把 MPdist 嵌入神经网络可微损失（DeepMatrixProfile）以与深度模型联合训练。

13. 总结与启示

  • 范式意义：把"局部子序列匹配数"做成距离度量，开辟了"基于 Matrix Profile 的分类/检索"这一独立支线。
  • 对后续工作影响：直接启发后续 STOMP/STAMP/Valmod 等线性时间 MP 算法族，奠定 Time Series Insight Mining 的工具基础。
  • 对 AIOps 研究启示：KPI 序列的"自相似抖动"恰恰是 MPdist 的强项——比起 DTW，MPdist 在长时序 + 重复形态的故障分类上更准更快，可作为可解释距离 baseline。
