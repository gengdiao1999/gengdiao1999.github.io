核心问题在不做全序列相似度匹配的前提下，从时序中抽取能"解释类别"的最具判别性子序列（shapelet），并据此构造高精度、强可解释的分类器。

2. 研究背景

2009 年之前的 TSC 主要走两条路：

  1. 全局相似度（Whole-series）：DTW / Euclidean 距离 + 1-NN，简单但对相位/噪声敏感，且不告诉人"为什么"。
  2. 特征工程（Hand-crafted features）：统计/频域特征 + 经典 ML，依赖专家知识。

作者观察到，许多真实分类问题里类别的区分点其实出现在 局部子序列（local pattern） 上——比如 ECG 的某段 QRS 形态、传感器日志里的尖峰、火山地震记录的波形片段。这些"局部判别子序列"作者命名为 shapelet（脱胎于 Keogh 团队同年稍早 shapelet 聚类工作）。Shapelet 的核心承诺是：

  • 分类准确率与 1-NN DTW 相当或更好
  • 决策可解释（直接给出是哪段子序列在"说话"）
  • 速度显著快于 DTW（无需两序列两两比对）

3. 论文目的与解决的问题

论文要解决三个具体问题：

  1. 定义：什么是 shapelet？如何严格定义其"判别性"？
  2. 发现：如何在指数级候选中高效枚举并打分？
  3. 应用：如何把 shapelet 嵌入决策树、1-NN、boosting 等分类器，使其兼具精度与可解释？

作者同时强调：shapelet 不仅是分类器，更是一种新的 数据挖掘原语（primitive），可服务于聚类、异常检测、规则可视化等下游任务。

4. 核心原理（含数学公式）

4.1 符号定义

给定训练集 \(D = \{(T_i, y_i)\}_{i=1}^{N}\)，每条时序 \(T = (t_1, \dots, t_m)\)，类别集合 \(C = \{c_1, \dots, c_k\}\)。定义 候选 shapelet 为：

\[ S = (s_1, s_2, \dots, s_\ell), \quad 1 \le \ell \le m \]

其中 \(S\) 是从某条训练序列中切出的连续子序列，长度 \(\ell\) 可调。整篇论文的核心问题转化为：

从所有训练序列的有界长子序列集合中，找到使"信息增益"最大化的 \(S\)。

4.2 子序列到序列距离（subDist）

在用 shapelet 分类时，需要度量"某条时序 \(T\) 与 shapelet \(S\) 的接近程度"。最朴素的方案是把 \(S\) 在 \(T\) 上滑动，取最小 z-normalised 欧氏距离：

\[ \mathrm{subDist}(S, T) = \min_{1 \le j \le m-\ell+1} \; \sqrt{\sum_{u=1}^{\ell} \bigl( s_u - t_{j+u-1} \bigr)^2 } \]

论文中距离采用 z-normalised（即对参与比较的两段都做均值-方差归一化），以消除振幅/偏置差异。子序列归一化公式：

\[ \hat{S} = \frac{S - \mu_S}{\sigma_S}, \quad \hat{T}_{[j:j+\ell)} = \frac{T_{[j:j+\ell)} - \mu_{T_{[j:j+\ell)}}}{\sigma_{T_{[j:j+\ell)}}} \]

4.3 候选质量度量：信息增益（Information Gain）

对每个候选 shapelet \(S\)，将训练集投影到一维距离空间：

\[ d_i = \mathrm{subDist}(S, T_i), \quad i = 1, \dots, N \]

对距离值排序后，用 候选阈值 \(d^\*\) 把样本分成"在阈值之内 / 之外"两组，记为 split \( \pi(S) = (d^\*, \text{side}) \)。该 split 的信息增益定义为：

\[ \mathrm{IG}(S, \pi) = H(\mathcal{C}) - \frac{|\{i: d_i < d^\*\}| \cdot H_< + |\{i: d_i \ge d^\*\}| \cdot H_>}{N} \]

其中 \(H(\mathcal{C}) = -\sum_{c \in C} p(c) \log_2 p(c)\) 是原始类别熵，\(H_<, H_>\) 分别是两个子集内的条件熵。取遍所有可能的 \(d^\*\) 即可得：

\[ \mathrm{IG}^\*(S) = \max_{\pi} \mathrm{IG}(S, \pi) \]

这个 \(\mathrm{IG}^\*(S)\) 就是 shapelet 质量的统一打分。

4.4 Shapelet 树（决策树版本）

把上述打分嵌入决策树：在每个内部节点，对落在该节点的样本子集枚举所有候选 shapelet，挑选 \(\mathrm{IG}^\*\) 最大的那个作为切分特征；递归直到叶节点纯度阈值。最终形成一棵 shapelet 决策树——既可分类，又可"读出判别子序列"。

4.5 Shapelet 变换（Shapelet Transform, ST）

ST 把每条时序 \(T\) 映射为它到 \(k\) 个最佳 shapelet 的距离向量：

\[ \Phi(T) = \bigl( \mathrm{subDist}(S_1, T), \mathrm{subDist}(S_2, T), \dots, \mathrm{subDist}(S_k, T) \bigr) \in \mathbb{R}^k \]

ST 把分类问题转化为"在 \(k\) 维距离空间里学习"，下游可以挂任意分类器（论文里用了 C4.5、贝叶斯网络、1-NN、SVM），往往能比原始 shapelet 树更准——这正是后来 LE/Shapelet-Transform 系工作的引子。

6. 关键算法详解

6.1 朴素枚举

for each T_i in D:
  for len = m_min .. m_max:
    for j = 1 .. (|T_i| - len + 1):
      S = T_i[j : j + len]            # 候选 shapelet
      for each T_k in D:
        d_k = subDist(S, T_k)         # 滑动最小 z-norm 距离
      sort d_k, find best split d* that maximises IG
      keep (S, d*, IG*) if best

复杂度：候选数 \(\sim N \cdot m^2\)，每个候选对每条序列做 \(O(m \cdot \ell)\) 距离计算 → 总计 \(O(N^2 m^4)\)，对长序列不可接受。

6.2 加速技巧

  1. Early abandon on distance：计算 subDist 时一旦累计平方差超过当前最佳距离，停止。
  2. Caching subDist：子序列距离通过归一化后可拆为几个标量（均值、方差、互相关），用预计算 + 增量更新避免冗余。
  3. Orderlines & pruning by entropy：Keogh 2010 后续工作将候选投影到"距离直方图"上做上界剪枝，把搜索量从 \(O(N^2 m^4)\) 降到 \(O(N m^3)\)。

6.3 Shapelet 决策树伪代码

def shapelet_tree(D, depth):
  best = None
  for S in candidate_pool(D):
    gains = evaluate_candidate(S, D)   # 遍历 d* 求 IG*
    if best is None or gains.IG* > best.IG*:
      best = (S, gains.d*, gains.IG*)
  if best.IG* < min_gain or depth >= max_depth:
    return LeafNode(majority_class(D))
  D_left  = {T: T in D if subDist(best.S, T) < best.d*}
  D_right = D - D_left
  return SplitNode(best.S, best.d*,
                   shapelet_tree(D_left,  depth+1),
                   shapelet_tree(D_right, depth+1))

7. 实现细节

  1. z-normalisation：必须做，否则振幅差异主导距离；论文建议在提取 \(S\) 与滑动窗口时都做。
  2. 长度范围：\(\ell \in [\ell_{min}, \ell_{max}]\)，默认取训练集最短序列的 5%–95% 分位。
  3. Shapelet 数量：ST 通常取前 10–50 个 IG* 最大的 shapelet。
  4. 下采样与候选数控制：大训练集先做交叉验证估计每类所需样本数，控制候选池 ≤ 10⁵。
  5. 早期停止：在 Shapelet 树中设 min_samples_split 与 max_depth 防过拟合。

8.4 关键结论

  • Shapelet 树在多数数据集上 超过或接近 1-NN DTW。
  • ST + 任意下游分类器几乎 全面超过 1-NN DTW。
  • 训练时间从 1-NN DTW 的 0 秒（懒学习）来看更慢，但分类时间从 O(N) 降到 O(k)，对大测试集加速显著。

11. 工程落地与部署

shapelet 路线在 2009 年后的工业落地主要见于：

  • 可解释告警 / 故障诊断：把 shapelet 视为"局部模式签名"，与运维告警系统结合，给出"为何被判为某类故障"。
  • 嵌入式 / 边缘场景：训练完的 k 个 shapelet + 距离阈值可以写成 几行 C 代码，单片机即可运行，资源占用远低于 NN。
  • 金融 / 医疗：心电图 QRS 形态、股价"头肩顶"子序列，给监管/医生提供"白盒"判据。
flowchart LR Sensor[传感器 / 业务日志] --> Cap[滑动窗口切分候选] Cap --> Sset[已训练 shapelet 库 S] Sset --> Dist[并行计算 subDist S_i, T] Dist --> Phi[特征向量 ΦT] Phi --> Model[轻量分类器] Model --> Action[告警 / 推送 / 入库] Model --> Expl[解释器: 输出最相关 shapelet]

12. 局限性与未来工作

  1. 候选爆炸：序列越长，候选数 O(m²) 迅速失控；论文给出的 orderline 加速是后续 LE（Learned Shapelets）、Fast Shapelets 的伏笔。
  2. 单变量假设：原始 shapelet 仅对单变量 T 工作；MTSC 需要扩展到多通道（见 Hills 2014、Shifaz 2020）。
  3. 对噪声敏感：低质量候选 shapelet 易在树中造成过拟合，需剪枝与正则化。
  4. 缺乏概率输出：决策树只能给硬标签，后续 LE Shapelet 与 RDST 等工作把 shapelet 嵌入概率框架。

13. 总结与启示

  • 范式意义：shapelet 是 TSC 第一个真正"可解释"的高性能基线，把"局部子序列"提升为一类独立原语。
  • 对后续工作的影响：HIVE-COTE、TS-CHIEF、ROCKET 都在不同程度上借鉴"距离特征 + 轻分类器"思路；近年 TimeMixer / TimesNet 等深度模型也常把 shapelet 当作可解释的 attribution 工具。
  • 对 AIOps 研究的启示：KPI 曲线里"某段抖动"即可视为 shapelet，运维判别"性能故障 vs 抖动"可直接借用 shapelet 决策树——在轻量、可解释场景里仍然是首选 baseline。
